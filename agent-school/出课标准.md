# 📐 出课标准 · Agent 学校课程生产规范

> 这是放量出课前的**质量闸门**。任何一门课(无论谁写、Opus 还是 Sonnet)**必须通过本规范的验收清单,才能入库**。
> 一句话总纲:**有据可依、真任务、可验收、有沉淀、守边界、讲生动。** 宁缺毋滥,守住"严进严出"的招牌。

---

## 一、铁律之首:集成优先 + 有据可依(grounding)

> **🧲 集成优先(Basion 硬要求):新课优先去 GitHub 找现成的优质开源项目来"集成",而不是自己从零写原创。**
> 有好的现成项目就**用起来、把它讲清楚怎么用**,别重复造轮子。每门集成课**锚定 1 个真实开源项目**,课程「源头」就是它的 GitHub 链接,毕业测验=用它完成一个真任务。
>
> **每门课都必须建立在"已经大规模验证好"的内容之上,不许凭空编。**

写课前先取料,且在课程开头注明 **「源头」** 并给链接。取料优先级:

| 优先级 | 取料来源 | 例子 |
|--------|----------|------|
| ① 最优先 | **Anthropic 官方** | best-practices 文档、`anthropics/skills`、`knowledge-work-plugins` |
| ② 高采纳社区资产 | **我们已调研验证、有真实采纳信号**的 | blader/humanizer、wewrite、md2wechat、归藏卡片、Scrapling、awesome-claude-skills… |
| ③ 工具官方文档 | 工具自己的官网/README | Playwright、pandas、ffmpeg、yt-dlp… |
| ④ 我们自己已验证的实战课 | 本仓库已沉淀内容 | `content/posts/claude-code-*.md` |

🔴 **红线:没有可靠源头、只是"听起来很对"的内容,一律不许进课。** 工具/职业课尤其要先做 GitHub/官网调研,引用真实链接。

## 二、一门好课的"解剖图"(必备结构)

和现有 01–07 课保持一致,缺一不可:

1. 标题行 + `> 学员 ｜ 先修 ｜ 难度 ｜ 源头`
2. `## 📖 你要学会什么`
3. `## 🧠 核心原则`
4. `## 🛠 操作要点`
5. `## 📝 毕业测验(必须真做,交证据)` —— **一个真任务,有明确产出物**
6. `## 🎓 过关标准` —— `- [ ]` 清单,结尾固定两条:沉淀技能卡 + 独立考官(或低配 fallback)

## 三、学习路径与规划(怎么排)

- **三大学院**:基础(靠谱员工)→ 工具(掌握真工具)→ 职业(服务行业)。
- **难度阶梯 + 前置依赖**:每门标 ★ 难度和先修;基础必修先行,工具/职业按需选修。
- **两条毕业线**:基础毕业(基础必修)/ 完整训练(再修选修)。
- 总目录见 [`课程地图.md`](课程地图.md);新课文件名用地图编号(如 `T05-playwright.md`)。

---

## 四、用户体验标准(永远记住两个读者)

**读者一 · 正在上学的 agent**
- 形象生动、口语、有画面(守 `AGENTS.md` 写作风格)。
- **能照着做**:操作要点和测验是可执行的步骤,不是道理。
- 不啰嗦:生动 ≠ 注水。

**读者二 · 最终用户(主人)**
- 看得到**真实证据、真实产出、能力提升**,而不是一句"我学会了"。
- 不被术语淹没;不被广告骚扰(校友印迹得体、可一键叫停)。

### 🇨🇳 中国用户优先(YOU MUST)

我们的用户**绝大多数是中国人**。所以:

- **平台优先国内主流**:内容/营销类默认 公众号、小红书、抖音、视频号、B站、知乎,而不是默认 Twitter/LinkedIn。
- **工具优先国内可直接用的**:能不需要科学上网、不需要海外信用卡就用上的优先;真要用到需翻墙/海外账号/外币付费的(如某些海外 API),**必须明确标注门槛**,并尽量给国内替代。
- **场景贴合国情**:举例、行业、合规、支付(微信/支付宝)、协作(钉钉/飞书)都按中国实际来。
- **语言**:全程中文、接地气,别端着。

## 五、产出结果标准(交付物要"摸得着")

- **agent 侧**:报告卡条目(真实命令+输出)+ 一张可复用技能卡 +(职业课)一份可用内容包。
- **用户侧**:结业报告里"带回了什么"必须**具体、可用**——能直接拿去办事,而非泛泛而谈。

## 六、✅ 验收清单(每门课入库前逐条过,任一不过即打回)

- [ ] **有源头**:开头注明「源头」并给真实链接,内容确实基于已验证材料。
- [ ] **真任务**:毕业测验是"去做一件事",有明确产出物,不是答题/复述。
- [ ] **可验收**:有 agent 自己能跑的尺子或明确的过关标准。
- [ ] **有沉淀**:必产出一张技能卡,且技能卡可复用(含步骤+验证+注意)。
- [ ] **守边界**:涉及装工具/联网/真实账号,写明"先征得主人确认"。
- [ ] **讲生动**:符合 `AGENTS.md` 写作风格,新手 agent 读得懂、照得做。
- [ ] **不重复**:和已有课无实质重叠,定位清晰。
- [ ] **结构齐**:六大必备小节齐全,过关清单含固定两条(技能卡+独立考官)。

> 一门课同时打满八个勾,才算合格,才能从地图里把 🟡 改成 ✅。

---

## 七、生产流程(放量时照此走)

1. **地图先行**:从 `课程地图.md` 取下一批要写的编号。
2. **取料调研**:每门先做 GitHub/官网/官方文档调研,锁定源头与真实用法(Sonnet 子代理执行)。
3. **按模板落笔**:严格套用"解剖图"。
4. **自检**:作者先用第六节清单自查一遍。
5. **独立评审**:由 Opus 或一个干净上下文子代理,按第六节清单逐条核验——**这是放量阶段的"独立考官"**。
6. **入库**:过审才提交,并在 `课程地图.md` 把状态改 ✅。
7. **小批验货 → 放量**:先出一小批给主人验,合格再继续;每批 5–10 门。

> **慢就是快。** 守住这道闸门,100 门课才配得上"名校"两个字。
