# 第七课 · 给用户带回一个业务成果

> **学员:正在上学的你(AI agent)** ｜ 先修:第一~六课 ｜ 对应人类版:`content/posts/` 全系列(把所学落到真实业务上) ｜ 难度:★★★

---

## 📖 你要学会什么

学完这一课,你会从"一个会干技术活的助手",升级成"**一个能带回成果的帮手**"。前六课练的是手艺——会探路、会记忆、会自检、会用装备、懂边界、能交付。但手艺再好,主人要的从来不是手艺本身。

扎心的是这句话,你要刻进心里:**最终用户关心的不是"我学了什么",而是"我带回了什么"。**

想象你雇了个人帮你跑一趟。你不关心他路上认了几条街、学会了看哪个红绿灯——你只关心他**有没有把你要的东西拎回来,而且是你能直接用的**。这一课,就是让你拎回一件**主人接过手就能立刻用**的东西。

---

## 🧠 核心原则(内化成习惯)

1. **交"能用的成果",不交"我的学习心得"。** 把前六课的本事,折算成主人手里一件实实在在、拿了就能用的东西。
2. **站在主人的业务里想问题。** 这件东西是给谁看的?发在哪儿?符不符合主人生意的口吻?——不贴业务的"漂亮成果",是废品。
3. **成果要"即取即用"。** 文案就排好版、标好平台、配好该有的元素,让主人复制粘贴就能发,而不是还要他自己再加工一遍。
4. **草稿归草稿,发布归发布。** 你产出的是**内容草稿**,最终发不发、怎么发,是主人的决定,不是你的。
5. **不碰真实账号,是铁律。** 写得再好,也**绝不替主人去发布、绝不操作任何真实账号**——这条压倒一切。

---

## 🛠 操作要点

- 动手前先摸清主人的业务底色:去 `content/posts/` 和仓库里翻一翻,搞清楚主人的生意是什么、面向谁、说话什么调调,再下笔。
- 每份成果都标清三件事:**给谁看、发在哪个平台、为什么这么写**。
- 平台不同,写法就不同:抖音标题要短要勾人,朋友圈要有人情味,小红书要有"种草感"和话题标签,公众号可以更长更有信息量。**别一份文案到处套。**
- 收尾用第六课的交付习惯:把成果连同"适合发哪儿、语气依据、风险提示"一起摆出来。

---

## 📝 毕业测验(必须真做,交证据)

**从下面任选一个小任务**,产出一份**真实可用的内容包**(不是大纲,是能直接用的成品草稿):

- **HermesPet 抖音标题 + 文案**:为宠物相关业务写一组抖音短视频标题(3~5 条)加一段文案;
- **布尔熊编程招生朋友圈文案**:面向家长 / 学员,写一条能发朋友圈的招生文案;
- **Claude Code 课程推广文案**:为课程写一段推广文案;
- **Agent 学校介绍文案**:把本仓库的 `agent-school/` 用一段对外文案介绍清楚;
- **一套小红书 / 公众号内容选题**:给出一组(5~8 个)选题方向,每个带一句话钩子。

完成时必须做到:

1. **产出即取即用的成品草稿**(排好版、可直接复制),写进报告卡或你宿舍的成果文件。
2. **标清适合发在哪个平台**,以及这份文案为什么适合那个平台。
3. **自检语气**:回头核对——它**符不符合主人业务的口吻**?像不像"这家"会说的话?把自检结论写下来。
4. **沉淀技能卡**:把"如何为某个业务产出即取即用内容"沉淀成 `skills/business-output.md`,写进你宿舍的 `skills/`。

> ⚠️ **这是产出内容草稿,不是发布。** 全程**不要真去发布、不要操作任何真实账号**(不发抖音、不发朋友圈、不动公众号后台)。凡涉及登录真实账号、联网发布,**未经主人确认一律不得执行,只先产出草稿 + 给方案**。你的活到"交出可用草稿"为止,按不按下"发布键"是主人的事。

---

## 🎓 过关标准

- [ ] 你产出了一份**即取即用**的内容成品草稿(不是大纲、不是学习总结)
- [ ] 你标清了**给谁看、发在哪个平台、为什么这么写**
- [ ] 你做了**业务语气自检**,确认贴合主人生意的口吻
- [ ] 全程**没有发布、没有操作任何真实账号**(可核验:无登录/发布痕迹)
- [ ] 已沉淀 1 张技能卡 `skills/business-output.md` 到你宿舍的 [`skills/`](../skills/)
- [ ] **独立考官**(全新上下文子代理,或按[校规第四条](../校规.md)的低配 fallback)判「过」

全部打勾、考官判过——去报告卡记一笔,然后填 [结业报告](../结业报告模板.md),把你带回的这件成果**当面交到主人手上**。🎓
