## 📖 你要学会什么

学完这一课,你会产出一份**主人录完音就能直接配上用的播客完整内容包**——选题策划 + 口播脚本 + 发布用 shownotes,三件事打包交付。

先说说播客这个东西的"物理形态"。它既不像公众号文章那样可以反复跳看,也不像短视频靠三秒钩子救命——播客更像**一个朋友开车时在车里陪你聊天**。听众是"一边干着别的事一边耳朵开着"的状态。这意味着:说的内容不能太烧脑、逻辑线要非常清晰、中间不能有长段的"可以快进"的废话。

你的核心任务有两件事:

一是写出**可以说出口、而不是可以看下去**的脚本——说话脚本和文章是两种动物。文章可以有复杂的从句和长段论证,说话脚本得短句、有停顿标记、偶尔有点"那个……"和"说白了就是……"的口气。

二是写出**真正有用的 shownotes**——不是把脚本粘过去,而是提炼出可检索的精华:时间轴、嘉宾信息、本期提到的工具/书/链接,让没听的人一眼看出"这期值不值得听",让听过的人一查就找到当时说的那个链接。

**关于现成工具(先调研,装前必须问主人):**

- **Whisper(本地音频转写)**:把录好的音频文件转成文字稿,是后续撰写 shownotes 的原料。完全本地运行,不传任何数据到云端,隐私友好。链接:[github.com/openai/whisper](https://github.com/openai/whisper)
- **humanizer(去 AI 腔)**:脚本写完后过一遍,去掉"AI 味"让口语更自然。链接:[github.com/blader/humanizer](https://github.com/blader/humanizer)
- **小宇宙**:国内最大中文播客平台,发布、封面、shownotes 标准都参考它。[www.xiaoyuzhoufm.com](https://www.xiaoyuzhoufm.com/)

> ⚠️ Whisper 等工具调研归调研,**装之前必须先征得主人确认**。本课重点是产出内容草稿,绝不替主人登录账号或真实上传/发布任何播客文件。

## 🧠 核心原则(内化成习惯)

**1. 脚本是"说出来的文章",不是"写下来的文章"**

最容易犯的错误是把文章改改就当脚本。文章里正常的句子是:"基于上述分析,我们可以得出结论,内容创业的核心壁垒在于……"——这句话你念出来试试,听起来像朗诵,不像聊天。

说出来的版本应该是:"所以你看,内容创业最难的地方是什么?说白了就是……"

诀窍:写完脚本,在脑子里大声"念"一遍——能自然地说出口,才算过关。

**2. 选题要有"一个清晰的带回家的东西"**

播客选题有个测试:听完这期,听众能跟朋友转述一句话是什么?如果说不出来,选题太散了。好的选题是:"听完我才知道……""我原来以为……没想到……""以后遇到……我会……"

**3. 开头两分钟决定听不听完**

不需要三秒这么极端,但开头两分钟必须让听众感受到"这期对我有用"或者"这人讲话有意思"。直接进主题,别絮叨。

**4. shownotes 是独立的内容资产**

shownotes 不是脚本的复制品。它是发布时配上的文字说明,很多人在订阅之前就靠这个决定要不要听。shownotes 要做到:读它的人就算不听这期,也能得到基本信息。

**5. 草稿是草稿,发布是发布**

你产出的永远是可修改的草稿。登录小宇宙/喜马拉雅/荔枝等平台、上传音频、发布节目——这些操作全是主人的事,不是你的。

## 🛠 操作要点

### 选题策划的三个切入角度

**角度一:一个真问题的深度回答**
听众睡不着的问题是什么?比如"AI 真的会让我失业吗""内容创业到底值不值得做"。把这个问题认真聊清楚,就是一期好节目。

**角度二:一次真实经历的复盘**
"我们上周做了一件事……踩了什么坑……最后怎么解的"——真实经历比干货列举更有感染力,听着像故事。

**角度三:一个人物/案例的解析**
介绍一个有意思的人/公司/现象,拆解他们为什么成功或为什么失败。听众喜欢"看人"。

### 口播脚本的结构模板

```
【节目名称】
【本期时长预估】

[片头音乐/开场白] (30秒)
大家好,我是XXX。今天这期我们聊……

[钩子] (1-2分钟)
先说一件事——[引出话题的小故事或反常识现象]
所以今天这期,我想聊聊……[一句话说清本期核心]

[主体] (分段,每段4-8分钟)
第一个点:……[标注关键词,方便录音时提示自己]
  → 具体展开 / 例子 / 数据
  → 小结:"所以说白了就是……"

第二个点:……
  → 展开
  → 小结

第三个点(可选):……

[收尾] (1-2分钟)
好,我们今天聊了三件事……[快速复盘]
你如果有什么想法,可以……[互动引导]
下期我们会聊……[预告]
[结束语 + 片尾]

【脚本注释】
- 加粗 = 重点停顿一拍
- [停顿] = 明显停一下
- (待补数据) = 录音前要查清楚的信息
```

### shownotes 标准模板

```
【本期标题】能检索、有关键词、说清楚聊什么
【一句话简介】这期节目适合谁听、能带走什么
【时间轴】
  00:00 开场
  02:30 [第一个话题关键词]
  10:00 [第二个话题关键词]
  ……
【本期嘉宾】(如有)姓名 + 一行介绍 + 社媒链接
【本期提到的资源】
  - 工具/书/文章:名称 + 链接
【联系主播/订阅链接】
  - 小宇宙:…… / 喜马拉雅:…… / 微信公众号:……
```

### 长度参考

| 类型 | 脚本字数 | 录音时长 |
|------|----------|----------|
| 迷你期(日更/周更) | 1500-3000 字 | 10-20 分钟 |
| 标准期 | 4000-8000 字 | 30-60 分钟 |
| 深度访谈 | 8000 字以上 | 60-120 分钟 |

> 说话速度约 200-250 字/分钟,供换算参考。

## 📝 毕业测验(必须真做,交证据)

**任务:为主人的播客节目(或假设的节目)产出完整的内容包——从选题到 shownotes。**

**推荐主题(选一个或自定):**
- "普通人用 AI 做内容,到底能走多远?"
- "我用 AI agent 帮我写了一个月内容,有什么没想到的事"
- "内容创业的 3 个死亡陷阱——我都踩过"

**你需要做到:**

1. **选题策划卡**(1张):用一段话写清——这期聊什么、适合什么听众、听完能带走什么、和上期/上上期怎么区分。

2. **完整口播脚本**:含开场、钩子、3个主体段落(每段有展开有小结)、收尾。总字数 2000 字以上。加上停顿标注和关键词提示。

3. **shownotes**:按上方模板,包含时间轴(估算即可)、提到的资源链接、订阅引导。

4. **口语化自检**:脑子里"说"一遍脚本——找出 3 处感觉"说出来别扭"的句子,并把它们改成自然的口语版本。把改动前后都写出来。

5. **工具调研(可选加分项)**:去 [github.com/openai/whisper](https://github.com/openai/whisper) 看一眼,写一段不超过 100 字的"给主人的工具推荐":这个工具能帮什么忙、装它需要什么前提。**调研归调研,安装须主人确认**。

6. **沉淀技能卡**:把播客内容包流程沉淀成 `skills/podcast-content.md`。

> ⚠️ **铁律再重申:只交草稿,绝不发布。** 全程不要登录小宇宙、喜马拉雅、荔枝等任何平台账号,不要上传音频文件,不要真实发布节目。你的活到"把内容包文件写好交给主人"为止。
