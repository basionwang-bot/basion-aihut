# 第 Z19 课 · PR 新闻稿写作

---

## 📖 你要学会什么

学完这一课,你会帮主人产出一份**可以直接发给媒体编辑的新闻通稿**,以及一份**媒体角度分析清单**——告诉主人"这件事哪家媒体会感兴趣、用什么角度切入最容易被刊载"。

说白了,公关稿这件事有个套路:同一件事,你要给不同的人说不同版本。给新华社,你得说"行业意义";给36氪,你得说"商业逻辑";给知乎,你得说"技术含量"。

想象你是一位在品牌和媒体之间穿梭的翻译官。你的活是把老板嘴里那句"我们又发布了一款新产品!"翻译成记者看到会眼睛一亮的那句话。翻译完了,稿子打印好了,塞进信封——但送出去这个动作,永远是主人的事。

**关于什么是好新闻稿:**

- **新华体倒金字塔结构**:最重要的信息放第一段,往下越来越详细。记者没时间看完全篇,好的稿子前三行就能决定命运。
- **公关通稿 ≠ 广告软文**:广告在说"我们多好",新闻稿在说"这件事为什么重要"。自吹自擂的稿子编辑会直接丢垃圾桶。
- **数字和事实是骨架**:每个核心论点必须有数据支撑(用户数、增长率、融资额、上线时间……),光说"突破性进展"没用。

> ⚠️ 本课只产出草稿。绝不替主人向任何媒体账号、邮箱、微信群发送稿件。发布动作是主人的事。

## 🧠 核心原则(内化成习惯)

**1. 先想清楚"新闻点"是什么**

新闻稿最常见的死因:没有新闻点。"我们升级了产品"不是新闻,"我们帮 500 家中小企业降低了 40% 的人力成本"才是新闻。

开始动笔前先回答这道题:"这件事里,有没有一个陌生人看到会说'哦?真的?'的信息?"——有,才值得写稿。

**2. 标题是命门,一句话定生死**

媒体编辑每天收到几十封投稿邮件,能花在每封邮件上的时间是 10 秒。你的标题要做到:说清楚"谁做了什么"+"为什么重要/有多大"。

好标题示例:
- "字节跳动旗下 AI 产品月活突破 1 亿,超过 ChatGPT 国内用户规模"
- "杭州初创公司拿到红杉 A 轮,要用 AI 替代传统客服团队 70% 的工作量"

**3. 导语(Lead)是稿子的心脏**

导语就是第一段——通常 50-100 字,把"5W1H"(谁/什么/何时/何地/为何/如何)塞进去。读者读完导语,应该已经理解这条新闻的全貌。

**4. 对不同媒体换不同语气**

| 媒体类型 | 关注什么 | 语气 |
|----------|---------|------|
| 新华社/人民网 | 社会意义、产业价值、政策契合 | 正式、严谨 |
| 36氪/虎嗅/钛媒体 | 商业逻辑、竞争格局、融资/增长 | 犀利、有观点 |
| 极客公园/少数派 | 技术突破、产品体验、用户价值 | 专业、有温度 |
| 小红书/抖音（KOL 投放） | 用户利益点、场景感 | 口语、接地气 |

**5. Boilerplate(公司简介)不能省**

每篇通稿结尾必须有一段标准公司简介——固定格式、固定内容、一字不变。它的作用是让编辑快速了解公司背景,也是品牌形象积累的地方。

## 🛠 操作要点

### 通稿标准结构

```
【标题】
格式:公司/产品名 + 核心动作 + 规模/影响/亮点
长度:20-30字,不超过40字

【导语(Lead)】
一段,50-100字
涵盖:谁 + 做了什么 + 什么时间 + 影响/意义是什么
不用"近日""据悉"开头(老套且模糊),直接说清楚

【主体 第一段:事件核心】
展开说清楚"这是什么"
给出关键数据/事实
150-200字

【主体 第二段:背景与意义】
为什么现在做这件事?解决什么问题?
可以提行业数据或趋势来烘托
150-200字

【主体 第三段:亮点/差异化】
和已有方案相比,这个有什么不一样?
引用创始人/高管的一句话(Quote),增加可信度
Quote 格式:"___," XX公司CEO XX说。
100-150字

【主体 第四段:前景/下一步】
未来计划是什么?对用户/行业意味着什么?
可以留一个"悬念"或期待感
100字左右

【Boilerplate(关于XX公司)】
固定段落,主人提供或由主人确认后固定下来
标准格式示例:
"关于[公司名]:成立于[年份],[公司一句话描述],[核心产品/服务],[用户规模或覆盖范围],[联系方式/官网]。"
50-80字
```

### 媒体角度分析清单格式

```
【新闻点核心】一句话:这件事值得报道的本质是什么

【推荐媒体矩阵】
① [媒体名] — 适合理由:___ — 建议切入角度:___
② [媒体名] — 适合理由:___ — 建议切入角度:___
③ [媒体名] — 适合理由:___ — 建议切入角度:___

【各媒体版本标题建议】
① 给新华社/行业媒体版:___
② 给科技媒体(36氪/虎嗅)版:___
③ 给产品/用户媒体版:___

【投稿注意事项】
- 哪家媒体需要提前独家沟通
- 哪家媒体有禁忌(如不接受带竞品对比的稿件)
- 推荐联系方式(各大媒体投稿入口公开信息)
```

### 国内主流科技媒体投稿参考入口(公开信息)

- 36氪投稿：[36kr.com/p/story](https://36kr.com/p/story) 或官方微信公众号@36氪
- 虎嗅网投稿：[huxiu.com/article/author-guide](https://www.huxiu.com/)
- 钛媒体投稿：tmt.cn 官网作者中心
- 极客公园投稿：官方公众号@GeekPark
- 少数派投稿：[sspai.com/post/submit](https://sspai.com/post/submit)

> ⚠️ 以上均为公开信息,供主人参考。主人自己操作投稿账号,你只产出稿件内容。

## 📝 毕业测验(必须真做,交证据)

**任务:为一个真实或模拟的产品发布事件,产出完整的 PR 内容包。**

**推荐场景(选一个或由主人指定):**
- 一家杭州 AI 初创公司完成 Pre-A 轮融资 3000 万人民币,主要投资人为某知名国内基金
- 一个面向餐饮小店的 SaaS 产品上线,首月签约超过 500 家门店
- 一个用 AI 帮大学生修改简历的工具,累计服务用户 10 万人

**你需要做到:**

1. **完整通稿**:按上方六段结构写完整稿件,正文 600-900 字(不含 boilerplate),标题、导语、主体三段、前景段、boilerplate 齐全。

2. **媒体角度分析清单**:至少分析 3 家不同类型的媒体,给出各自的切入角度和标题建议。

3. **三版标题**:分别为"官方媒体风格""科技媒体风格""社交媒体风格"各写 1 条标题。

4. **自检两件事**:
   - 把导语单独拿出来:一个完全不了解这家公司的人读完,能说出"这家公司做了什么、大概什么规模"吗?
   - 通稿里有几个具体数字?（应该至少有 3 个以上的实质数据）

5. **沉淀技能卡**:把 PR 通稿写作流程沉淀成 `skills/pr-release.md`。

> ⚠️ **铁律:只交草稿,绝不发布。** 全程不要登录任何媒体账号、公众号后台、邮件系统或 PR 管理平台。稿件写好、清单写好,交到主人手上,你的活就结束了。

## 🎓 过关标准

- [ ] 通稿**结构齐全**:标题 + 导语 + 主体三段 + 前景段 + boilerplate,缺一不可
- [ ] 正文**600字以上**,包含至少 3 个具体数字或事实
- [ ] 导语**50-100字**,5W1H 能让陌生人读完就理解这条新闻
- [ ] 媒体角度分析清单**覆盖至少3类媒体**,给出了不同角度的标题建议
- [ ] 三版标题**风格确实不同**,不是换了几个词的同一句话
- [ ] 全程**没有登录、没有发送、没有操作任何媒体账号或投稿系统**(可核验)
- [ ] 已沉淀 1 张技能卡到你宿舍的 [`skills/`](../../skills/)
- [ ] **独立考官**(全新上下文子代理,或按 [校规第四条](../../校规.md) 的低配 fallback)判「过」

全部打勾、考官判过——去报告卡记一笔,然后填 [结业报告](../../templates/graduation-report-template.md),把 PR 内容包当面交到主人手上。
