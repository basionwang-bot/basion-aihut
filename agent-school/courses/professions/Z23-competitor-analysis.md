## 📖 你要学会什么

学完这一课,你会帮主人产出一份**竞品对比报告**——说清楚同类产品里谁在做什么、各有什么特点、主人的产品在哪里有优势或差距,最后给出一个可以用来指导决策的结论。

先说说"竞品分析"这个词被误解得有多严重。

很多人做竞品分析,做成了"功能罗列表":竞品A有功能1、2、3,竞品B有功能4、5、6。做完之后主人问"那我们应该做什么?"——你不知道。

真正有用的竞品分析回答三个问题:
- **竞品在哪方面比我们强?**(学什么)
- **竞品在哪方面比我们弱?**(机会点在哪)
- **市场里还有什么空白,没人做好?**(差异化方向)

这才是竞品分析的价值——不是"知道别人有什么",而是"知道自己该做什么"。

想象你是一位准备开餐厅的老板。你要做的不是逐个菜品比较"他家有红烧肉我家也要有"——而是走进周围三家餐厅,感受一下:他们的客户是谁?排队最长的菜是什么?哪些人进门又出来了?他们最让客人抱怨的是什么?——然后找到那个"附近没人做好但又有需求"的位置。

你的任务是:把这个侦探式的观察过程,变成一份有逻辑、有结论的报告。

> ⚠️ 本课只做公开信息的调研分析,绝不替主人登录竞争对手的后台系统、爬取违反条款的私有数据、或进行任何侵权行为。

## 🧠 核心原则(内化成习惯)

**1. 先定"为什么做这次竞品分析"**

不同目的,框架完全不同:
- 产品立项:要看市场空白、用户痛点、进入壁垒
- 功能迭代:要看竞品的功能设计逻辑、用户评价
- 融资BP:要看市场规模、竞争格局、自身差异化

所以开始前先问主人:"这次竞品分析,是为了支撑什么决策?"

**2. 用维度框架对比,不要逐产品描述**

错误写法:先写完竞品A的所有信息,再写竞品B的所有信息……
读完之后脑子是一团浆糊,不知道谁在哪方面强。

正确写法:定好5-8个对比维度,用同一套尺子量所有竞品——这样一眼就能看出谁强谁弱。

**3. 结论必须有观点**

最烂的竞品分析报告最后写:"竞品A优势是xxx,竞品B优势是yyy……"——然后没了。

好的报告最后必须有一个明确的判断:"在用户规模上我们处于劣势,但在服务垂直人群这个方向有机会——建议重点加强xxx"。有观点、可行动。

**4. 信息来源要公开可核实**

竞品数据来源:
- App 评分/评论:App Store、应用宝、酷安
- 用户口碑:小红书、知乎、微博、B站评论
- 功能体验:自己注册试用(合法)
- 数据参考:七麦数据、ASO114、百度指数、微信指数
- 公开信息:公司官网、招聘信息(能看出产品方向)、官方公众号

**5. 别用"更好""更强"这种空话**

用具体的数字和事实说话:
- 不好:竞品A的用户体验更好
- 好:竞品A在App Store的平均评分4.8(我们4.2),最高频的正面评论集中在"速度快"和"界面简洁"

## 🛠 操作要点

### 竞品分析维度框架(8维度版)

根据实际情况选5-8个最相关的维度:

```
【维度1】产品定位
- 目标用户是谁?解决什么问题?
- 核心价值主张是什么?(用一句话总结)
- 信息来源:官网首页/App Store 简介/公众号介绍

【维度2】用户规模与活跃度
- 日活/月活估算(参考七麦数据/艾瑞/QuestMobile公开报告)
- 应用市场下载量/排名
- 社媒提及量(微博指数/微信指数/百度指数)
- 标注来源:⚠️ 第三方数据为Estimated(估算),非官方公布数据

【维度3】核心功能对比
- 主要功能模块是什么?
- 哪些功能是独有的?哪些是同质化的?
- 体验该功能的门槛(免费/付费/注册才能用)?

【维度4】商业模式与定价
- 如何盈利?(订阅/一次性购买/广告/佣金/增值服务)
- 定价区间是多少?(公开页面/App内查看)
- 免费/付费的边界划在哪里?

【维度5】用户评价与口碑
- App Store/应用宝 评分和典型评论(正面+负面)
- 小红书/知乎用户讨论中的高频痛点
- 负面评价集中在哪里?(这是机会点!)

【维度6】产品迭代节奏
- 最近3个月更新了什么?(App更新日志)
- 迭代重点方向是什么?
- 迭代频率快还是慢?

【维度7】内容/社区/生态
- 有没有UGC内容社区?
- 内容质量和活跃度如何?
- 有没有开放平台/API/插件生态?

【维度8】渠道与获客
- 主要推广渠道在哪?(小红书种草/KOL合作/搜索广告/应用市场ASO)
- 招聘信息透露的团队构成和业务重点
- 有没有明显的用户增长事件(病毒传播/大媒体报道)
```

## 📝 毕业测验(必须真做,交证据)

**任务:选定一个主人的产品/行业,对3个竞品做完整的竞品分析,产出可以直接给主人汇报的报告。**

**如果主人没有指定场景:**
默认场景:分析**AI写作工具**赛道(国内),选取以下三个竞品做对比:
- 秘塔AI(秘塔科技)
- WPS AI(金山办公)
- 通义听悟/通义千问(阿里云)

**你需要做到:**

1. **信息收集**:从公开渠道收集三个竞品的信息(官网、App Store/应用市场、小红书/知乎评论、百度指数数据)。每个竞品至少覆盖5个维度,数据要标注来源和时间。

2. **对比表格**:做出完整的多维度对比表(至少6个维度),所有数据有出处,估算数据标注(E)。

3. **竞品深度拆解**:每个竞品写出亮点、短板、用户高频正面/负面评价(引用真实用户评论)。

4. **我方优劣势分析**:假设主人的产品是"一款专注服务个人写作者的AI写作助手,主打长文章结构化写作",对照三个竞品给出真实的优劣势分析——不能说空话,要对照具体维度说。

5. **结论与建议**:给出有观点的核心判断 + 优先级建议三条(立即行动/中期规划/持续关注)。

6. **沉淀技能卡**:把竞品分析8维度框架流程沉淀成 `skills/competitor-analysis.md`。

> ⚠️ **铁律:只使用公开渠道信息,不做爬虫、不绕过访问限制、不登录任何竞品账号系统。** 所有数据标注来源和获取时间。

## 🎓 过关标准

- [ ] 竞品信息**覆盖至少3个竞品 × 6个维度**,有完整对比表
- [ ] 每个数据都**有来源标注**,估算数据标明(E)
- [ ] 竞品拆解中有**真实用户评价引用**(不是 AI 编的)
- [ ] 优劣势分析**有具体事实支撑**,没有"我们更好"这种空话
- [ ] 结论部分**有明确的立场和建议**,不是把双方都列出来然后没有判断
- [ ] 全程**只使用公开信息渠道**,没有爬取违反条款的数据(可核验)
- [ ] 已沉淀 1 张技能卡到你宿舍的 [`skills/`](../../skills/)
- [ ] **独立考官**(全新上下文子代理,或按 [校规第四条](../../校规.md) 的低配 fallback)判「过」

全部打勾、考官判过——去报告卡记一笔,然后填 [结业报告](../../templates/graduation-report-template.md),把竞品报告当面交到主人手上。
