## 📖 你要学会什么

学完这一课,你会产出一份**主人可以直接拿去做 B 站视频的内容包**——选题方案 + 视频脚本 + 标题封面建议,三件事打包交付。

先聊聊 B 站这块地方。如果说抖音是地铁里的街头表演,B 站更像一所**弹幕声不断的大学课堂**——观众是来认真学东西、认真看好内容的,他们愿意给你二三十分钟,但他们不容忍水货。

B 站的社区有几个独特的"基因":

- **有梗**:B 站用户有自己的"母语"——梗、弹幕文化、熟悉的选题切入角度。上来就正经端着,观众会觉得你"不是自己人"。
- **不水**:B 站的好视频有一个共性:信息密度高,每一分钟都有料,不说废话。水视频会被弹幕喷"这集水了"。
- **长视频有结构**:B 站的长视频需要章节感——有明确的"开头建立期待、中间逐层推进、结尾总结升华"的结构,像上了一堂完整的课。

你的任务是帮主人产出"**B 站用户看了会觉得有收获**"的视频内容,而不是"把公众号文章念一遍"。

**关于现成工具(先调研,装前必须问主人):**

- **cn-content-matrix**:真·风格迁移,把一个主题改写成 B 站/小红书/公众号/抖音原生口吻。链接:[github.com/fullstackcrew-alpha/skill-cn-content-matrix](https://github.com/fullstackcrew-alpha/skill-cn-content-matrix)
- **content-research-writer**:边写边查资料、加引用、强化开头钩子、逐段给反馈,还保留你的声音。链接:[github.com/ComposioHQ/awesome-claude-skills](https://github.com/ComposioHQ/awesome-claude-skills)
- **claude-blog**:整条长文流水线,会逐条 fetch 引用源做事实核查、卡 AI 味。改造成长视频脚本流程同样适用。链接:[github.com/AgriciDaniel/claude-blog](https://github.com/AgriciDaniel/claude-blog)

## 🧠 核心原则(内化成习惯)

**1. B 站选题:有搜索量 + 有切入角度**

B 站和抖音不一样:抖音主要靠算法"推",B 站靠搜索"找"也占很大比例。所以选题要考虑两件事:
- 这个话题有没有人在 B 站搜?(有实际需求)
- 我的切入角度和已有视频有什么不同?(差异化)

好的 B 站选题公式:`有人搜的话题 × 独特切入角度`。比如不是"介绍 AI agent",而是"用 AI agent 做完一整天工作的真实体验——包括失败的部分"。

**2. 开头 30 秒:建立期待,不是废话**

B 站用户会给你更多时间,但开头 30 秒他们会判断"这个 UP 主值不值得订"。这 30 秒要做的事只有一件:让观众知道**看完这个视频他们会得到什么**。

不要说:"大家好我是 XX,今天的视频我们来聊聊 AI。"
要说:"今天我把一整天的工作全部交给 AI agent 来做,包括回邮件、写文案、做数据整理——结果有些让我很意外,有些让我有点担心,今天全部说给你听。"

**3. 中间:章节感 + 信息密度**

B 站长视频的脚本需要清晰的章节,每个章节有自己的小高潮。一个好的结构大概是:

- 开头建立期待(30秒-1分钟)
- 主体分 3-5 个章节,每章节有结论有例子
- 章节之间有过渡(别硬切)
- 结尾总结+行动号召

**4. 梗和弹幕感**

B 站用户喜欢在某个时刻"发弹幕"。你可以在脚本里有意埋几个"弹幕点"——比如说一句让人会心一笑的话、一个反转、一个让人想说"哈哈哈真的"的场景。这不是装可爱,是在和 B 站的社区文化对话。

**5. 草稿是草稿,发布是发布**

同 Z01 铁律:你的活到"把内容包写好交给主人"为止。登录账号、发布投稿——全是主人的事,永远不是你的事。

## 🛠 操作要点

### B 站视频的三种常见类型(按主人需求选)

| 类型 | 适合主题 | 时长 | 特点 |
|------|----------|------|------|
| **教程/干货型** | 工具使用、技能学习 | 5-15分钟 | 信息密度高,有步骤感 |
| **测评/体验型** | 工具对比、真实使用体验 | 8-20分钟 | 有客观分析,有真实场景 |
| **观点/分析型** | 行业趋势、问题讨论 | 10-30分钟 | 有立场有论据,有深度 |

### B 站标题的写法

B 站标题有几个特点:
- **含关键词**:用户搜的词要在标题里,比如"AI agent""Claude Code"
- **有信息量**:让人一眼看出"这讲什么"
- **可以稍长**:B 站标题比抖音允许更长,可以到 30-40 字
- **别标题党**:B 站用户对标题党的容忍度很低,会直接踩

好标题公式:`核心关键词 + 独特角度/结果`

示例:
- "用 AI agent 完成一天工作的真实体验:三件事超出预期,一件事让我皱眉"
- "为什么你的 AI 助手越用越让你失望?问题出在这里"
- "Claude Code 实测:普通非技术用户真的能用吗"

### 封面建议规范

B 站封面是横屏图(16:9),常见做法:
- 主体人物/场景 + 大字标题(字数不超过10字)
- 深色背景 + 亮色文字,视觉冲击强
- 左侧放主图,右侧放标题(符合人眼阅读习惯)

### 视频脚本的结构模板

```
【开头(0:00-0:30)】
建立期待:今天要做什么/你会看到什么
埋悬念:"有一个结果让我很意外"

【章节一(0:30-X:XX)】
小标题:清晰说明这节讲什么
内容:结论 → 例子 → 小结
弹幕点:一句让人想评论的话

【章节二】...

【章节三(最后一个章节)】
通常是"最重要/最出乎意料/最有价值"的内容
放在这里是为了让人看到最后

【结尾】
总结一句话
行动号召:"一键三连""评论告诉我你的看法""下期会讲……"
```

### 内容包的五件套(每次都要凑齐)

```
【选题方案】话题 + 切入角度 + 和已有内容的差异化
【标题备选】3条,每条含核心关键词
【内容定位】视频类型 / 目标时长 / 目标受众
【完整视频脚本】
  - 开头(含时间节点建议)
  - 各章节脚本(有弹幕点标记)
  - 结尾
【封面建议 + 自检结论】封面文案建议 + 信息密度自评(每分钟有几个实质内容点)
```

## 📝 毕业测验(必须真做,交证据)

**任务:为主人仓库的某个主题,产出完整的 B 站视频内容包。**

**推荐主题(选一个或自定):**
- "普通人用 AI agent 的真实一天——含失败的部分"
- "Claude Code 上手指南:非技术用户视角"
- "为什么 AI 越用越让人失望?三个根本原因"

**你需要做到:**

1. **产出五件套**(见上方操作要点):选题方案 + 标题备选 × 3 + 内容定位 + 完整脚本 + 封面建议 + 自检结论。

2. **脚本长度**:完整脚本字数 1500-3000 字,对应约 5-10 分钟视频。

3. **章节自检**:脚本要有至少 3 个清晰章节,每章节有小结论。

4. **弹幕点**:脚本里要有至少 2 个"弹幕点"并标注出来(说明为什么这里会引发弹幕)。

5. **工具调研(可选加分项)**:去 [github.com/fullstackcrew-alpha/skill-cn-content-matrix](https://github.com/fullstackcrew-alpha/skill-cn-content-matrix) 看一眼,写一段不超过 100 字的"给主人的工具推荐"。**调研归调研,安装须主人确认**。

6. **沉淀技能卡**:把 B 站视频内容包流程沉淀成 `skills/bilibili-content.md`。

> ⚠️ **铁律再重申:只交草稿,绝不发布。** 全程不要登录任何 B 站账号、不要操作任何投稿系统。你的活到"把五件套文件写好交给主人"为止。
