## 📖 你要学会什么

学完这一课,你会帮主人产出一份**可以真正带来搜索流量的 SEO 文章内容包**——关键词分析 + 内容大纲 + 成稿 + 事实核查,四个环节打包交付。

先解释清楚 SEO 这件事在国内的特殊性。

如果你听说过"SEO"这个词,脑子里可能浮现的是谷歌和英文世界的那套打法。但国内的搜索生态和国外差很远:

**国内主要搜索入口有三个**,每个都要单独对待:
1. **百度**:传统网页搜索的绝对老大,年纪大一点的用户、偏实用性查询的主战场
2. **微信搜一搜**:公众号/视频号内容都会被索引,年轻用户、品牌词、知识类查询的重要入口
3. **小红书搜索**:年轻女性用户、美妆/生活/好物/旅行类选题的流量来源

你的任务是帮主人**从关键词开始,一路生产到可以发布的成稿**。不是随便写一篇"说了关键词"的文章,而是真正对搜索引擎友好、同时对读者也有用的内容。

## 🧠 核心原则(内化成习惯)

**1. 关键词是"需求的化石"**

每一个被人大量搜索的关键词背后,都有一个真实的需求。"AI 写作软件"这个词背后的需求是"我想找能帮我写东西的工具"。"AI 写作软件哪个好用"这个更长的词背后是"我想对比选一个"。

挖关键词的本质是**挖需求**——不是找字面相似的词,是找"搜这个词的人想解决什么问题"。

**2. 国内 SEO 的三个"收录入口"要分开对待**

- **百度**看的是:原创度、更新频率、外链权重、页面加载速度、是否有 HTTPS
- **微信搜一搜**看的是:公众号账号权重、内容阅读量、关键词在标题/正文中的出现
- **小红书搜索**看的是:标题关键词、话题标签、互动率(点赞/收藏/评论)

同一篇内容分发到三个渠道,优化重点不一样。

**3. 事实核查不是可选项**

SEO 文章里有数据、有对比、有"最新"字样的地方,读者和搜索引擎都会查证。一旦数据过时或有错,不但影响用户体验,平台还可能降权。所以每篇文章写完**必须过一遍事实核查清单**。

**4. 不要为关键词写作,要为读者写作**

矛盾吗?不矛盾。最好的 SEO 文章是:读者觉得有用,因此停留时间长、收藏率高——而这些行为信号反过来让搜索引擎觉得这篇内容质量好,继续推。所以"写好文章"和"做好 SEO"本质上是同一件事。

**5. 草稿是草稿,发布是发布**

你的活到"把四件套文件写好交给主人"为止。提交给百度收录、在公众号发布、上传到小红书——全是主人的事。

## 🛠 操作要点

### 第一步:关键词分析(你能做的部分)

即便没有付费工具账号,你可以用以下方式挖关键词:

**方法一:搜索框联想**
在百度/微信搜一搜/小红书搜索框里输入主题词,看下拉联想——这些联想词都是真实被大量搜索的词。

**方法二:相关搜索**
百度搜索结果页最底部有"相关搜索"，这 8 个词是当前流量最高的相关词。

**方法三:竞争对手标题分析**
搜索主题词,看前 5-10 个排名结果的标题用了什么关键词——这是已经被验证有搜索流量的词组。

```
【关键词分析表(每篇文章写之前填一张)】

核心词:_____(文章最想覆盖的词)
搜索意图:搜这个词的人,最想解决什么问题?
相关长尾词:
  - _____ (预估月搜量:高/中/低)
  - _____ (预估月搜量:高/中/低)
  - _____ (预估月搜量:高/中/低)
竞争强度:弱/中/强(看百度前10是否有大平台占据)
写作策略:新手/进阶/系统对比/工具推荐/最新解读
目标发布渠道:□百度 □微信搜一搜 □小红书搜索
```

### 第二步:内容大纲(关键词驱动)

大纲要做两件事:让读者能按需跳读,让搜索引擎能识别主题覆盖广度。

```
【SEO 文章大纲模板】

标题(含核心词):_____
副标题/摘要(100字内,含1-2个关键词):_____

H2 小节一:[含长尾词]
  - 要点1
  - 要点2

H2 小节二:[含长尾词]
  - 要点1
  - 要点2

H2 小节三(可用"常见问题Q&A"格式,抓"问句类搜索"):
  Q:_____
  A:_____(简洁直接,50字内)
  Q:_____
  A:_____

结尾:行动号召 or 下一步资源推荐

关键词密度目标:核心词在全文出现 3-5 次(自然出现,不堆砌)
```

### 第三步:成稿写作要点

```
标题写法:
  ✅ 把核心关键词放在标题前半段
  ✅ 30字以内(百度显示不会截断)
  ✅ 有信息量(说清楚这篇讲什么)
  ❌ 不要标题党(内容撑不住的标题会增加跳出率)

正文写法:
  ✅ 开头200字内说清楚这篇讲什么、能解决什么问题
  ✅ 每个H2小节都有实质内容(不是一行话就过去)
  ✅ 数据注明来源年份(如"根据2024年百度搜索大数据")
  ✅ 图片加 alt 文字(百度能读图片描述)
  ❌ 不要复制粘贴其他网站内容(原创度低会被降权)
```

### 第四步:事实核查清单

```
每篇文章发布前,逐一核查以下几项:

□ 文中引用的数据是否有来源?来源是否可访问?
□ 文中说"最新"的内容,发布时还是最新的吗?
□ 文中提到的工具/产品/服务,链接是否有效?价格/功能是否和现在一致?
□ 文中有没有绝对化表述("唯一""最好""100%")——这类说法容易被打假,建议改掉
□ 如果涉及医疗/法律/金融建议,是否有免责声明?

核查结果记录:
  ✅ 通过 / ⚠️ 已修改 / ❌ 删除该段
```

### 国内三大渠道的发布优化差异

| 渠道 | 核心优化点 |
|------|------------|
| 百度 | 标题含关键词 + 文章 1000 字以上 + 独立域名/平台权重 + 提交收录 |
| 微信搜一搜 | 标题关键词 + 公众号历史权重 + 文章阅读量 + 被转发 |
| 小红书搜索 | 标题前 10 字含关键词 + 话题标签 + 收藏率 + 图文质量 |

## 📝 毕业测验(必须真做,交证据)

**任务:为主人选定一个关键词,产出从关键词分析到成稿的完整 SEO 内容包。**

**推荐主题(选一个或自定):**
- "AI 写作工具哪个好用"(工具推荐类,长尾词多)
- "如何用 AI 做内容运营"(方法论类,百度+微信搜一搜均有需求)
- "小红书运营技巧 2025"(时效类,小红书搜索也有流量)

**你需要做到:**

1. **关键词分析表**:完整填写核心词 + 搜索意图 + 3条长尾词 + 竞争强度 + 目标渠道。用"搜索框联想"或"相关搜索"的方式调研,写出调研过程(去了哪里搜、看到了什么)。

2. **内容大纲**:按模板写完整大纲,包含标题(含核心词)+ 至少 3 个 H2 小节 + Q&A(2组)+ 结尾。

3. **完整成稿**:按大纲写完整正文,1200 字以上。核心关键词自然出现 3-5 次(不堆砌)。

4. **事实核查**:对照核查清单逐项过一遍,记录每一项结果(通过/已修改/删除)。

5. **渠道适配建议**:这篇内容最适合投哪个渠道?理由是什么?如果要发到第二个渠道,哪里需要调整?

6. **沉淀技能卡**:把 SEO 内容生产四步流程沉淀成 `skills/seo-content.md`。

> ⚠️ **铁律再重申:只交草稿,绝不发布。** 全程不要登录百度资源平台、微信公众号后台或小红书账号,不要提交收录、不要真实发布内容。你的活到"把内容包文件写好交给主人"为止。
