## 📖 你要学会什么

学完这一课,你会产出一份**主人接过去就能用的小红书内容包**——爆款笔记草稿 + 配图卡方案,两件事打包交付。

先说说小红书这块地方的"地形"。如果公众号像一本杂志,小红书就更像**闺蜜在茶桌上的安利**。不是讲道理,是分享;不是告诉你"应该怎样",是"我试过了,超好用,给你看"。这种氛围决定了写法:要真实、要有温度、要让人觉得"这好像在说我"。

你的任务是让主人的内容"**像一个真实的小红书用户写的**",而不是"AI 帮公司写的推广文"。

## 🧠 核心原则(内化成习惯)

**1. 先搞懂小红书的"三板斧":标题 emoji + 痛点 + 种草**

想象你走进一家超市,货架上摆满了笔记。读者的手指在屏幕上滑,每条笔记只有**不到 1 秒**的机会抓住眼球。小红书的流量逻辑就是这样冷酷。

所以小红书的"三板斧"缺一不可:
- **标题先放 emoji**:不是装饰,是"色彩信号",帮读者在信息流里一眼定位你的类目。比如 🍵 代表生活方式、💼 代表职场、✨ 代表美妆好物。
- **戳到痛点**:开头 1-2 行必须让人觉得"这说的就是我",比如"熬夜改 PPT 熬到凌晨三点,发现可以让 AI 五分钟搞定??"
- **种草而非广告**:说"我用了 XX 之后……"而不是"XX 产品功能包括……"。体验感比参数重要。

**2. 小红书是竖屏手机上看的**

这意味着:段落每 2-3 行换一行、多用换行而不是长段落、配图是 3:4 竖图(不是横图)、正文里适当插入 emoji 作为分隔符。

**3. 评论区互动是内容的一部分**

高质量的笔记会在结尾埋一个"互动钩子"——比如"你有没有遇到过这种情况?评论告诉我💬"。这不是废话,是让算法把你推给更多人的信号。

**4. 草稿是草稿,发布是发布**

同 Z01 铁律:你的活到"把内容包写好交给主人"为止。登录账号、发布、操作后台——全是主人的事,永远不是你的事。

## 🛠 操作要点

### 小红书爆款笔记的结构

把小红书笔记想象成一个**三明治**:

```
[面包片-上] 标题:emoji + 痛点/悬念/利益点,30字以内
[馅料]      开头:前2行,放钩子——要么戳痛点,要么给惊喜
            中间:3-5个"好物理由"或"步骤",每条短句+换行
            配图说明:每张图对应什么内容
            互动钩子:一句引发评论的问句
[面包片-下] 话题标签:#相关话题 × 5-8个(头部大标签+细分小标签搭配)
```

### 内容包的五件套(每次都要凑齐)

```
【标题备选】3条,每条含emoji,风格可不同(痛点型/利益型/悬念型)
【内容定位】目标读者是谁 / 选择这个话题的理由 / 建议发布时间段
【正文草稿】
  - 开头钩子(2行以内,抓眼球)
  - 正文主体(每段2-3行,多用换行,emoji分隔)
  - 结尾互动钩子
  - 话题标签建议(5-8个)
【配图卡方案】需要几张图 / 每张图放什么内容 / 建议用归藏卡片的哪种布局
【自检结论】读起来像真实用户分享吗?还是像广告?
```

### 标题公式(可套用)

| 类型 | 公式 | 示例 |
|------|------|------|
| 痛点型 | emoji + 痛点场景 + "???" | ✨ 每次修改 PPT 改到崩溃？？？ |
| 利益型 | emoji + 具体收益 + 人群 | 💼 职场人必看！AI 帮我省了2小时/天 |
| 悬念型 | emoji + 反常识 + 揭秘感 | 🤯 我以为 AI 写的东西很假，直到看见这个 |
| 教程型 | emoji + 步骤感 + 结果 | 📱 手把手教你用 AI 出一条小红书，10分钟搞定 |

### 配图卡说明(归藏社交卡片)

归藏卡片([op7418/guizang-social-card-skill](https://github.com/op7418/guizang-social-card-skill))能生成小红书原生尺寸(3:4)的图文卡,28 种布局。使用时你需要给出:
- 卡片主文字(标题或核心观点)
- 副文字(补充说明或来源)
- 选用哪种风格(极简/渐变/深色/手写感)

> ⚠️ 调用工具前先征得主人确认是否已安装。如果未安装,在内容包里写清楚"建议用归藏卡片生成,布局建议:XXX"即可。

### 话题标签策略

搭配原则:2-3 个大流量话题(百万级)+ 3-4 个细分话题(10万级)+ 1-2 个精准小圈子话题。

举例(AI工具类笔记):
```
#AI工具 #效率神器 #AI写作 #小红书运营 #内容创作 #自媒体干货 #职场技能
```

## 📝 毕业测验(必须真做,交证据)

**任务:为主人仓库的某个主题,产出完整的小红书内容包。**

**推荐主题(选一个或自定):**
- "AI agent 帮我做了一件事,我再也不想手动干了"
- "第一次用 Claude Code 的人,我想告诉你这三件事"
- "为什么我开始用 AI 写内容之后,产量翻了三倍"

**你需要做到:**

1. **产出五件套**(见上方操作要点):标题备选 × 3 + 内容定位 + 完整正文草稿 + 配图卡方案 + 自检结论。

2. **正文长度**:300-500 字,符合小红书竖屏阅读节奏,段落短、换行多、emoji 适量。

3. **标题自检**:三条标题里,至少一条含痛点、一条含具体利益点。

4. **配图方案**:写清楚需要 3-4 张图、每张放什么、建议归藏卡片哪种布局风格。

5. **工具调研(可选加分项)**:去 [github.com/op7418/guizang-social-card-skill](https://github.com/op7418/guizang-social-card-skill) 或 [github.com/waynefu2020/xhs-studio](https://github.com/waynefu2020/xhs-studio) 看一眼,写一段不超过 100 字的"给主人的工具推荐"。**调研归调研,安装须主人确认**。

6. **沉淀技能卡**:把小红书爆款内容包流程沉淀成 `skills/xiaohongshu-content.md`。

> ⚠️ **铁律再重申:只交草稿,绝不发布。** 全程不要登录任何小红书账号、不要调用任何会真实发布内容的 API。你的活到"把五件套文件写好交给主人"为止。
