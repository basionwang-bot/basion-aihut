## 📖 你要学会什么

学完这一课,你会帮主人设计一套**A/B 测试方案**——从"我有个想法"到"我能用数据证明这个想法是否有用",中间这段路怎么走,你会全程陪着走。

先说说为什么要做 A/B 测试,而不是"直接改了就完事"。

假设你的 App 有个绿色的"立即购买"按钮。有人说:"改成红色会不会点击率更高?"——你觉得呢?

直觉说"可能吧"。但直觉错了很多次了。

如果你直接把按钮改成红色,然后点击率真的高了——你怎么知道是红色导致的,而不是那段时间你刚好在搞促销活动?

A/B 测试干的就是这件事:同时给一半用户看绿色按钮(对照组),给另一半看红色按钮(实验组),在同样的时间、同样的条件下比较结果——这叫**控制变量**。只有这样,你才能说"红色按钮确实导致了更高的点击率"。

想象你是一位大厨想改菜谱。科学的做法是:把今晚的 100 桌客人随机分成两组——50 桌吃老菜谱,50 桌吃新菜谱——然后看哪组回头客更多。这就是 A/B 测试。

你的任务是:帮主人把"一个想法"变成一份可执行的实验方案——假设写清楚、样本算明白、防作弊措施想周全、结果分析不出错。

> ⚠️ 本课只产出方案,绝不替主人操作任何实验平台账号或修改任何生产代码。

## 🧠 核心原则(内化成习惯)

**1. 一次实验只测一个变量**

最常见的实验失败原因:同时改了三个地方,然后不知道是哪个起了作用。

规则:每次实验,对照组和实验组之间**只能有一个不同**。

**2. 先想清楚"成功长什么样"**

做实验前先定好:如果这个假设是对的,什么指标会变化、变化多少才算成功?

这个指标叫"主要评估指标(Primary Metric)"——先定好,实验结束后才能判断成败,不能事后再挑一个"恰好变好了"的指标来说"我们成功了"。

**3. 样本量要够,不然结论是假的**

这是很多人忽略的坑。样本量不够时,你看到的"差异"可能只是随机波动,不是真实效果。

统计学上有个词叫"统计显著性"——你的结论置信度要 ≥ 95%(p < 0.05),才能说"这个差异是真实的"。

样本量不够就结束实验、下结论——这和扔硬币没什么区别。

**4. 防作弊是必须考虑的问题**

实验会被污染。常见的污染来源:
- 用户看到了两个版本(泄漏):比如 A/B 用户分享了页面链接给彼此
- 同一个用户多次计算(重复计数):同一个人用不同设备进来被算了两次
- 机器流量(爬虫):不是真实用户,但混入了实验数据

**5. 警惕"多重检验"陷阱**

你同时跑 10 个实验,每个都有 5% 的概率得到"假阳性"结论——一下子全跑完,平均会有 0.5 个假成功。所以要控制同时运行实验的数量,并且做多重检验校正。

### 样本量计算脚本

```python
# 样本量计算(不需要外部平台账号,本地运行)
from scipy import stats
import math

def calculate_sample_size(
    baseline_rate,    # 当前转化率,如 0.05 表示 5%
    mde,              # 最小可检测变化,如 0.01 表示提升1个百分点
    alpha=0.05,       # 显著性水平,通常 0.05
    power=0.8         # 检验功效,通常 0.8
):
    """计算A/B测试所需的最小样本量(每组)"""
    p1 = baseline_rate
    p2 = baseline_rate + mde

    z_alpha = stats.norm.ppf(1 - alpha/2)  # 双侧检验
    z_beta = stats.norm.ppf(power)

    p_bar = (p1 + p2) / 2
    n = (z_alpha * math.sqrt(2 * p_bar * (1 - p_bar)) +
         z_beta * math.sqrt(p1 * (1 - p1) + p2 * (1 - p2))) ** 2 / (p2 - p1) ** 2

    return math.ceil(n)

# 示例:当前注册转化率5%,希望检测到提升1个百分点(至6%)
baseline = 0.05
mde = 0.01

n_per_group = calculate_sample_size(baseline, mde)
print(f"当前转化率: {baseline*100}%")
print(f"最小可检测变化(MDE): +{mde*100}个百分点")
print(f"每组需要样本量: {n_per_group:,} 人")
print(f"两组合计: {n_per_group*2:,} 人")

# 估算实验周期
daily_traffic = 1000  # 修改为实际每日流量
days_needed = math.ceil(n_per_group * 2 / daily_traffic)
print(f"按每天{daily_traffic}人流量,需要约 {days_needed} 天")

# 注意:实验至少要跑满1-2个完整周期(通常是7天的倍数),避免星期效应
min_days = max(days_needed, 14)
print(f"建议实验周期:至少 {min_days} 天(满足最小样本+完整周期)")
```

### 结果分析脚本

```python
# A/B 测试结果显著性检验(实验结束后使用)
from scipy import stats
import numpy as np

# 输入实验数据(由主人从实验平台导出后提供)
# 不要在此直接处理含用户个人信息的原始数据
control_conversions = 482   # 对照组:转化人数
control_total = 10000       # 对照组:总样本数
treatment_conversions = 531 # 实验组:转化人数
treatment_total = 10000     # 实验组:总样本数

# 计算转化率
p_control = control_conversions / control_total
p_treatment = treatment_conversions / treatment_total
lift = (p_treatment - p_control) / p_control

print(f"对照组转化率: {p_control*100:.2f}%")
print(f"实验组转化率: {p_treatment*100:.2f}%")
print(f"相对提升(Lift): {lift*100:.1f}%")

# 假设检验(双比例 z 检验)
from statsmodels.stats.proportion import proportions_ztest

count = np.array([treatment_conversions, control_conversions])
nobs = np.array([treatment_total, control_total])
z_stat, p_value = proportions_ztest(count, nobs)

print(f"\n统计检验结果:")
print(f"Z统计量: {z_stat:.4f}")
print(f"P值: {p_value:.4f}")

if p_value < 0.05:
    print(f"✅ 结论:差异统计显著(p={p_value:.4f} < 0.05)")
    print(f"   实验组转化率比对照组高 {lift*100:.1f}%,置信度 ≥ 95%")
else:
    print(f"❌ 结论:差异不显著(p={p_value:.4f} ≥ 0.05)")
    print(f"   现有数据不足以证明实验组更好,建议继续收集数据或重新审视假设")

# 95% 置信区间
from statsmodels.stats.proportion import proportion_confint
ci_low, ci_high = proportion_confint(treatment_conversions, treatment_total, alpha=0.05)
print(f"\n实验组转化率 95% 置信区间: [{ci_low*100:.2f}%, {ci_high*100:.2f}%]")
```

### 防作弊检查清单

```
【实验启动前防作弊检查】

□ AA测试通过:在正式实验前,先跑一个"两组都是对照组"的AA测试,
  验证两组基线无显著差异(如果AA测试就有显著差异,说明分流有问题)

□ 分流单元正确:确认用的是用户ID而非Session,避免同一用户看到两个版本

□ 样本互斥:每个用户只进入一个实验组,没有跨实验的样本污染

□ 过滤机器流量:实验数据中排除爬虫/机器人访问
  (通常通过User-Agent过滤,具体由研发配置)

□ 避免污染效应:实验变化不会通过社交分享等方式"泄露"给对照组用户

□ 时间窗口完整:实验覆盖完整的自然周期(建议7天的整数倍),
  避免星期效应(周末用户行为和工作日差异很大)

【实验过程中监控】
□ 每日检查护栏指标:一旦护栏指标变差超过阈值,立即停止
□ 不提前"偷看"结果:看到数据变好就停,会导致假阳性(p-hacking)
□ 流量分配是否均匀:两组样本量每天是否大致相等
```
