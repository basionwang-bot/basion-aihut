## 📖 你要学会什么

学完这一课,你会产出两类可立刻交付的文档:

1. **员工手册条目草稿**:把主人提供的公司制度要点,变成条理清晰、员工读得懂的手册文字
2. **内部通知文案**:各类企业内部通知的标准格式和现成示例——从"全员大会通知"到"离职交接提醒"

这两件事表面上看只是"写写字",其实有个隐藏挑战:手册写得太法条化,员工看了头疼、记不住、跳过去;写得太随意,又没有制度感、发生争议时没有依据。你要走一条中间路:**说人话,但不失严谨**。

想象手册是一本"新员工入职第一周必读的地图"——它得告诉人家:这家公司的规矩是什么、能做什么不能做什么、遇到问题找谁。地图越清晰,新人少走的弯路越多。

> ⚠️ 你产出的是草稿。员工手册在中国涉及劳动法合规,正式发布前**必须由主人或专业HR/法务审核确认**,AI草稿不等于法律文件。

---

## 🧠 核心原则(内化成习惯)

1. **先问"这条规定为什么存在"。** 每条制度背后都有一个原因。帮主人把原因也写进手册("为了保护员工和公司信息安全,因此……"),员工才会愿意遵守,而不是把手册当作"公司又来立规矩了"。

2. **一条规定,一个意思。** 手册最怕歧义。"工作时间灵活处理"——什么叫灵活?早上九点到几点算正常?写不清楚的规定,将来一定引发争议。帮主人把模糊表达变成清晰描述。

3. **通知≠命令。** 内部通知文案的口吻决定了信息的传递效果。"各部门注意:明天必须参加会议,不得请假!"和"明天下午 3 点全员会议,议题是 Q3 规划,请优先安排,如有特殊情况请提前告知 HR"——后者给了原因、给了弹性,员工更容易配合。

4. **格式统一是专业感的来源。** 手册用标题层级、编号清单;通知用固定抬头格式。读者看一眼就知道这是什么类型的文件、从哪里找到他们需要的信息。

5. **你只写草稿,发布由主人决定。** 手册和通知发出去就有法律效力(手册尤其如此),你不替主人点发送、不上传钉钉/飞书、不替主人签字盖章。

## 🛠 操作要点

### 第一步:员工手册条目结构

一本完整的员工手册通常包含以下章节,帮主人按需选择:

```
【第一章 公司介绍】
  - 公司简介与发展历程(1-2段,让新员工感受到公司文化)
  - 使命/愿景/价值观(用自己的话说,别复制粘贴口号)

【第二章 劳动合同与入职】
  - 合同签订流程
  - 试用期规定(时长/考核标准/转正条件)
  - 背景调查说明

【第三章 工作时间与考勤】
  - 工作时间(标准/弹性/远程的具体规则)
  - 打卡规定
  - 加班政策

【第四章 薪酬与福利】
  - 工资发放时间和方式
  - 社会保险与公积金
  - 年假/病假/事假等假期规则(需符合《劳动法》相关规定)
  - 其他福利(餐补/交通/体检等)

【第五章 行为准则】
  - 职业道德要求
  - 保密协议说明
  - 利益冲突禁止规定
  - 社交媒体使用规范

【第六章 绩效与晋升】
  - 绩效评估周期和方式
  - 晋升/调薪流程

【第七章 离职流程】
  - 辞职/辞退的基本流程
  - 交接规范
  - 离职证明和竞业限制说明

【附录】
  - 常见问题 FAQ
  - 重要联系人名单
  - 员工确认签字页
```

### 第二步:单条手册条目写作格式

```markdown
## [条目标题]

**适用范围**:全体正式员工 / [特定群体]

**政策说明**
[2-4句话说明这条规定是什么、为什么有这条规定]

**具体规定**
1. [具体条款，用清晰数字和日期表达，不用"尽快""适当"等模糊词]
2. ...
3. ...

**例外与特殊情况**
如遇 [特殊情形]，请提前联系 [负责部门/人] 申请 [处理方式]。

**违规处理**
违反本条规定的，将 [具体后果，如：书面警告/扣发绩效奖金/解除劳动合同]。

> 如有疑问，请联系 [HR邮箱/钉钉群]。
```

### 第三步:内部通知文案模板库

**▌全员会议通知**
```
[企业名称] 内部通知

致：全体员工
时间：[具体日期] [具体时间]
地点：[地点 / 线上会议链接]
主题：[会议主题]

本次会议将就 [具体议题] 进行讨论，预计时长 [X 小时]，请各位提前安排好工作。

如有特殊情况无法参加，请提前 24 小时告知直属上级及 HR，并自行向同事了解会议内容。

人力资源部
[日期]
```

**▌制度更新通知**
```
关于更新 [制度名称] 的通知

各位同事：

经公司管理层审议，[制度名称] 自 [生效日期] 起正式更新。

主要变化：
1. [变化点1，旧规定→新规定]
2. [变化点2]

变更原因：[一句话说明为什么调整]

完整版手册已更新至 [存放位置：飞书/钉钉/共享盘]，请各位自行下载阅读。

如有疑问，请联系 [HR 姓名 / 联系方式]。

人力资源部
[日期]
```

**▌离职交接提醒(发给交接双方)**
```
关于 [离职员工姓名] 离职交接安排

[离职员工姓名] 的最后工作日为 [日期]，请按以下安排完成交接：

交接内容：
- 工作文件：[存放位置及权限]
- 进行中项目：[项目名称及现状]
- 重要联系人：[客户/供应商/内部协作方]
- 系统账号：由 IT 统一处理，请勿私自转移密码

交接确认：请双方在 [截止日期] 前完成 [交接表/确认文件]，交 HR 存档。

如有疑问，联系 [HR 姓名]。

人力资源部
[日期]
```

**▌节假日放假通知**
```
关于 [节日名称] 放假安排的通知

全体同事：

根据国家法定假日及公司相关规定，[节日名称] 放假安排如下：

放假时间：[日期] 至 [日期]，共 [X] 天
上班时间：[复工日期]

请各位提前做好工作安排，确保假期前完成 [紧急事项说明，如有]。

假期期间如有紧急事务，请联系 [值班人员/紧急联系方式]。

祝大家假期愉快！

人力资源部 / 行政部
[日期]
```

## 📝 毕业测验(必须真做,交证据)

**任务:为一家模拟公司,产出员工手册的3个完整条目 + 3份内部通知文案。**

**练习场景:一家50人左右的互联网公司,主营电商运营服务,员工以运营和销售为主。**

**你需要做到:**

1. 写**员工手册 3 个条目**,每个条目按"第二步"格式写完整:
   - 条目A:弹性工作时间规定
   - 条目B:年假规则(需符合中国劳动法规定:工作满1年享有5天带薪年假,满10年10天,满20年15天)
   - 条目C:社交媒体使用规范

2. 写 **3 份内部通知文案**:
   - Q2 全员业务复盘会议通知
   - 考勤打卡系统升级通知
   - 某员工离职交接提醒(虚构姓名)

3. 每份通知写完后,自检:**有没有"尽快/近期/适当"等模糊时间词?有没有清楚写明负责人和截止时间?**

4. **沉淀技能卡**:把"手册条目格式+通知文案四类模板"写成 `skills/employee-handbook.md`。

> ⚠️ 再次提醒:真实公司在正式使用手册条目前,**必须经过HR或法务审核**,特别是涉及假期天数、违规处理等劳动法相关内容。你产出的是可讨论的草稿,不是法律文件。
