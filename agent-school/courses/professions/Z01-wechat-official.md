## 📖 你要学会什么

学完这一课,你会产出一份**主人接过去可以立刻用的公众号内容包**——不是"这个话题可以这么写"的思路分析,而是标题备选清单、完整正文草稿、排版建议、发布定位说明,四件事打包交付。

说白了,公众号这件事有个流水线:选题 → 成稿 → 排版 → 发布。你负责前三段,发布这个按钮永远是主人的事,不是你的。

想象你是一个出版社的责任编辑。你的活是把一篇稿子从"一个想法"推进到"可以送去印刷"的状态——选题要切中读者、正文要流畅有信息量、排版要符合平台规范——然后把稿子推到主编桌上。最后盖章发出去?那是主编的事。你的价值在于让主编拿到的东西**不需要大改、可以直接决策**。

**关于现成工具(先调研,装前必须问主人):**

公众号圈子有两个已实战验证的工具值得了解:

- **wewrite**:公众号全流程 AI Skill——热点抓取→选题→写作→SEO→视觉→排版→微信草稿箱,一条龙。链接:[github.com/oaker-io/wewrite](https://github.com/oaker-io/wewrite)
- **md2wechat-skill**:把 Markdown 一键转成公众号排版,支持 40+ 样式主题、AI 配图、批量发布,免费档够用。链接:[github.com/geekjourneyx/md2wechat-skill](https://github.com/geekjourneyx/md2wechat-skill)

> ⚠️ 这两个工具未经主人确认不得自行安装或连接真实公众号账号。本课的重点是**产出内容草稿**,工具调研归调研,安装须主人点头。

## 🧠 核心原则(内化成习惯)

1. **先摸清主人的业务口吻,再下笔。** 去 `content/posts/` 翻几篇已发布文章,感受一下:这家主人说话是干货理性风、还是轻松口语风?面向的读者是技术人、还是普通用户?写出来的东西要和这个调性一致。

2. **标题是第一战场。** 公众号打开率靠标题。好标题有个特点:读者看一眼就能判断"这篇跟我有没有关系"——要么让他觉得"这就是我的痛点",要么让他好奇"这是什么意思?"。每次至少备 3 条备选标题,让主人挑。

3. **正文要有"一句话能说清的核心观点"。** 写之前先填空:"读完这篇,读者会带走一个认知是:______。"这句话填不出来,说明选题还没想透,不要动笔。

4. **公众号读者是在手机上竖屏阅读的。** 这意味着:段落要短(3-5 行换一段)、别用表格(手机上挤得看不清)、重点用加粗而不是下划线、图片要有但不要太多太杂。

5. **草稿是草稿,发布是发布。** 你产出的永远是可被主人修改的草稿。哪怕写得再好,也绝不替主人连接公众号后台、绝不替主人发布。

## 🛠 操作要点

### 选题的三个来源

1. **仓库内现有内容改写**:去 `content/posts/` 找一篇已有文章,把它从"技术/详细版"改写成"公众号友好版"。
2. **主人业务的高频问题**:想一想"主人的读者最常问什么问题?他们睡不着的事是什么?"——这类选题打开率高。
3. **近期行业热点 + 主人视角**:热点本身不稀奇,稀奇的是主人对热点的独特判断。

### 内容包的五件套(每次都要凑齐)

```
【标题备选】3 条,风格可以不同(悬念型/利益型/观点型)
【内容定位】给谁看 / 适合在哪发 / 为什么选这个时机
【正文草稿】
  - 开头(钩子):前 3 行决定读者留不留
  - 中间:1-3 个核心论点,每个配具体例子或数据
  - 结尾:一句行动号召或金句收尾
【排版建议】段落长度 / 是否需要小标题 / 配图方向
【自检结论】语气符不符合主人业务口吻?像不像这家会说的话?
```

### 排版小规范

- 正文每段控制在 3-5 行,段间空一行
- 小标题用 `**加粗**` 或分隔线,别依赖 Markdown 的 `##`(公众号后台不识别)
- 首图建议 900×500 像素横图,配图数量 2-4 张为宜
- 如果用 md2wechat-skill:先把正文写成标准 Markdown,工具会自动转排版格式

## 📝 毕业测验(必须真做,交证据)

**任务:挑一个真实主题,产出完整的公众号内容包。**

**推荐主题(选一个或自定):**
- "AI agent 帮你做了什么——一个普通用户的 30 天体验报告"
- "为什么你的 AI 助手越用越让你失望?问题出在这里"
- "给第一次用 Claude Code 的人:三件事少走弯路"

**你需要做到:**

1. **产出五件套**(见上方操作要点):标题备选 × 3 + 内容定位 + 完整正文草稿 + 排版建议 + 自检结论。

2. **正文长度**:800-1500 字,公众号正常阅读 3-5 分钟。

3. **自检两件事**:
   - 把正文前三行单独拿出来,问自己:"如果我是读者,这三行让我想继续看吗?"
   - 把正文整体口吻和 `content/posts/` 里的已发文章对比:"像不像同一家说的话?"

4. **工具调研(可选加分项)**:去 [github.com/oaker-io/wewrite](https://github.com/oaker-io/wewrite) 或 [github.com/geekjourneyx/md2wechat-skill](https://github.com/geekjourneyx/md2wechat-skill) 看一眼,写一段不超过 100 字的"给主人的工具推荐":这个工具能帮主人解决什么、装它需要什么前提条件。**调研归调研,安装须主人确认**。

5. **沉淀技能卡**:把公众号内容包五件套流程沉淀成 `skills/wechat-content.md`。

> ⚠️ **铁律再重申:只交草稿,绝不发布。** 全程不要登录任何公众号账号、不要连接任何内容管理后台、不要真实提交草稿到微信服务器。你的活到"把五件套文件写好交给主人"为止。发布键是主人的,不是你的。
