## 📖 你要学会什么

学完这一课,你会产出一套**主人拿去就能用的面试工具包**——包含分层题库(必问题 + 岗位专属题 + 追问题)和配套候选人评估表。

面试这件事,就像是打猎。经验不足的面试官扛着枪进了森林,却不知道自己在找什么——一路乱问,问完还是说不清楚这个人到底行不行。你的任务是在面试前帮主人想好:"我们今天要找什么猎物、用什么工具去找、找到了怎么认出来。"

换句话说,你负责的是:
1. **备好弹药**(题库:分层、分岗位、有追问链)
2. **备好评分表**(面试官填一张表就能留下可比较的记录)
3. **提醒面试官用法**(写清楚怎么用这套工具)

> 提醒:面试本身是主人的事,你不参与真实面试,也不替主人做录用决定。

---

## 🧠 核心原则(内化成习惯)

1. **题目要"钓"行为,不要钓自我描述。** "你是一个有责任心的人吗?"——这问题任何人都会说"是"。好问题是:"讲一件你主动承担了本不属于你职责的事,结果怎样?"有行为、有情境,才有信息量。这就是 STAR 法的精髓。

2. **分层出题,不一刀切。** 基础题问的是"会不会",进阶题问的是"能不能做好",压力题问的是"在复杂情况下怎么处理"。三层混用,才能看清候选人的天花板在哪。

3. **追问是点睛之笔。** 候选人给了一个漂亮的答案?好,追一句:"那个项目最后结果的数据是多少?""如果重来一次你会改什么?"——追问让真实水平无处遁形。

4. **一套题只服务一个岗位。** 给产品经理的题不能直接拿去面试销售。从 JD 出发,每套题都有自己的锚点。

5. **评估表是为了公平比较,不是为了排座次。** 三个面试官用同一张表,填完之后可以对比讨论,这才是结构化面试的价值所在。

## 🛠 操作要点

### 第一步:确定面试层次框架

每套题库分三层,每层推荐题量如下:

```
【L1 基础核实层】(5题)
  目的:验证简历真实性 + 基本能力门槛
  特点:答案相对客观,能对照简历核实
  示例通用题:"请介绍一下你在上一份工作里日常最频繁的三类任务"

【L2 能力深挖层】(5-8题,按岗位定制)
  目的:考察核心能力是否达到岗位要求
  特点:STAR 行为题为主,追问链2-3步
  示例通用题:"讲一件你独立完成的项目,从立项到交付,最大的挑战是什么?"

【L3 情景压力层】(3-5题)
  目的:考察在复杂/边界情况下的判断力和应变
  特点:假设情景题,没有标准答案,看思维过程
  示例通用题:"你被要求在资源减半的情况下完成同样的目标,你会怎么做?"
```

### 第二步:分岗位题库模板

下面是**三类常见岗位**的题库样板,照此格式扩展:

---

**▌运营类岗位(产品运营/用户运营/内容运营)**

```
L1 基础核实
Q1. 你上一份工作负责的核心指标是什么?最好的一个月数字是多少?
Q2. 你日常用哪些数据工具?举一个你用数据发现问题并解决的例子。

L2 能力深挖
Q3. 讲一个你做过的用户留存/增长项目:背景是什么、你做了什么、结果怎样?
    → 追问1:数据是怎么验证效果的?
    → 追问2:这个方案是你一个人想的还是团队共创?你的贡献具体是什么?
Q4. 你做过哪次内容/活动效果比预期差很多?原因是什么?你从中学到什么?
Q5. 如果你发现一个你运营的渠道 ROI 下滑了 30%,你第一步做什么?

L3 情景压力
Q6. 产品突然要你在 3 天内上线一场大促活动,你平时需要 2 周。你怎么办?
Q7. 你的一个拉新方案被老板否掉了,但你觉得这是目前最优解。你会怎么处理?
```

---

**▌销售类岗位(BD/大客户销售)**

```
L1 基础核实
Q1. 你上一份工作的销售目标是多少?实际完成率?
Q2. 你通常是怎么找到潜在客户的?举一个具体的例子。

L2 能力深挖
Q3. 讲一个你把一个最开始完全没意向的客户最终拿下的案例。
    → 追问1:中间有没有某次谈判几乎谈崩的时刻?你怎么救回来的?
    → 追问2:你们的产品当时有什么明显劣势?你怎么应对客户的质疑?
Q4. 你同时在跟进 10 个客户,但只有 1/10 的时间。你怎么排优先级?
Q5. 你有没有拒绝过一个客户订单?为什么?

L3 情景压力
Q6. 客户在合同签订前一周突然提出要砍 20% 的价格,否则换供应商,你怎么谈?
Q7. 你的销售数字本月距离 KPI 还差 40%,还有最后一周,你会怎么做?
```

---

**▌HR/行政类岗位**

```
L1 基础核实
Q1. 你负责过几个人规模的招聘工作?平均一个岗位从开启到入职用多久?
Q2. 你熟悉哪些招聘渠道?哪个渠道效率最高?

L2 能力深挖
Q3. 讲一次你处理员工投诉或内部冲突的经历:怎么介入、怎么处理、结果如何?
    → 追问1:事后两边当事人的反馈是什么?
    → 追问2:这件事有没有推动你们改变某个制度或流程?
Q4. 你怎么设计一个岗位的薪酬范围?你会参考哪些数据来源?
Q5. 你有没有因为合规原因拒绝过某个领导提出的 HR 要求?怎么处理的?

L3 情景压力
Q6. 公司要快速扩招 50 人,但预算比同行平均低 30%,你怎么做招聘策略?
Q7. 一个绩效优秀但和团队关系很差的员工该怎么处理?
```

## 📝 毕业测验(必须真做,交证据)

**任务:为一个真实或模拟岗位,产出完整的面试工具包。**

**推荐练习岗位(三选一,或主人指定):**
- 产品运营专员(1-3年)
- 销售代表(B2B方向)
- 招聘专员

**你需要做到:**

1. 写出该岗位的**分层题库**(L1 基础 5题 + L2 能力 5题+追问链 + L3 压力 3题)。
2. 为 L2 的每道题至少设计 **2步追问链**,并注明"好答案长啥样"(参考答案关键词)。
3. 产出**候选人评估表**模板(按上方格式,维度对应岗位调整)。
4. 写一段不超过 200 字的**"给面试官的用法说明"**:怎么用这套题、需要注意什么。
5. **沉淀技能卡**:把"面试题库三层结构+评估表"写成 `skills/interview-bank.md`。
