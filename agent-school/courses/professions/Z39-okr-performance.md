# 第 Z39 课 · 绩效与 OKR 梳理

---

## 📖 你要学会什么

学完这一课,你会产出两样东西:

1. **OKR 草稿**:给主人或主人的团队用的目标+关键结果,直接拿去填飞书/钉钉/Excel 都行
2. **绩效梳理报告**:把一段时间的工作成果整理成一份结构清晰、数据说话的绩效总结

想象一下:年底到了,你的同事小王要写绩效自评。他盯着空白表格发呆了半小时,脑子里只有"这一年好像做了很多,但说不清楚"。你的工作就是帮他把这"好像做了很多"变成一张清清楚楚的成绩单——每件事情有数字、有结果、有影响力,让老板看一眼就知道这人干了什么。

OKR 是"目标+结果"的组合拳:先锁定方向(Objective),再设3-5个可量化的里程碑验证有没有走到(Key Results)。听起来简单,但很多人写出来的 OKR 要么太虚("提升用户满意度"),要么太细("每天回复50封邮件")——都不对。你这一课专门训练这个拿捏。

---

## 🧠 核心原则(内化成习惯)

1. **O 是方向,KR 是尺子。** Objective 要有野心、有激励性,但不能是一句口号。KR 要是数字或具体产出物,季度末能明确判断"达到了"还是"没达到"。

2. **KR 不是任务清单。** "完成 A 项目"是任务(To-do),不是 KR。"A 项目上线后 DAU 突破 10000"才是 KR。区别在于:KR 在乎结果,不在乎过程。

3. **绩效总结要用数字撑腰。** "负责了用户运营"——没信息量。"3 季度共触达 5 万用户,转化率 12%,高于行业均值 3 个百分点"——有信息量。能量化的一定要量化,不能量化的要说清楚影响范围。

4. **一个周期只设 2-4 个 O。** OKR 不是愿望清单,贪多嚼不烂。帮主人梳理时,如果他提了七八个方向,你要帮他收敛到最重要的几个。

5. **草稿是你的,决策是主人的。** OKR 方向和绩效评级,由主人和团队定,你产出的是"可以直接讨论的草稿",不是"最终答案"。

## 🛠 操作要点

### 第一步:OKR 拆解模板

```
【季度/年度 OKR 草稿】
部门/个人:____  周期:____年 Q_

O1:【目标标题——一句激励性描述】
  KR1.1:____ [具体可量化目标，含数字和截止时间]
  KR1.2:____
  KR1.3:____
  本条 O 的战略意义:为什么这个方向重要?和公司目标怎么挂钩?

O2:【目标标题】
  KR2.1:____
  KR2.2:____
  ...

【设置说明】
- 来源:基于主人提供的 [业务方向/上级OKR/战略重点]
- 建议对齐周期:____
- 建议复盘频率:____（推荐：双周1对1 + 季度书面复盘）
```

### 第二步:OKR 质量自检清单

写完 OKR 草稿后,用这张清单过一遍:

```
✅ 每个 O 能一句话说清楚"我们要往哪走"?
✅ 每个 KR 都含有数字(或明确产出物)?
✅ KR 是结果而非任务?(避免"完成XX"这类动作导向的写法)
✅ 全部 O 加起来不超过 4 个?
✅ 季度末能明确判断每个 KR 是否达成?
✅ O 与公司/上级 OKR 有明确挂钩?
❌ 有没有"提升""加强""持续优化"这类无法量化的词?有则替换
❌ 有没有只写了活动计划而非结果?有则改成结果描述
```

### 第三步:绩效总结梳理模板

```markdown
# 绩效自评 / 工作总结
姓名(或岗位):____  评估周期:____  填写日期:____

## 核心成果(Top 3-5)
> 按重要程度排序,每条一句话+数据支撑

1. [成果名]:在 [时间段] 内完成了 [具体事项],结果是 [量化数据],与同期相比 [对比基准]。
2. ...

## OKR 完成情况
| KR | 目标值 | 实际完成 | 达成率 | 简短说明 |
|----|--------|----------|--------|---------|
| KR1.1 | ... | ... | __% | 超额/未达/持平+原因 |
| KR1.2 | ... | ... | __% | |

## 亮点&创新
- (主动做了哪些不在 KR 里、但对团队有价值的事?)

## 未完成/低于预期
- (哪条 KR 没达到?原因分析+下期改进计划)

## 下周期重点
1.
2.
3.

## 需要支持的资源
(需要上级/公司提供什么支持才能完成下期目标?)
```

### 第四步:给不同场景的提问方式

帮主人梳理 OKR 时,按顺序问这几个问题(不要一口气全问,一次一个):

```
1. "你们这个季度/年度最重要的1件事是什么?做成了公司会有什么变化?"
2. "你个人能控制的、最影响这件事的指标是什么?"
3. "你希望3个月后,哪3个数字能证明你干得不错?"
4. "有没有什么约束条件要提前说清楚?(资源、依赖其他团队的事)"
5. "上级的 OKR 是什么?你这条 O 怎么支撑上级的目标?"
```

## 🧰 配套开源项目(可选集成)

> 这门课的活,也有现成开源项目能帮你省力。**连真实数据库/客户数据/账号前先问主人**,优先只读、先脱敏、用完即删;星数为调研约数,装前再核。

- **okr_os** ([github.com/credifit-br/okr_os](https://github.com/credifit-br/okr_os), 星数较少) —— 可自托管的开源 OKR 管理工具,支持目标树展示和进度追踪。**用法**:把本课产出的 OKR 草稿录入 okr_os 作为团队共同可见的目标看板,每周更新 KR 进度。诚实提示:OKR 开源工具普遍星数低、活跃度一般,装前评估是否值得维护;对于小团队用飞书/钉钉的 OKR 模块可能更省事。

## 📝 毕业测验(必须真做,交证据)

**任务:为一个真实或模拟的岗位/部门,产出 OKR 草稿 + 绩效总结模板。**

**练习场景(选一个):**

场景A:某电商公司用户运营部门,下季度重点是拉新和提升复购率。

场景B:某 SaaS 公司销售团队,下半年目标是扩大企业客户签约量。

场景C:主人自己的团队(需主人提供背景信息)。

**你需要做到:**

1. 产出该场景的 **OKR 草稿**(2-4 个 O,每个 O 带 3 个 KR)。
2. 用"OKR 质量自检清单"过一遍,把发现的问题一并列出来并修改。
3. 产出**绩效总结模板**(填充假设数据,让主人看到格式和写法)。
4. 写一段不超过 150 字的**"给主人的说明"**:这套 OKR 的核心逻辑是什么、主人在用之前需要确认什么。
5. **沉淀技能卡**:把"OKR 拆解四步法+绩效总结模板"写成 `skills/okr-performance.md`。
