## 📖 你要学会什么

学完这一课,你会产出一份**主人接过去可以立刻用的简历筛选报告**——不是"这几个人还不错"的感性评价,而是对照岗位要求的逐项打分表、候选人排序清单、以及每人一段"为什么推/不推"的简短说明。

想象一下:HR 的桌上堆着 200 份简历,截止日期是明天。你就是那个能在 10 分钟内把这 200 份按岗位关键词过一遍、给每份贴上分数标签、再把前 10 名整整齐齐排好递上去的"聪明实习生"。HR 只需要确认你的结论,而不是从头再翻一遍。

**你要做的三件事:**

1. **理解岗位** — 从主人给的 JD(岗位描述)里提取评分维度
2. **逐份打分** — 对照维度给每份简历打分,写理由
3. **出排序报告** — 汇总成可直接用的推荐清单

> ⚠️ **隐私铁律:简历是非常敏感的个人数据。** 你打分时用的所有信息,只能在当前对话里流转,绝不可以把候选人姓名、手机、身份证号、照片等信息发送到任何第三方工具、API 或网络请求里。最小化原则:能完成打分任务的,只用工作经历+技能,其余隐私字段视而不见。

---

## 🧠 核心原则(内化成习惯)

1. **岗位说了算,不是你的审美说了算。** 打分唯一的尺子是主人给的 JD。你觉得某人"看起来很厉害",但 JD 没要求这个技能?那就不加分。同理,JD 明确要求的,缺了就扣分。

2. **维度要量化,不要只有感觉。** "沟通能力强"是感觉,"有 3 年以上跨部门项目管理经验"才是可量化的维度。从 JD 里提炼的每个维度,都要能在简历里找到对应的"证据句"。

3. **打分不等于判人生死。** 你的报告是草稿,HR 或主人才是终审官。低分候选人可能有你看不到的优点,高分的也可能有不适合这家公司文化的地方。你的工作是把信息整理清楚,决策权在人。

4. **保持一致性。** 第 1 份和第 100 份简历用同一套标准,不能因为今天心情好分数就宽松了。每次打分前先把维度表钉在脑子里。

5. **简历是别人的隐私。** 评分报告只给主人,不转发、不存档到第三方、不截图分享。

## 🛠 操作要点

### 第一步:从 JD 里提炼评分维度表

拿到 JD 之后,先按下面这个模板拆解:

```
【岗位名称】____
【硬性门槛】(不达标直接淘汰)
  - 例:大专及以上学历
  - 例:5年以上同岗经验
  - 例:有 XX 行业从业证书

【核心能力维度】(每项 0-10 分)
  维度1:____  (从 JD 提取的关键词) 权重 ___%
  维度2:____  权重 ___%
  维度3:____  权重 ___%
  维度4:____  权重 ___%
  (建议 3-5 个维度,权重加总 100%)

【加分项】(有则额外加 1-3 分)
  - 例:有大厂背景
  - 例:有海外留学经历(岗位有国际业务的才算)

【减分项】(有则扣 1-3 分)
  - 例:频繁跳槽(平均在职不足 1 年)
  - 例:学历/工作经历造假嫌疑(简历前后矛盾)
```

> 实战提示:让主人把 JD 文字直接发给你,你来拆解。拆完先给主人确认一遍维度表,确认了再开始打分——这能防止后面返工。

### 第二步:逐份打分(单份简历处理模板)

```
【候选人编号】#001(不用姓名,用编号保护隐私)
【硬性门槛】✅ 全过 / ❌ 未过(未过的直接标注理由,不再打分)

【各维度得分】
  维度1 - [维度名]:  _/10  证据:"简历原文引用句"
  维度2 - [维度名]:  _/10  证据:"简历原文引用句"
  维度3 - [维度名]:  _/10  证据:"简历原文引用句"
  维度4 - [维度名]:  _/10  证据:"简历原文引用句"

【加减分】+_ / -_  理由:____

【加权总分】___/100

【一句话结论】推荐进入下一轮 / 候补备选 / 不推荐
【推荐理由】50字以内,说最关键的一点
```

### 第三步:汇总报告(交付给主人的最终格式)

```markdown
# 简历筛选报告
岗位:____  共收到简历:__ 份  筛选日期:____
生成方:AI 草稿,终审:HR/主人

## 推荐进入下一轮(前 N 名)
| 排名 | 编号 | 加权总分 | 一句话亮点 |
|------|------|----------|-----------|
| 1    | #005 | 87/100   | 5年同赛道经验+带过15人团队 |
| 2    | #012 | 82/100   | 证书齐全+项目数据可量化 |
| ...  |      |          |           |

## 候补备选(可选看)
(同上格式)

## 未入围
共 __ 人,主要原因:__(学历不符/经验不足/频繁跳槽)

## 本次筛选说明
- 评分维度来自[岗位JD,时间:____]
- AI 评分仅作参考,最终决策由 HR 或主人确认
- 简历原件已妥善保管,本报告不含候选人手机/身份证等敏感字段
```

## 🧰 配套开源项目(可选集成)

> 这门课的活,也有现成开源项目能帮你省力。**连真实数据库/客户数据/账号前先问主人**,优先只读、先脱敏、用完即删;星数为调研约数,装前再核。

- **ResumeParser** ([github.com/OmkarPathak/ResumeParser](https://github.com/OmkarPathak/ResumeParser), ~1k★) —— 本地 LLM 简历解析工具,把 PDF/Word 简历自动提取为结构化字段(姓名、学历、工作经历等)。**用法**:批量解析候选人简历,输出标准化 JSON 供 AI 对比评分,省去手工录入。**隐私优先**:本地运行,简历数据不上任何云端。

- **Resume-Matcher** ([github.com/srbhr/Resume-Matcher](https://github.com/srbhr/Resume-Matcher), ~6k★) —— 简历与 JD 匹配度打分工具,用向量相似度计算简历和岗位描述的契合度。**用法**:把 JD 和候选人简历批量丢进去,快速拿到匹配分排名,作为初筛参考。**⚠️ 隐私铁律**:简历含大量个人隐私,**必须本地部署、用完即删**,不上传第三方服务,处理真实简历前必须问主人确认。

## 📝 毕业测验(必须真做,交证据)

**任务:拿一份真实岗位 JD + 至少 5 份模拟简历,走完全程,产出筛选报告。**

如果主人暂时没有真实 JD,用下面这份练习用 JD:

> **练习用 JD:产品运营专员**
> 要求:本科及以上,1-3年互联网产品/运营经验,熟悉数据分析工具(Excel/SQL 任一),有过用户增长或留存项目经验;加分项:有短视频/直播平台运营经验。

**你需要做到:**

1. 从上面的 JD 提炼**评分维度表**(3-5 个维度+权重),给主人确认后再打分。
2. 为至少 **5 份模拟简历**(你自己虚构 5 个不同背景的候选人)逐份填写打分表。
3. 汇总成**筛选报告**,格式严格按上方"第三步"模板。
4. 自检:报告里有没有出现候选人手机号、身份证号等隐私字段?有的话删掉。

> ⚠️ 如果主人提供了真实简历:你只在对话内使用,绝不把候选人信息发送到任何外部 API、工具或链接里。打分完成后,告知主人报告已就绪,由主人决定如何保存和流转。

5. **沉淀技能卡**:把"简历筛选三步法"写成 `skills/resume-screening.md`。

## 🎓 过关标准

- [ ] 你从 JD 提炼了**维度表**(3-5 个维度+权重+硬性门槛),并经主人确认

- [ ] 全程**没有把候选人信息发送到任何第三方接口或外部链接**
