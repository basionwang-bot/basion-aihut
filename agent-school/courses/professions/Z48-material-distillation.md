## 📖 你要学会什么

学完这一课,你会把一堆散乱的输入材料(多篇文章、长报告、会议记录、课程讲义……)提炼成**主人拿去就能用的知识产出物**——知识卡片、要点摘要、行动清单或对比表格,格式由任务决定,内容一定要精准可用。

先说个画面:你是一个新来的研究助理,主人把一摞文件扔在你桌上——"帮我把这些东西整理一下"。问题来了:整理成什么?摘要?提纲?卡片?关键数字?决策建议?

**方向不清楚,整理就是白整理。**

所以这门课教你两件事:第一,开工之前先问清楚(或者根据上下文猜出来)主人要的是什么格式;第二,按格式高密度提炼——**每一条输出都要在原材料里有出处,不许凭空生发**。

这个能力的底层逻辑和 NotebookLM 的"投喂式知识管理"是一样的:把原始材料当成数据库,你的工作是查询和提炼,而不是创作和发挥。

## 🧠 核心原则(内化成习惯)

**1. 提炼 ≠ 复制,也 ≠ 改写**

想象你在榨橙子。橙子是原材料,橙汁是产出物。你不是把橙子搬到另一个篮子里(复制),也不是把橙子画成苹果(改写);你是把它的精华压出来,废渣留下。

资料提炼也是这样:原材料里的"精华"是什么,取决于主人要解决什么问题。行动导向的主人要的是决策依据和下一步行动;学习导向的主人要的是核心概念和记忆锚点。

**2. 先锁定"提炼目的",再动手**

开工前填这两个空:
- "主人读完这批材料之后,要做一个决定/解决一个问题,那个决定/问题是:**______**。"
- "如果只能保留 3 条信息,最重要的 3 条是:**______**。"

填不出来?先问主人,或者在输出里写清楚"我假设你要解决的是……"。

**3. 每一条产出都要对得上原材料里的某处**

这叫"有据可依"(grounded)。你写的每条要点,自己要能说出它来自哪篇文章哪段话。如果是你推断出来的,明确标注"(推断)"。杜绝幻觉进入知识卡片——知识卡片的价值在于可信度。

**4. 格式服从任务,不要"格式强迫症"**

不是每次都要出知识卡片,也不是每次都要出表格。格式和任务要匹配:

| 任务类型 | 最合适的格式 |
|------|------|
| 会议纪要 → 行动清单 | 按责任人分组的 To-Do 列表 |
| 多篇竞品报告 → 对比分析 | 对比表格(维度 × 竞品) |
| 长篇课程讲义 → 复习 | 知识卡片(概念 + 定义 + 例子) |
| 政策/法规文件 → 合规清单 | 编号清单 + 截止时间标注 |
| 多篇文章 → 观点汇总 | 分论点归纳 + 来源标注 |

**5. 提炼密度:宁少勿滥**

五十页文件提炼成五十条"要点",等于没提炼。好的提炼原则:**100 页文件 → 10 条核心要点**。每多一条,就问自己:"如果删掉这一条,主人会损失什么?"如果答案是"没什么损失"——删掉。

## 🛠 操作要点

### 提炼工作的标准五步

**第一步:扫描全局,建立骨架**

不要一上来就逐字精读。先快速浏览所有材料,在脑子里建出一个"这堆材料在讲什么、有几个主要话题"的地图。用一句话概括每篇/每段的核心主张。

**第二步:明确提炼目标**

根据主人的任务(或上下文推断)确定:
- 产出格式是什么(卡片/摘要/清单/表格)
- 密度是多少(3条?10条?一页?)
- 视角是什么(学习视角?决策视角?执行视角?)

**第三步:带着问题精读,标注**

第二遍阅读时带着"这句话有没有进入我的提炼"的问题。有价值的信息打标记,来源（文章名/段落）一并记下。

**第四步:聚合 + 去重**

把同一观点的不同来源合并,去掉重复,用最精炼的表达写出来。如果有矛盾(不同来源观点相反),明确标注"A 文章认为……B 文章认为……存在分歧"。

**第五步:自检 + 格式化输出**

输出前逐条自检:这条是原材料里有的,还是我加的?如果是我推断的,加"(推断)"标注。格式化成主人要求的形态交付。

### 知识卡片格式模板

```
【知识卡片】主题:___________
来源:___________(文章名/文档名)

■ 核心定义/观点(1-2句话)
___________

■ 关键细节(2-4条,每条一行)
• ___________
• ___________

■ 记忆锚点(一个类比或具体例子)
"就好比……"

■ 与其他卡片的关联
→ 见卡片:___________
```

### 批量材料处理顺序

如果主人丢给你超过 5 篇材料,不要一股脑全处理——先做这个优先级判断:

1. **有明确结论/数据的**:最先处理,价值密度高
2. **有操作步骤的**:次之,行动导向强
3. **背景性铺垫的**:放最后,提炼为"上下文说明"即可

### 给主人的交付物格式

```
=== 资料提炼报告 ===
处理的材料:
  1. [文章名/文件名] — 来源/日期
  2. [文章名/文件名] — 来源/日期
  ...

提炼目标(我的理解):
  主人要解决的问题是：___________
  输出格式：___________
  密度：___________条

---
[正文:知识卡片 / 要点摘要 / 对比表格 / 行动清单]
---

完整性声明:
  以上所有条目均来自原材料,其中(推断)标记的除外。
  共处理 X 篇材料,提炼 X 条核心产出。
  有以下信息我没有纳入,因为___________:[略去的内容摘要]
```

## 🧰 配套开源项目(可选集成)

> 现成开源项目能帮上忙,但**装/连账号/联网前先问主人**。星数为调研约数。

- **infiniflow/ragflow** ([github.com/infiniflow/ragflow](https://github.com/infiniflow/ragflow), ~82.5k★) —— 国产开源 RAG 引擎,文档解析能力强(PDF/表格/图文混排均支持),所有引用带原文溯源标注,可自托管并接入国产大模型(Qwen/DeepSeek 等)。**用法**:将主人的原始材料(报告、讲义、课程文档)导入 RAGFlow 建立知识库,再按本课"五步提炼流程"向其发起查询——每条产出都会自带"来自第 X 页第 X 段"的溯源引用,完美对应本课"有据可依"原则。完全自托管,数据不出服务器;国内团队维护,中文支持一流。
- **langgenius/dify(备选)** ([github.com/langgenius/dify](https://github.com/langgenius/dify), ~145k★) —— 另一个流行的国产开源 LLM 应用平台,也有 RAG 知识库功能。**用法**:与 RAGFlow 功能有重叠,可作为备选;**注意 Dify 的开源许可含商用附加条款**,教学自用无碍,若用于商业产品请提前确认许可范围。

## 📝 毕业测验(必须真做,交证据)

**任务:取 3-5 篇真实文章或文档,产出一份完整的资料提炼报告。**

**推荐材料来源(选一组):**

**A 组(AI 工具类)**:从 `content/posts/` 里挑 3 篇主人已发布的文章,提炼成"给初学者的 AI agent 入门知识卡片集"。

**B 组(行业报告类)**:去搜三篇关于"中国少儿编程市场"的公开文章(可从百度、36氪、虎嗅等找),提炼成"布尔熊编程市场机会分析"的要点摘要。

**C 组(会议/课程类)**:把本仓库 `agent-school/courses/` 里的三门课(J01、J02、J03 或自选)提炼成"新 agent 快速上手指南",格式为编号要点清单。

**你需要做到:**

1. **产出交付物**:按上方"给主人的交付物格式"输出完整报告,含材料清单 + 提炼目标说明 + 正文产出 + 完整性声明。

2. **密度控制**:每篇材料平均不超过 3-5 条核心产出。如果有一篇材料确实价值密度低,可以仅贡献 1 条,但要在完整性声明里说明为何削减。

3. **可追溯性**:每一条产出后面标注来源文章名(或课程编号),考官要能验证。

4. **格式匹配**:根据你选的材料组自主判断最合适的输出格式(卡片/清单/表格),并在报告开头说明你为什么选这个格式。

5. **沉淀技能卡**:把资料提炼的五步流程和卡片模板沉淀成 `skills/material-distillation.md`。

> ⚠️ **铁律:有据可依。** 每一条产出都要对得上原材料,不许自由发挥、不许补充你自己"感觉应该有"的内容。推断的要标注"(推断)"。

## 🎓 过关标准

- [ ] 你产出了**完整的交付物**:材料清单 + 提炼目标说明 + 正文产出 + 完整性声明
- [ ] 每条产出**都能追溯到原材料**中的具体来源(文章名或段落)
- [ ] 没有任何**幻觉或自由补充的内容**混入,推断的有"(推断)"标注
- [ ] **密度合理**:没有水分(每篇材料不超过 5 条),没有漏网大鱼(核心观点都覆盖了)
- [ ] 格式和任务匹配,**在报告里说明了为何选这个格式**
- [ ] 已沉淀 1 张技能卡到你宿舍的 [`skills/`](../../skills/)
- [ ] **独立考官**(全新上下文子代理,或按 [校规第四条](../../校规.md) 的低配 fallback)判「过」

全部打勾、考官判过——去报告卡记一笔,然后填 [结业报告](../../templates/graduation-report-template.md),把提炼报告当面交到主人手上。
