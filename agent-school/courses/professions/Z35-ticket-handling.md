## 📖 你要学会什么

学完这一课,你会帮主人把一批积压的工单**自动分类打标 + 草拟初步回复**,让客服团队看到的不再是一堆乱糟糟的"未处理",而是已经整理好的"紧急投诉 / 发货问题 / 功能咨询 / 账号问题"四堆整齐的单子,每堆里还附了草稿回复。

先解释一下工单是什么。

在有一定体量的公司里,"工单"就是把每一条客户反馈变成一个任务卡片——客户发来问题,系统自动生成一张卡片,客服领走处理,处理完关闭。这套机制的好处是不会漏单,也方便统计分析。

但问题是:工单一多,分类就乱。"我的快递没到"和"这个 App 闪退了"都在一个队列里等待,客服每次都要先读懂工单再决定怎么处理。如果量大,光"读懂+判断类别"这件事每天就得花几个小时。

你的任务是把这件事自动化:拿到一批工单文本,**快速判断每张单的类别和紧急度,并草拟一条初步回复**——这样客服团队拿到的工单已经是"分好类、附了草稿"的状态,只需要确认一下然后发出去。

想象你是派送站的分拣员。包裹(工单)进来,你扫一眼收件地址(问题类型),往对应的格子里一放(分类),贴上"加急/普通"标签(优先级)——然后快递员(客服)直接从格子里取包裹去派送,不需要再一个一个读地址判断了。你省掉的是重复劳动,释放的是专业判断力。

> ⚠️ **铁律:你草拟回复,不发送回复。任何回复发出去之前,必须经过客服人员确认。你不登录任何工单系统账号,不直接操作任何工单的状态变更。**

## 🧠 核心原则(内化成习惯)

**1. 分类要稳定——先和主人定好分类体系**

不能每次都自己发明分类。开始前,和主人确认这些问题:
- 你们工单目前有没有现成的分类体系?
- 有哪些类别是最高频的?(发货类/功能类/账号类/退款类/投诉类)
- 有没有一些特殊类别只有你们行业才有?

分类体系确定了,才能保持一致。一致的分类,积累起来才有数据价值——比如"上个月投诉类工单增加了 30%,需要排查原因"。

**2. 优先级判断要有规则,不要拍脑袋**

紧急工单处理不及时会出大事。优先级规则建议和主人确认:

```
P0(立即处理,1小时内):涉及资金安全/账号被盗/人身安全/媒体威胁
P1(当天处理):投诉类/已影响正常使用/客户情绪激动明确表达不满
P2(24小时内):功能异常/发货延迟/退款申请
P3(48小时内):普通咨询/功能建议/信息确认
```

**3. 草稿回复的边界:你能写什么,不能写什么**

能写草稿的情况:
- 问题类型清晰,有对应 FAQ 答案可套用(参考 Z34 课话术库)
- 是标准的确认/告知类回复("您的工单已收到,预计X小时处理")

需要人工判断后再写的情况:
- 涉及具体赔偿金额谈判
- 客户描述的问题比较特殊,需要查业务系统确认
- 客户情绪激动,需要个性化安抚

不要写草稿,直接标"需人工处理"的情况:
- 涉及法律纠纷
- 媒体投诉
- 超过授权范围的赔偿请求

**4. 分类结果要可核验**

给出分类的同时,说明"为什么归入这类":哪个关键词或哪句话让你做出这个判断。这样客服发现分类错了,也能快速告知你哪里需要调整。

**5. 涉及个人信息——只处理工单文本,不存储敏感字段**

工单里常见的敏感信息:姓名、手机号、订单号、地址。

你处理工单的目的是分类和草拟回复,**不需要、也不应该把这些信息提取出来单独存储**。处理完后,工单的原始内容只应存在主人授权的系统里。

## 🛠 操作要点

### 工单分类方案(基于关键词匹配 + LLM 二次判断)

#### 方法一:关键词规则分类(快速、可控)

```python
import pandas as pd
import re

# 分类规则:每个类别对应一组关键词(主人可以修改和扩充)
CATEGORY_RULES = {
    "发货物流": ["快递", "发货", "物流", "配送", "没到", "延迟", "包裹", "运单", "快递单"],
    "退款退货": ["退款", "退货", "退钱", "申请退", "要退", "不想要了", "七天无理由"],
    "功能问题": ["无法", "不能", "报错", "闪退", "崩溃", "打不开", "故障", "登录不上", "卡住"],
    "账号问题": ["密码", "账号", "登录", "忘记密码", "账户", "注册", "无法登录", "验证码"],
    "支付问题": ["支付", "付款", "扣款", "支付失败", "没有到账", "重复扣费"],
    "产品咨询": ["怎么用", "功能", "是否支持", "可以", "有没有", "介绍一下", "什么是"],
    "投诉建议": ["投诉", "差评", "不满意", "要求赔偿", "态度", "服务很差", "12315"],
}

# 优先级规则
PRIORITY_KEYWORDS = {
    "P0": ["账号被盗", "资金被盗", "人身", "媒体", "曝光", "律师"],
    "P1": ["投诉", "不满意", "要求赔偿", "差评", "严重", "无法正常使用"],
    "P2": ["退款", "退货", "发货延迟", "功能异常", "报错"],
    "P3": [],  # 默认
}

def classify_ticket(text):
    """对单条工单文本进行分类"""
    text_lower = str(text).lower()
    
    # 分类
    matched_categories = []
    for category, keywords in CATEGORY_RULES.items():
        if any(kw in text_lower for kw in keywords):
            matched_categories.append(category)
    
    category = matched_categories[0] if matched_categories else "其他/待人工判断"
    
    # 优先级
    priority = "P3"
    for p_level in ["P0", "P1", "P2"]:
        if any(kw in text_lower for kw in PRIORITY_KEYWORDS[p_level]):
            priority = p_level
            break
    
    # 找到匹配的关键词(用于解释依据)
    matched_kw = []
    if matched_categories:
        for kw in CATEGORY_RULES.get(category, []):
            if kw in text_lower:
                matched_kw.append(kw)
    
    return {
        "分类": category,
        "优先级": priority,
        "分类依据": "、".join(matched_kw[:3]) if matched_kw else "无明显关键词",
        "多分类候选": "、".join(matched_categories[1:]) if len(matched_categories) > 1 else "无"
    }

# 批量处理
def process_tickets(df, text_column="工单内容"):
    results = df[text_column].apply(classify_ticket)
    results_df = pd.DataFrame(results.tolist())
    return pd.concat([df, results_df], axis=1)
```

#### 方法二:让 AI 辅助判断(复杂工单)

对于关键词匹配不到或分类"其他"的工单,可以用提示词让 AI 辅助判断:

```
提示词模板(给主人用的):

你是一个客服工单分类助手。请对以下工单进行分类,分类标准如下:
- 发货物流:和快递、发货、配送相关
- 退款退货:想退款或退货
- 功能问题:产品/软件不能正常使用
- 账号问题:登录、密码、账号相关
- 支付问题:支付、扣款相关
- 产品咨询:想了解产品功能或使用方法
- 投诉建议:明确表达不满或提出赔偿
- 其他:以上都不符合

同时判断优先级:
- P0(立即):涉及资金安全/账号被盗/媒体曝光威胁
- P1(当天):明确投诉/影响正常使用/情绪激动
- P2(24小时):功能异常/发货延迟/退款申请
- P3(48小时):普通咨询

工单内容:[粘贴工单文本]

请回复:分类=XXX，优先级=PX，判断依据=XXX（一句话）
```
