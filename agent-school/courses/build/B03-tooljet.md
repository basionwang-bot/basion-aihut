你见过那种花好几万定制开发、小团队根本负担不起的内部管理系统吗?进销存、客户管理、订单跟踪……这些东西对小店和小团队来说像奢侈品。

ToolJet 想打破这个局面。它是一个**无代码/低代码的内部工具构建平台**,拖拖拽拽几个小时,就能搭出一个可以真正使用的管理后台——连接你的数据库或表格,控制数据的增删改查,设置权限,多人协作。

ToolJet 在 GitHub 上是开源版"内部工具构建器"这个赛道的头部项目(可替代 Retool、Appsmith),社区版免费自托管,企业版收费。

## 🧠 核心原则

1. **ToolJet = 积木式后台搭建器。** 想象给你一盒乐高——按钮、表格、表单、图表、下拉框……60+ 种 UI 积木,拖到画布上,连上数据源,就变成了一个真实的管理界面。不用写 HTML,不用写 CSS,逻辑用可视化流程图点出来。

2. **数据源是它的"血管"。** ToolJet 的精髓是"连接器"——它能直接连 PostgreSQL、MySQL、MongoDB、REST API、Google Sheets、飞书表格……把数据拉进来展示、操作。你不是在"造假数据",是在操作真实的数据。这也是安全红线的来源:**连上真实数据库的那一刻,这就不是玩具了。**

3. **内置数据库降低入门门槛。** ToolJet 自带一个叫 **ToolJet Database** 的无代码数据库(基于 PostgreSQL),你可以直接在界面里建表、填数据,不需要额外装数据库软件。新手入门时就用这个,不需要折腾 PostgreSQL。

4. **多人协作是它的强项。** ToolJet 支持邀请团队成员、设置角色权限(Admin/Developer/Viewer),这让它真正适合小团队共用一套后台。**但"共用"意味着多人在操作真实数据,上线前必须和主人确认权限设置。**

5. **社区版 vs. 企业版。** 社区版(CE)免费开源,已经够小团队用了;企业版增加 SSO、审计日志、自定义 SAML 等企业功能。这门课只讲社区版。

## 🛠 操作要点

### 第一步:用 Docker 启动 ToolJet

```bash
docker run \
  --name tooljet \
  --restart unless-stopped \
  -p 80:80 \
  --platform linux/amd64 \
  -v tooljet_data:/var/lib/postgresql/13/main \
  tooljet/try:ee-lts-latest
```

命令说明:
- `--name tooljet`:给容器起名,方便后续管理
- `--restart unless-stopped`:电脑重启后自动恢复
- `-p 80:80`:把容器的 80 端口映射到本机 80 端口
- `-v tooljet_data:/var/lib/postgresql/13/main`:数据持久化到本机 Docker volume,容器删了数据还在

启动后浏览器访问 `http://localhost:80`,看到注册/登录页面就成功了。

> ⚠️ **启动前先征得主人确认。** Docker 会拉取镜像(几百 MB),会占用 80 端口,会在本机创建数据卷——这些都要先告知主人。
>
> 如果主人的 80 端口被占用,改成 `-p 3000:80`,然后访问 `http://localhost:3000`。

### 第二步:首次注册管理员账号

访问 `http://localhost:80`,按提示注册第一个管理员账号。**这是本地账号,数据全在你自己机器上**。

### 第三步:建一个进销存后台(用内置数据库)

**建表:**
1. 左侧菜单点 **Database → ToolJet DB**
2. 点 **+ New Table**,建三张表:
   - `products`(产品表):字段 `name(文本)`、`stock(数字)`、`price(数字)`、`category(文本)`
   - `customers`(客户表):字段 `name(文本)`、`phone(文本)`、`note(文本)`
   - `orders`(订单表):字段 `customer_name(文本)`、`product_name(文本)`、`qty(数字)`、`total(数字)`、`status(文本)`

**搭界面:**
1. 左侧点 **Apps → + New App**,起名"进销存后台"
2. 拖一个 **Table** 组件到画布,连接 `products` 表,显示产品列表
3. 拖一个 **Form** 组件,让用户可以添加新产品
4. 给"添加"按钮加事件:触发一个 Query,向 `products` 表插入数据
5. 重复以上步骤给 `customers` 和 `orders` 表搭界面

> **小技巧:ToolJet 有 AI 辅助功能。** 在 Query 编辑器里可以用自然语言让 AI 生成 SQL,不会写 SQL 的主人也能用。

### 第四步:预览和分享

点右上角 **Preview** 按钮,进入真实交互界面,测试添加/查看/删除数据。

如果要让团队成员也能用:Settings → Members 里邀请成员,设置权限角色。

### ⚠️ 安全红线

**无代码/AI工具一旦"连真实数据库、配生产key、对外部署"就从玩具变真系统。三件事——连真实数据、暴露上线、产生账单——都要明确"先问主人再做":**

```
□ 启动 Docker 容器——先征得主人确认,说明会占用端口和硬盘空间
□ 如果要连接外部真实数据库(MySQL/PostgreSQL)——先问主人,这不是测试环境
□ 如果要让团队成员访问——先问主人配置好权限再开放
□ 如果要把 ToolJet 暴露到公网(云服务器部署)——先问主人,涉及安全配置
□ 生产环境务必修改默认密码,配置 HTTPS——上线前告知主人,由主人决策
```
