## 🧠 核心原则

1. **本地运行 = 隐私 + 速度 + 无锁定。** 把 Dyad 想成"安装在你电脑里的私人建站助理"。所有的代码都在你的硬盘上,没有服务器知道你在做什么,网络断了只要本地还在跑就行,停止付费也不会失去任何东西。

2. **"自带 key"是核心设计。** Dyad 不自己提供 AI 能力——它是一个"框架",需要你插入自己的 AI API key 才能运转。好处是:费用是你的,用量是你的,选哪家模型是你的,没有任何中间商赚差价。

3. **代码是真实可用的代码。** Dyad 生成的是真实的 React / Next.js / HTML 项目,存在你的电脑里,可以用 VS Code 打开、继续开发、部署到任何你想去的地方。它不是"只能在平台里看"的代码。

4. **本地跑好 ≠ 可以直接上线。** 在自己电脑上预览一个网站和"把网站发布到互联网上给所有人访问"是完全不同的两件事。**部署上线前,必须先问主人。** 见安全红线。

5. **Bring Your Own Key 的账单责任。** 你填的是主人自己的 API key——每次 AI 生成代码都会消耗 token,产生费用。**在消耗 API 额度之前,必须告知主人**。

## 🛠 操作要点

### 第一步:下载并安装 Dyad

**方式一:直接下载安装包(推荐,新手最快)**

访问 [dyad.sh/#download](https://www.dyad.sh/#download),选择对应系统:
- Mac:下载 `.dmg` 文件,双击安装
- Windows:下载 `.exe` 安装包,双击安装

不需要注册,下载完打开就能用。

**方式二:从源码本地运行(开发者)**

```bash
# 克隆仓库
git clone https://github.com/dyad-sh/dyad.git
cd dyad

# 安装依赖
npm install

# 创建必要目录
mkdir -p userData

# 启动
npm start
```

> ⚠️ **安装前先征得主人确认。** 下载安装包或执行 npm install 前告知主人,说明这是什么软件、会安装在哪里。

### 第二步:配置 AI API Key

打开 Dyad 后,点击界面里的 **Settings(设置)** 或 API Key 配置入口,填入你选择的 AI 提供商 key。

**国内可直接使用的提供商(推荐):**

| 提供商 | 开通渠道 | 是否需翻墙 | 免费额度 |
|--------|----------|-----------|---------|
| DeepSeek | [platform.deepseek.com](https://platform.deepseek.com) | 否 | 有 |
| Qwen(通义千问) | [dashscope.aliyun.com](https://dashscope.aliyun.com) | 否 | 有试用 |

**海外提供商(需翻墙 + 海外信用卡,酌情使用):**

| 提供商 | 门槛说明 |
|--------|---------|
| OpenAI | 需海外手机号 + 信用卡 |
| Anthropic | 需海外手机号 + 信用卡 |
| Google Gemini | 需海外账号 |

> 🇨🇳 **建议:**中国用户优先选 **DeepSeek**——国内注册、有免费额度、编程能力强,和 Dyad 配合效果好。

### 第三步:用 AI 生成一个网站

1. 在 Dyad 的聊天输入框里描述你想要的网站,比如:

```
帮我做一个个人简介页面,包含:照片占位符、一句话介绍、技能列表、联系方式区域。
风格现代简洁,用浅色主题。
```

2. AI 会自动生成代码,Dyad 界面会显示实时进度。
3. 生成完毕后,点击**预览**按钮在浏览器里查看效果。
4. 如果不满意,继续在聊天框里说"把颜色改成蓝色"或"加一个作品集区域"——迭代修改。

### 第四步:查看和导出代码

Dyad 生成的代码存在你本地的项目文件夹里,可以直接用 VS Code 等编辑器打开继续修改。

### ⚠️ 安全红线

**无代码/AI工具一旦"连真实数据库、配生产key、对外部署"就从玩具变真系统。三件事——连真实数据、暴露上线、产生账单——都要明确"先问主人再做":**

```
□ 安装 Dyad——先征得主人确认(说明是什么软件)
□ 填写 API key——告知主人这是主人自己的 key,每次生成都消耗额度
□ 如果 API 免费额度用尽——告知主人,不要静默产生账单
□ 生成的网站"本地预览"没问题,要"部署到公网"——必须先问主人
□ 如果网站要收集用户信息/接入支付——涉及真实数据,先问主人
```
