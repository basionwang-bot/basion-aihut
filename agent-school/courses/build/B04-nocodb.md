## 📖 你要学会什么

Airtable 很好用,但在国内访问不稳定,而且数据都存在别人的服务器上,免费版限制多,进阶功能要月费。

NocoDB 就是**开源自托管版的 Airtable**——GitHub 约 63.4k ★。把它装在你自己的机器或服务器上,你就拥有了一个和 Airtable 几乎一样的界面:电子表格、表单、看板、画廊、日历……数据 100% 在你自己手里,完全免费。

学完这门课,你能:
1. 用 Docker 一条命令把 NocoDB 跑起来
2. 把一份数据(Excel/CSV 或手动录入)变成 NocoDB 表格
3. 搭一个表单用于收集数据,一个看板用于状态管理
4. 知道"本地测试"和"对外开放收集数据"之间的红线在哪

## 🧠 核心原则

1. **NocoDB = 给数据库换一张"好看的人脸"。** 想象一下:底层是一个 PostgreSQL 或 SQLite 数据库(冷冰冰的表格、SQL 命令),NocoDB 给它套上了 Airtable 那样的友好界面。你的团队成员不用懂 SQL,打开浏览器就能像操作电子表格一样操作数据库。

2. **视图是 NocoDB 的魔法。** 同一份数据,加个"视图"就能变换形态:
   - **Grid**:电子表格,看全部数据
   - **Gallery**:图片卡片式展示,适合产品库
   - **Kanban**:看板,适合状态流转(待办→进行中→完成)
   - **Form**:对外收集数据的表单,分享链接给任何人填写
   - **Calendar**:日历视图,适合有日期字段的数据

3. **数据就在你自己机器上。** Docker 启动时挂载了本地目录,数据库文件存在你指定的位置。这是 Airtable 给不了你的:数据主权。

4. **共享表单 = 对外收集数据。** NocoDB 可以生成一个公开链接,任何人都能通过浏览器填写表单把数据录进你的库。**这个功能一旦开启,就意味着真实数据会流入你的系统——先问主人再开放。**

5. **REST API 是它的隐藏宝贝。** NocoDB 为每张表自动生成 REST API,可以供外部系统读写数据。**连接外部系统就进入了生产级别的使用——先问主人。**

## 🛠 操作要点

### 第一步:用 Docker 启动 NocoDB

**SQLite 版(本地测试,最简单):**

```bash
docker run -d \
  --name noco \
  -v "$(pwd)"/nocodb:/usr/app/data/ \
  -p 8080:8080 \
  nocodb/nocodb:latest
```

启动后访问 `http://localhost:8080/dashboard`。

命令说明:
- `-v "$(pwd)"/nocodb:/usr/app/data/`:把当前目录下的 `nocodb` 文件夹挂载进容器,数据持久化在本地
- `-p 8080:8080`:映射 8080 端口
- 数据库用 SQLite,存在挂载目录里,简单轻量,适合本地测试和小团队

**PostgreSQL 版(团队使用,更稳定):**

```bash
docker run -d \
  --name noco \
  -v "$(pwd)"/nocodb:/usr/app/data/ \
  -p 8080:8080 \
  -e NC_DB="pg://host.docker.internal:5432?u=root&p=password&d=d1" \
  -e NC_AUTH_JWT_SECRET="569a1821-0a93-45e8-87ab-eb857f20a010" \
  nocodb/nocodb:latest
```

> ⚠️ **启动前先征得主人确认。** Docker 会拉取镜像、占用端口、在本地创建数据目录——这些都要先告知主人。PostgreSQL 版还需要主人已有可用的 PostgreSQL 数据库。**`NC_AUTH_JWT_SECRET` 在生产环境必须换成随机字符串,绝不能用示例值。**

### 第二步:首次登录并建库

访问 `http://localhost:8080/dashboard`,按提示创建管理员账号。然后点 **+ New Base** 创建一个数据库,起个名字。

### 第三步:导入数据或手动建表

**方式一:导入 Excel/CSV(最快)**

1. 进入你的 Base,点左上角 **+ → Upload File**
2. 上传一份 `.xlsx` 或 `.csv` 文件
3. NocoDB 自动识别列名和数据类型,生成对应的表和数据

**方式二:手动建表**

1. 点 **+ New Table**,填表名
2. 点 **+** 添加字段,选字段类型:
   - 单行文本、长文本、数字、货币、百分比
   - 日期、勾选框、下拉选择、多选标签
   - 附件、链接(外键关联)

### 第四步:添加不同视图

在左侧视图列表点 **+ Add view**,可选:
- **Gallery**:点 **+ Add view → Gallery**,选一个图片字段作为封面
- **Kanban**:选一个下拉或单选字段作为"列分组字段"(如状态字段:待处理/进行中/完成)
- **Form**:自动生成一个可分享的填写链接

### 第五步:搭一个收集表单

1. 在视图里点 **+ Add view → Form**
2. 拖动字段调整顺序,隐藏不想让填写者看到的字段
3. 点 **Share Form** 获取公开链接

> ⚠️ **分享表单前先问主人**——公开链接意味着任何人都能向你的数据库里写数据,确认主人知情并同意。

### ⚠️ 安全红线

**无代码/AI工具一旦"连真实数据库、配生产key、对外部署"就从玩具变真系统。三件事——连真实数据、暴露上线、产生账单——都要明确"先问主人再做":**

```
□ 启动 Docker——先征得主人确认,告知会占用端口和本地存储
□ 分享表单公开链接——先问主人,公开意味着外部可以向数据库写入数据
□ 开启 REST API 给外部系统调用——先问主人,这是生产级接入
□ 把 NocoDB 暴露到公网——先问主人,需要配置 HTTPS 和安全设置
□ 生产部署时 NC_AUTH_JWT_SECRET 必须换成随机值——告知主人,不能用示例值
```
