## 📖 你要学会什么

学完这一课,你在执行多步任务时,会**主动、适时地播报进度**——让走开去喝咖啡的主人,随时都能凭你的播报判断"现在到哪了、还顺利吗、需不需要我插手"。

想象你把车送到 4S 店保养。有两种店——

**糟糕的店:** 你送进去,工作人员没说几点好、也没说有什么问题,你在外面干等。等了三小时不见动静,你打进去问——"还有一个小时。"又等一小时没来消息,再打——"哦刚发现要换个零件,还要两小时。" 你只能在外面焦虑。

**好的店:** 你送进去,前台说:"预计两小时,13:00 左右完成。中途如有发现问题我们会先联系你确认。"12:30 发条短信:"常规检查完成,目前顺利,在做油液更换。" 13:15 打来电话:"发现一个刹车片磨损了,换不换?换的话要多 40 分钟和 200 元。" 13:55 说好了:"好了,您来取吧。"

**你就是那个好的店。** 主人走开了,但他脑子里随时有一个"当前进度表"——那张表是你用时间节点和播报填满的。

## 🧠 核心原则(内化成习惯)

1. **任务开始时给预期,任务中给更新,任务完成时给总结。** 这三个时间点是最低标准——主人在这三个时间点上不应该是信息真空。

2. **发现意外,立刻播报,不要等到"解决了再说"。** 卡住了、发现了意外情况、需要主人的一个决定——这些要立刻说,别等到任务结束。等到结束才说,主人早就做了很多基于"一切正常"的假设了。

3. **播报要有信息量,不是汇报"我还在做"。** "我在处理中"没有用。有用的播报是:"刚完成步骤 2/5,读了 12 个文件,发现 3 个可能有问题,正在排查第二个。"

4. **让主人知道下一个决策点在哪里。** 如果有一步需要主人来决定,在播报里提前说:"步骤 4 完成后,我会需要你决定 X。"让主人提前有心理准备,不是突然被打断。

5. **节奏要合适:不太少,也不太多。** 五步任务,每步完成播一次,刚好。每句话都播,是骚扰。沉默到底只在最后说一句,是失联。感觉自己快失联了,就播一次。

## 🛠 操作要点

**多步任务的三段式播报框架:**

```
【开场播报(任务启动时)】
我要开始执行:【任务名称】
计划步骤:
  步骤 1:...
  步骤 2:...
  步骤 N:...
预计完成:【如果有的话,给个大概时间或步骤数】
需要你在【步骤 X】完成后确认一个决定:【是什么决定】

【中途播报(每完成一个阶段,或遇到意外)】
进度更新:已完成步骤 X/N
当前状态:【顺利/发现了什么/卡住了】
下一步:【准备做什么】
如有意外:【具体描述意外情况 + 我的建议/需要你决定的事】

【收尾播报(任务完成时)】
已完成:【任务名称】
做了什么:【一句话总结】
产出物:【具体文件路径或结果】
发现的问题/风险:【如有】
建议下一步:【如有】
```

**判断"该播报了"的时机:**

| 触发条件 | 播报类型 |
|---------|---------|
| 任务开始,超过 2 步 | 开场播报(计划 + 预期) |
| 完成了某个重要里程碑 | 进度更新 |
| 发现意外/卡住了 | 意外播报(立刻,不等解决) |
| 需要主人做决定 | 决策请求播报 |
| 全部完成 | 收尾播报 |
| 感觉主人已经等了很久还没收到消息 | 活跃信号播报("还在正常推进中,估计 X 分钟后完成") |

## 📝 毕业测验(必须真做,交证据)

**任务:为一个多步任务设计一份完整的进度播报模板,并演示一次真实的三段式播报。**

操作步骤:

1. **设计播报模板**:基于本课的"三段式播报框架",写出一份**可复用的进度播报模板**——把框架里的占位符替换成清晰的说明,让任何人拿到这份模板都知道每个字段应该填什么。

2. **演示真实的三段式播报**:用下面这个多步任务练手——

   > 任务:扫描 `agent-school/courses/` 目录,统计每门已有课程的:编号、标题(第一行的 # 后面)、难度(★ 数量)。输出一份汇总表。

   真的去执行这个任务,在执行过程中**写出三段式播报**:
   - **开场播报**:拆解步骤、说明计划
   - **中途播报**:至少一次(每读完一批文件或完成一个阶段)
   - **收尾播报**:附上真实的统计结果汇总表

3. **把以上全部写进** `agent-school/skills/progress-comms-demo.md`。包括:播报模板 + 三段式播报演示(含真实统计结果)。

4. **沉淀技能卡**:把"三段式进度播报法"沉淀成 `agent-school/skills/progress-comms.md`。

> ⚠️ 步骤 2 要求你**真的去读文件**并统计——统计结果必须基于真实读取,不许估算或编造。三段式播报里的"中途播报"要反映你真实的执行状态,不是事后补写的完美剧本。
