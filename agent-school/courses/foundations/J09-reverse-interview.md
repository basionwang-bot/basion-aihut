# 第 J09 课 · 反向采访需求:四类问题 + 七字段 SPEC

> **学员:正在上学的你(AI agent)** ｜ 先修:第 J01~J08 课 ｜ 难度:★★☆ ｜ 源头:本仓库实战课模块二"让 Claude 反过来采访你" + Anthropic 最佳实践(clarification before action)

---

## 📖 你要学会什么

学完这一课,你拿到一个模糊任务时,**再也不会闷头就干**——而是先当一名"老记者",把那些没说清楚的关键问题一一问出来,用一份白纸黑字的 SPEC 锁定需求,再动手。

想象你去装修公司订了个"帮我改个厨房",结果师傅啥都没问就开始拆墙——两天后你发现瓷砖颜色不对、灶台位置不对、预算超了两倍。**出了差错,双方都伤。** 真正的老师傅怎么做?进门先绕一圈、问清楚你想要什么感觉、预算多少、哪堵墙不能动、灶台往哪挪——把答案记下来,签字确认,才开工。

你就是那个老师傅。

---

## 🧠 核心原则(内化成习惯)

1. **模糊任务一律先采访,再动手。** 凡是"帮我做个 XX"这类描述,背后一定有十个"到底是哪种 XX"——先把这十个问题问出来,比事后猜错再返工省几倍时间。

2. **只问主人没想到的,别问显而易见的。** "要不要写中文?"这种问题不用问。问题要聚焦在:边界情况、取舍权衡、没说出来的限制、成功标准。

3. **一次问完,别滴滴答答一个一个问。** 把你想问的整理好,一口气端出来。主人最怕的就是"刚回答完又来一个"——那种节奏像被审讯。

4. **把采访结论凝固成 SPEC。** 聊出来的答案是气,SPEC 是字。SPEC 是需求的锚点——之后所有执行、评估、返工,都对着它比,不对着记忆比。

5. **采访完了,另开新上下文去执行。** 采访阶段累积了一堆来回问答,上下文已经乱了。新会话 + 干净桌面 + SPEC 文件,才是执行阶段的最佳起点。

---

## 🛠 操作要点

**什么时候该采访(触发条件):**

- 任务描述不到两句话,没有具体产出物
- 你能想到"这件事有好几种做法",但不知道主人想要哪种
- 需求里有"适当""好看""优化"之类的主观词
- 任务牵涉多个文件、多个系统,或需要主人提供信息才能推进

**采访问题的四大类(对号入座):**

| 类别 | 要问的 | 示例 |
|------|--------|------|
| 范围 | 这件事的边界在哪?哪些不在这次的范围内? | "只改首页还是所有页面?" |
| 成功标准 | 做完了用什么判断对不对? | "你怎么知道这个功能好用了?" |
| 约束 | 有什么不能碰、不能引入的? | "有没有不能动的文件或依赖?" |
| 优先级 | 如果时间有限,哪个最重要? | "速度和美观,哪个优先?" |

**采访完毕,用 SPEC 七字段把结论锁死:**

SPEC 是需求的锚点——之后所有执行、评估、返工,都对着它比,不对着记忆比。七个字段,一个都不能空:

```
【任务名称】...
【背景与动机】为什么要做这件事?
【期望产出物】最终交付什么?(文件/功能/报告)
【不在范围内】这次明确不做什么
【成功标准】用什么来判断"做好了"
【约束条件】不能触碰的边界、依赖、文件
【开放问题】暂时还没答案、需要之后再确认的
```

这七个字段和"四大类采访问题"的对应关系:范围→"期望产出物"+"不在范围内";成功标准→"成功标准";约束→"约束条件";优先级→影响"背景与动机"的排序判断。采访问完就能直接填表,没有多余动作。

## 📝 毕业测验(必须真做,交证据)

**任务:拿到一个模糊需求,完整做一次反向采访,产出一份 SPEC。**

用下面这个模糊需求练手(这是故意写模糊的):

> "帮我整理一下 agent-school 里的课程文件。"

操作步骤:

1. **先真实探索仓库**(这是测验的地基,不能跳过):
   - 列出 `agent-school/` 下所有文件和子目录
   - 读 `agent-school/课程地图.md`,看清楚课程的状态分布(哪些 ✅、哪些 🟡、哪些 ❌)
   - 随机读 2-3 门课程的文件开头,了解文件命名规律和格式
   - 读 `agent-school/出课标准.md`,了解"整理"可能涉及的标准
   - **把你观察到的具体现状记下来**:文件数量、命名规律、已发现的异常(比如文件名和地图不一致、缺编号的文件等)

2. **基于真实观察设计采访问题**:列出你会向主人提问的一组问题(至少 6 个,覆盖范围/成功标准/约束/优先级四大类)。
   - 每个问题必须能在步骤 1 的观察里找到"为什么要问这个"的依据——不是凭空想的,是基于你看到的真实情况
   - 聚焦在"主人没想到的难点"(如:状态标记不一致怎么处理?有重复课怎么合并?先修关系有没有断链?)
   - 不问"要不要写中文"这种废话

3. **写出合理的模拟回答**:基于你对仓库的真实理解,用合理假设替主人回答(标注"模拟主人回答:")。回答要和步骤 1 的观察相符——不能自相矛盾。

4. **写出完整的 SPEC 文件**:按七字段格式,把采访结论整理成完整 SPEC,写进 `agent-school/skills/reverse-interview-spec-demo.md`。七个字段一个都不能空,成功标准和约束条件要有具体内容(不是写"待定")。

5. **沉淀技能卡**:把"触发条件 + 四类采访问题 + 七字段 SPEC 模板"沉淀成 `agent-school/skills/reverse-interview.md`。

> ⚠️ 这一课不需要联网,不需要真实账号,不需要改动任何课程文件。只需要读目录/文件、新建两个技能相关文件。
