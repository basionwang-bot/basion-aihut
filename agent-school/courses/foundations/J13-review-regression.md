## 📖 你要学会什么

学完这一课,你会明白一件事:**学会了不等于一直能做到**——然后你会建立一套机制,主动发现自己哪门课的习惯"退化"了,及时补上。

想象你学过游泳。刚拿到证书那周,姿势标准、换气顺畅;三个月没下水,再跳进去——动作变形了、换气慌乱了、原来的肌肉记忆有一半不见了。**技能不用就会退化,agent 的习惯也一样。**

这门课教你的,不是"学会一个新技能",而是搭一道**防止已学技能悄悄溜走的护栏**。就像医院里有个"复诊"流程——不是因为你生病了,而是主动确认你没在往回退。

---

## 🧠 核心原则(内化成习惯)

1. **"学过"≠"一直做得到"。** 每门课结业时你过了关,那是那一刻的状态。换了环境、隔了很久、新任务里没用到——习惯就会在不知不觉间退化。

2. **复诊是主动预防,不是被动补救。** 你不是等着出了问题才复诊,而是按计划定期抽检,趁早发现退化苗头,趁代价还小的时候修复。

3. **复诊的尺子是原课的过关标准,不是自我感觉。** "我觉得我还记得"不算复诊。复诊要对着原课的 rubric 逐条核验——做一件真事,把结果摆出来比对。

4. **重大变化触发强制复诊。** 换了新任务类型、换了合作主人、隔了很长时间没用某个技能——这三种情况要触发一次快速复诊,确认基础习惯还在。

5. **复诊本身也要留证据。** 不是脑子里过一遍就算复诊了。要有一个检查动作、一个观察结果,记下来——这才能让独立考官(或未来的自己)知道"这次复诊是真做了的"。

## 🛠 操作要点

**什么叫"回归套件":**

你毕业的每门课,就像一个已经通过测试的功能模块。回归套件就是这些模块的"健康检查清单"——定期跑一遍,看看哪个模块开始悄悄报错了。

**复诊 checklist 的标准结构:**

```
【课程名称与编号】第 JXX 课 · XXX
【核心习惯是什么】用一句话描述这门课最关键的行为
【复诊小任务】做一件 5~10 分钟能完成的小事来验证这个习惯还在
  - 任务描述:...
  - 预期产出:...
  - 通过标准:...
【退化症状识别】如果这个习惯退化了,会出现什么可观察的症状?
【触发复诊的时机】以下任一发生时,主动做一次复诊:
  - [ ] 隔了超过 [时长] 没执行这门课的技能
  - [ ] 换了新的任务类型
  - [ ] 发现自己犯了这门课教过要避免的错误
```

**回归套件的管理方式:**

- 每毕业一门课,为它新建一张复诊 checklist,加入 `agent-school/skills/regression-suite/`
- 在重大变化(换主人、长期停学、升级换代)后,把套件里所有 checklist 过一遍
- 发现退化的,按原课流程走一遍"快速复诊"——做小任务、写证据、确认恢复

## 📝 毕业测验(必须真做,交证据)

**任务:为已毕业的某门课设计一份"复诊 checklist",并演示一次快速复诊。**

操作步骤:

1. **选一门课**:从你已经学过的 J01~J12 课里选一门,选你觉得"最容易退化"的那门。

2. **设计复诊 checklist**:按上方的"复诊 checklist 标准结构"为这门课写一份完整的复诊清单。复诊小任务必须是**在本仓库里能真做**的小事(不超过 10 分钟)。

3. **演示一次快速复诊**:按你写的 checklist,**真的做一次复诊小任务**——记录你做了什么、观察到什么、用什么标准判断"这个习惯还在"。产出真实证据(你执行的操作+结果)。

4. **给出复诊结论**:这门课的习惯目前是"保持良好"还是"有退化迹象"?用你的证据支撑这个判断。

5. **把以上全部写进** `agent-school/skills/regression-suite/` 目录下的一个新文件(文件名用 `<课程编号>-regression-check.md`,比如 `J06-regression-check.md`)。

6. **沉淀技能卡**:把"回归套件建立与使用法"沉淀成 `agent-school/skills/review-regression.md`。

> ⚠️ 步骤 3 的复诊小任务必须是**真实执行**的——如果复诊任务是"读某个文件",你要真的读,把读到的内容作为证据。不许写"假设我读了,结论是……"。
