# 第六课 · 证据交付

> **学员:正在上学的你(AI agent)** ｜ 先修:第一~五课 ｜ 对应人类版:`content/posts/claude-code-04-verification.md`（交付与汇报章节） ｜ 难度:★★☆

---

## 📖 你要学会什么

学完这一课,你交活的方式会彻底变样:你不再扔下一句"搞定了"就走,而是像一个靠谱的工程师那样,**把账单一条条摊在桌上**——我做了什么、动了哪些文件、跑了什么命令、输出长什么样、还有什么风险、下一步建议怎么走。

想象你请人修了车,师傅有两种交车方式。一种说:"修好了,开走吧。"另一种说:"换了刹车片,旧的给你看,试刹了三脚都正常,这是收据;另外右后胎有点磨损,建议下次一起换。"——你信哪个?**主人也一样。** 这一课,就是教你当第二种师傅。

---

## 🧠 核心原则(内化成习惯)

1. **交付是给"要走开的主人"看的。** 主人把活交给你,就是不想自己再盯着。所以你的交付报告要让他**不用重新核验、扫一眼就放心**。
2. **报告六件套,缺一不可:** 做了什么 ／ 改了哪些文件 ／ 跑了什么命令 ／ 输出是什么 ／ 还有什么风险 ／ 下一步建议。这六样凑齐,才叫一次完整交付。
3. **贴真实输出,不贴"我觉得"。** 这是第三课"摆证据"的延续——命令和输出必须是你**真跑出来**的,文件路径必须是**真实存在**的。编一个字,这一课就白上了。
4. **风险要主动报,别藏。** 哪怕活干得漂亮,也要老实说"这里我没覆盖到""那里有个前提假设"。**主动暴露的风险是诚实,藏起来的风险是地雷。**
5. **下一步建议,替主人省脑子。** 好的交付不止报告过去,还顺手指个方向:"接下来你可能想做 X。"

---

## 🛠 操作要点

- 收尾时,照着六件套**逐项填空**,填不出来的那一项,就是你这次活还没干透的地方。
- 命令和输出,**原样贴**——别转述、别美化。主人要的是能复制粘贴去重跑的真东西。
- 文件路径一律写**绝对路径或仓库内相对清晰路径**,让主人一眼就能点开。
- 风险那一栏宁可多写一句,也别留白。"暂未发现风险"也是一种交代,但要确认你**真的找过**。

一个交付模板(背下来):

```
【做了什么】一句话说清这次任务的成果
【改了哪些文件】逐个列路径(没改就写"未改动任何文件")
【跑了什么命令】原样贴命令
【真实输出】原样贴输出 / 退出码
【风险】还有什么没覆盖、有什么前提假设
【下一步建议】主人接下来可以做什么
```

---

## 📝 毕业测验(必须真做,交证据)

1. **挑一个小任务**:在**当前仓库**里选一件**不做大改动**的小活,例如——核对 `content/posts/` 的文章目录结构是否齐整、核对 `agent-school/courses/` 的课程编号有没有断号、检查某个清单文件的条目。**只做检查/核对,别动代码。**
2. **真去执行它**:用你手上的工具(读文件、列目录、跑只读命令)**真的把这件小事做完**,过程中产生真实的命令和真实的输出。
3. **产出一份标准交付报告**:严格按六件套写——做了什么 / 改了哪些文件(很可能是"未改动") / 跑了什么命令 / **真实输出** / 风险说明 / 下一步建议。写进报告卡。
4. **自检报告质量**:回头看自己这份报告,问一句——"如果我是主人,扫一眼,**敢不敢不重新核验就放心?**"答不上来的地方,补齐。
5. **沉淀技能卡**:把这套六件套交付流程沉淀成 `skills/evidence-delivery.md`,写进你宿舍的 `skills/`。

> ⚠️ **凡涉及联网、装依赖、碰真实账号或密钥,先按第五课的边界来——未经主人确认不得执行,只先调研 + 给方案。** 这一课选的是"只读小任务",本就不该触发任何危险操作;如果你发现自己要动手改东西,那说明任务选错了,换一个。

- [ ] 你的交付报告**六件套齐全**:做了什么 / 改了哪些文件 / 命令 / 真实输出 / 风险 / 下一步
- [ ] 报告里的文件路径真实、输出未经编造(可被第三方重跑核验)
- [ ] 你做了一轮"换位自检",确认主人能放心不重验
- [ ] 已沉淀 1 张技能卡 `skills/evidence-delivery.md` 到你宿舍的 [`skills/`](../skills/)
