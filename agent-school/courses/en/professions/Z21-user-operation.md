# Lesson Z21 · User Operations: Retention & Re-engagement

> **Student: you, the AI agent currently enrolled** ｜ Prerequisites: Lessons J01–J07; Z20-User Research (recommended) ｜ Difficulty: ★★☆ ｜ Source: GrowthHackers community retention cases [growthhackers.com](https://growthhackers.com/); Mailchimp email best practices [mailchimp.com/resources](https://mailchimp.com/resources/); Braze / Iterable / Klaviyo lifecycle marketing guides; general push notification and SMS compliance standards

## 📖 What you'll learn

After this lesson, you will help the owner produce a **user retention and re-engagement plan**: segment users into tiers, specify the outreach cadence for each tier, and draft the copy — three pieces packaged together so the owner has a ready-to-execute operations playbook.

First, let's talk about why retention beats acquisition in ROI.

Many founders are obsessed with growth — acquire, acquire, acquire. But here's a sobering number: the cost of acquiring a new customer is typically 5–7× the cost of retaining an existing one. If you spend $100 bringing someone in and they leave after one use — you'd have done far better investing that $100 in the customers you already have.

Put simply: **keeping the people you have is almost always cheaper than finding new ones**.

Think of yourself as the owner of a neighborhood shop. Retention operations means: remembering your regulars' names, knowing what they usually buy, reaching out before they forget about you, sending a small discount on their birthday. Treating every returning customer like a friend — that's the craft of real operations.

Your job is to design that "greeting mechanism" — turn it into a system that can run at scale, so the owner always knows: "who to reach out to, when, and with what."

> ⚠️ This lesson produces plans and copy drafts only. Do not log in to any email platform, push notification service, SMS system, or community tool to send real messages on the owner's behalf.

## 🧠 Core principles (internalize these as habits)

**1. Segment first, then reach out — don't treat everyone the same**

Sending the same message to every user means no one feels it was written for them.

Good operations means: this message goes only to "users who haven't come back in three months" — they read it and think "oh, you remembered me."

**2. Outreach cadence should feel like a friend, not an advertiser**

A friend messages you occasionally, with substance; an advertiser messages you daily, always selling.

Good cadence: maximum 2 touches per week, with content that has warmth and information value — not just discount codes. Exceed that frequency and users unsubscribe/block, which is worse than no contact at all.

**3. Copy should speak the user's language, not the brand's language**

Brand voice: "We've fully upgraded our product, introducing exciting new features!"
User voice: "That report you never finished? You can knock it out in 5 minutes now. Try it here."

One says "look what we did"; the other says "here's what's in it for you."

**4. Every message must have one clear action**

After reading this message, the user should know exactly what to do next. "Click here" / "Reply YES" / "Open the app to claim" — one call-to-action per message, never more.

**5. Frequency and channel should match user state**

| User state | Recommended channel | Recommended frequency |
|-----------|--------------------|-----------------------|
| Active users | In-app notification / email newsletter | Once a week — feature/content push |
| 7 days inactive | Email / push notification | Once |
| 30 days inactive | Email + push (if enabled) | Once, with genuine value |
| 90+ days inactive | Email / SMS (low-cost channels) | 1–2 times, then stop |

## 🛠 How to do it

### Step 1: User segmentation framework

Divide users into four tiers by activity level, each with a distinct operations goal:

```
[USER LIFECYCLE FOUR-TIER MODEL]

Tier 1 · New users (within 7 days of registration/first use)
- Goal: Help them reach the "first moment of value" — actually experience what the product does
- Core action: Guide them through key onboarding steps (setup / core feature activation)
- Criteria: Registered within the past 7 days

Tier 2 · Active users (used the product at least twice in the past 30 days)
- Goal: Deepen usage habits, increase frequency
- Core action: Feature discovery, quality content, community engagement
- Criteria: ≥2 logins/uses in the past 30 days

Tier 3 · Dormant users (31–90 days without any activity)
- Goal: Reactivate — bring the memory back
- Core action: Re-engagement copy + incentive (discount / new feature / personalized nudge)
- Criteria: No login/use in the past 31–90 days

Tier 4 · Churned users (90+ days without any activity)
- Goal: Low-cost probe; stop high-cost investment
- Core action: One final touch; if no response, stop
- Criteria: No activity of any kind in 90+ days
```

### Step 2: Outreach cadence plan

A ready-to-use 90-day cadence template:

```
[NEW USER ACTIVATION CADENCE (first 7 days)]
Day 0:  Sign-up instant → Welcome message + core feature walkthrough (auto-triggered)
Day 1:  Next day → "You're off to a great start" encouragement + next-step prompt
Day 3:  Day 3, if key action not yet completed → Reminder + help link
Day 7:  Day 7 → "You've been here a week" milestone message + advanced feature recommendation

[ACTIVE USER MAINTENANCE CADENCE (weekly)]
Weekly once:   In-app or email push (new feature / curated content / user story)
Monthly once:  Email newsletter (depth content — no sales intent)
Special moments: Product anniversary / user birthday / major holidays → personalized message

[DORMANT USER RE-ENGAGEMENT CADENCE (three-step)]
Re-engagement touch #1 (dormant day 31):
  Channel: Email + push notification (if opted in)
  Content: Personalized reminder — reference what they left unfinished
  If no response in 7 days → proceed to touch #2

Re-engagement touch #2 (dormant day 45):
  Channel: Email
  Content: Real value offer (limited discount / major new feature announcement)
  If no response in 7 days → proceed to touch #3

Re-engagement touch #3 (dormant day 60):
  Channel: Email / SMS (if opted in for marketing)
  Content: Sincere farewell + one last value reminder
  If no response → stop outreach; move to "churned" pool
```

A ready-to-adapt copy template for each touchpoint scenario:

```

[Product name] team

Hi [name],

It's been a while. Last time you were in [product name], you [specific action or where they left off].

Since then, we've shipped [one feature or improvement genuinely relevant to them] — it might solve exactly what you ran into before.

[Come take a look] (link)

If you'd rather not hear from us, you can unsubscribe below.

[Product name] team
---

[DORMANT USER — RE-ENGAGEMENT TOUCH #2 — Email with incentive]
Subject: A 30% discount, just for you — expires in 3 days
---

Use code COMEBACK30 for 30% off — good for 3 days only.

[Claim offer] (link)

---

[DORMANT USER — FINAL FAREWELL — Email]
Subject: [Product name] — last message from us
---
Hi [name],

You've been away for a while, and we respect that. Maybe we didn't deliver what you were hoping for.

If you're willing to tell us what went wrong, just reply to this email — we read every one.

If not, that's completely fine. Wishing you well.

[Product name] team
To stop receiving messages: [unsubscribe link]
---

[ACTIVE USER — NEW FEATURE PUSH — Email / In-app]
Subject: [Feature name] is live — fixes what you told us was annoying
---
A lot of you told us [pain point scenario] was a hassle.

We fixed it.

→ [Feature name]: [One sentence on what it does now]
→ How to use it: [2–3 steps]
→ [Try it now] (button)

[Product name] team
---
```
