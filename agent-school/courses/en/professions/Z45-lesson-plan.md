## 📖 What you'll learn

After this lesson, you will help the owner produce a **lesson plan ready to use for teaching preparation or a program pitch** — not a pile of knowledge points, but a complete instructional document with learner analysis, learning objectives, class flow, activity design, and assessment.

Picture an experienced curriculum coach. A new teacher walks in and asks: "I have to teach 'Introduction to Artificial Intelligence' next week — how do I prepare?" You don't hand them a textbook and say "read it yourself." You sit down with them and work through: What is this lesson trying to accomplish? What do students already know, and what don't they know yet? How do we divide 45 minutes? What activity will make the concept stick? How do we know whether they learned it?

That's what this lesson teaches you — how to systematically design a single class session or a unit.

---

## 🧠 Core principles (internalize these as habits)

1. **Know your learners before you know your content.** "Introduction to AI" taught to Grade 4 students is a completely different lesson than the same topic taught to Grade 8 students — different vocabulary, different examples, different activities. The first step in lesson design is answering "who exactly are these students, what do they already know, and what do they care about?" This is called learner analysis. It's the skeleton; everything else is flesh.

2. **Write learning objectives as observable behaviors, not vague wishes.** "Students will understand AI" is not an objective — how do you know whether a student "understood"? A good objective is: "Students will be able to name 3 real-world applications of AI." or "Students will successfully build and run a simple classification program using block-based coding." An observable behavior = a good objective.

3. **One lesson solves one core problem.** The most common mistake new teachers make is trying to cram everything in. In 45 minutes, if you genuinely teach one core concept and run one hands-on activity well, that's a great lesson. Trying to cover everything means students walk away remembering nothing — the real failure.

4. **Activity design must get students moving.** Forty-five minutes of straight lecture puts brains into sleep mode fast. A good lesson plan always has students talking, doing, discussing, or demonstrating — even just "turn and share with your neighbor for 2 minutes" is far more effective than monologue.

5. **Assessment is not a test — it's how the teacher finds out what students actually learned.** The lesson plan must include "how do I know at the end whether the objective was met?" — this could be an exit ticket, a show-of-hands question, a work-in-progress display, or a quick quiz. A lesson plan without assessment is flying blind.

## 🛠 How to do it

### Standard lesson plan structure (general K12 / training format)

```
I. Basic Information
   Course title:
   Grade / age group:
   Duration: (XX minutes / lesson X of X in a unit)
   Setting: (School classroom / enrichment centre / online live session)
   Designer: (Leave blank for the owner to fill in)

II. Learner Analysis
   Target students: (Age / grade / baseline level)
   Prior knowledge: (What students already know coming in)
   Anticipated difficulty: (Where students are most likely to get stuck)

III. Learning Objectives
   Knowledge & Skills: (Students will be able to "name / identify / complete / operate" _____)
   Process & Method: (How students will learn — by doing / exploring / discussing)
   Attitude & Engagement: (What attitude or interest this lesson aims to develop)

IV. Key Teaching Points & Challenges
   Key points:
   Challenges:

V. Resources & Preparation
   Materials / references:
   Equipment / tools:
   Student preparation:

VI. Instructional Sequence (the main body — time-allocated)
   ┌─────────────────────────────────────────────────────────────┐
   │ Phase          │ Time  │ Teacher activity │ Student activity │ Intent │
   ├─────────────────────────────────────────────────────────────┤
   │ Hook / warm-up │ 5 min │                  │                  │        │
   │ Instruction    │20 min │                  │                  │        │
   │ Practice /     │15 min │                  │                  │        │
   │   activity     │       │                  │                  │        │
   │ Wrap-up /      │ 5 min │                  │                  │        │
   │   assessment   │       │                  │                  │        │
   └─────────────────────────────────────────────────────────────┘

VII. Assessment Plan
   Assessment method: (Observation / Q&A / work display / exit ticket / quiz)
   Success criteria: (How to judge whether objectives were met)

VIII. Homework / Extension (optional)
   Assignment:
   Extension resources:

IX. Teaching Reflection (filled in after delivering the lesson; may be left blank at design stage)
```

### Special notes for Boolean Bear Coding context

If designing a lesson for **Boolean Bear Coding** (enrollment demos or regular instruction), keep in mind:

- **Coding tools**: The main tools for K12 programming courses are Scratch (block-based, elementary school) and Python (text-based, middle school and up). The lesson plan must specify which tool and which version.
- **Curriculum alignment**: Map lesson objectives to relevant learning standards in the target country / region (e.g., computational thinking, digital literacy, information-technology skills).
- **Parent perspective**: Add a "Parent note" subsection — what will the child take away from this lesson? How does it connect to school curriculum? What level of prior experience is assumed?
- **Enrollment demo use**: If the lesson plan will also be used for enrollment pitches, make the objectives and student activities more vivid and concrete — "Students design their own game and present it to the class" sells better to parents than "students master conditional statements."

### Quick-reference parameters for different contexts

| Context | Suggested duration | Emphasis | Assessment method |
|---------|-------------------|----------|-------------------|
| Elementary Scratch intro (enrollment demo) | 45 min | Hands-on experience + sense of achievement | Work display |
| Middle school Python basics (regular class) | 90 min | Concept instruction + lab practice | Code runs correctly |
| K12 AI literacy elective | 45 min | Discussion + case analysis | Exit ticket + group report |
| Adult / parent AI tools intro (training) | 2 hours | Hands-on + scenario demo | Complete a task on the spot |
