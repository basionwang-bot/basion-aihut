# Lesson Z06 · One Draft, Many Platforms: Adapting a Single Piece into Platform-Native Versions

## 📖 What you'll learn

After this lesson, you can take any existing piece of content and **do genuine style migration** — making it read as though it was written natively for each of Newsletter/Blog, Instagram, Reddit/Medium/Quora, and short-form vertical video, rather than pasted from somewhere else.

Let's talk about what this really is.

The laziest way to publish on multiple platforms is **direct copy-paste** — the same suit worn to the library, the backyard barbecue, and the gym. Not wrong, just tone-deaf. Slightly better is **adjusting the format** — add sub-headers for the blog, add emojis for Instagram. But that's still treating the symptom, not the cause: the layout changed, the voice didn't.

Real multi-platform publishing is **style migration** — taking the same core idea and saying it again in each platform's native language. It's like translating a book: not converting Chinese sentences word-for-word into English, but re-saying the thing in English the way an English speaker would say it, so readers feel it was written for them.

This lesson's core value comes down to three things:
1. Recognize the tonal differences between five platforms (one reference table)
2. Master style migration as an operation (not formatting — perspective-switching)
3. Know the publishing-order strategy (so platform algorithms don't flag it as reposts)

> 📌 For detailed production techniques per platform, see the relevant prior courses: Instagram → **Z02**, long-form platforms → the Z09-equivalent lesson. This lesson doesn't repeat those. It covers only "how to migrate the same piece to multiple platforms."

## 🧠 Core principles (internalize these as habits)

**1. Extract the core idea first, then start rewriting**

The prerequisite of multi-platform publishing is a clear understanding of the piece's core idea. Before rewriting, complete this sentence: "The one thing this piece most wants to convey to the reader is: ______." That thing stays constant across all platforms. Only how it's said changes.

**2. Five-platform tone reference table**

| Platform | Tone | Language characteristics | Sweet-spot length | Reader mindset |
|----------|------|------------------------|-------------------|----------------|
| **Newsletter / Blog** | Depth, perspective | Somewhat formal, clear logical structure | 600–2,000 words | Sitting down to read properly |
| **Instagram** | Real, recommendatory, friend-energy | Conversational, emoji-heavy, very short paragraphs | 200–500 words | Scrolled past and stopped |
| **Reddit / Medium / Quora** | Expert, rational, authoritative | Data / citations, tight logic | 800–3,000 words | Came here specifically looking for this |
| **Short-form vertical video caption** | Warm, friend-sharing feel | Concise + genuine, shareable | 100–300 words | Saw it while scrolling, about to share |
| **TikTok / Reels script** | Fast-paced, hook-driven | Short sentences, pause feel, hook formulas | 150–400 words (spoken) | Three seconds before swipe |

**3. Change the perspective, not just the format**

Take the same "AI boosts productivity" piece:
- **Newsletter version**: from the "analyst" perspective — "Why so many people use AI wrong, and the problem is…"
- **Instagram version**: from the "first-person experiencer" — "Three months of writing content with AI — here are five things I genuinely felt"
- **Reddit / Quora version**: from the "explainer" perspective — "How do you actually improve content creation efficiency with AI agents? Let me break it down properly"

Same core idea. Three different answers to "who am I, and why am I telling you this?" **That is style migration — not swapping a header and some emojis.**

**4. Publishing order is a strategy, not an afterthought**

Don't publish everywhere simultaneously on the same day. Recommended sequence:
- **Newsletter / Blog** first (most permanent, establishes original authorship)
- **Reddit / Medium / Quora** 1–2 days later
- **Instagram** another 1–2 days after that
- **Short-form video** flexibly

Two reasons: first, this reduces the chance that platform algorithms flag the content as reposts; second, publishing on newsletter/blog first establishes an original-creation record that helps if plagiarism issues arise elsewhere.

**5. Draft is draft, publish is publish**

Your job ends at "hand all platform-version drafts to the owner." Logging in and posting — those are the owner's moves, never yours.

## 🛠 How to do it

### The four-step style migration workflow

Think of yourself as a **translation studio** that received a source text and needs to produce editions in multiple languages:

```
Step 1: Understand the source
  - Distill the core idea in one sentence (must be articulable)
  - Tag what is "essential information that must survive" vs. "packaging that can be rephrased"
  - Identify who the original audience was

Step 2: Rewrite for each platform separately (not minor edits — rewrites)
  - For each platform, first ask "what posture does this platform's reader use while consuming content?"
  - Check the tone table above; confirm perspective and register
  - Don't tweak the original — start fresh from the core idea

Step 3: De-AI-ify every version
  - Check: does it contain "certainly," "firstly / secondly / finally," "in conclusion,"
    "overall," "it's worth noting"? Change them.
  - Tool-generated versions are especially prone to template feel

Step 4: Side-by-side self-check
  - Put three versions next to each other and ask: "Are the tone / structure / length
    noticeably different?"
  - If differences are minor, you only changed format not style — rewrite.
```

### Core handles for each platform

**Newsletter / Blog version:** Opening needs a hook (intrigue / pain point / counter-intuitive claim); each section 3–5 paragraphs under a sub-header; closing has a call to action or memorable line. Full techniques in Z01.

**Instagram version:** Caption leads with emoji, first two lines hit a pain point; paragraphs very short (2–3 lines then break); "recommendation feel" not "ad feel"; closing has engagement hook. Full techniques in **Z02**.

**Reddit / Medium / Quora version:** Opening can be "problem context analysis" or a clean direct conclusion; logically ordered (1, 2, 3 or first / then / finally); data / case studies / citations for credibility; longer word count than other platforms.

**Short-form vertical video caption version:** Opening like a quick friend's note, concise and warm; under 300 words; format suitable for sharing directly from a mobile screen.

## 📝 Graduation test (do it for real, submit evidence)

**Task: take a real article from the repo's `content/posts/` directory, rewrite it into at least 3 platform-native versions, and side-by-side verify that differences are meaningful.**

**Steps:**

1. **Select source material**: browse `content/posts/`, pick an article worth adapting (recommended: `claude-code-01-mindset.md` or `claude-code-03-workflow.md`). Write one sentence summarizing the core idea of this article.

2. **Rewrite three platform versions**: from the core idea, **rewrite separately** for Newsletter/Blog + Instagram + Reddit/Medium. Each version must be complete and usable — not an outline.
   - These are rewrites **starting from the core idea**, not light edits of the original.

3. **Side-by-side self-check**: place the three versions next to each other and explicitly answer:
   - Are the **tones** noticeably different? (e.g., analyst / experiencer / explainer)
   - Are the **structures** noticeably different? (e.g., Instagram very short paragraphs vs. Reddit logical layers)
   - Do the **lengths** fall within the reference table ranges?
   - If any dimension shows little difference — the rewrite is surface-level. Redo it.

4. **De-AI-ify check**: in each version, find one paragraph and check for "certainly," "in conclusion," "overall," "not only that," "it's worth noting" — find them and replace them. Record your changes.

5. **Publishing-order recommendation**: write a ≤100-word "publishing plan for the owner": in what order do these three versions get published, with how much spacing, and why?

6. **Skill card**: Distill "four-step style migration workflow + five-platform tone table + publishing-order template" into `skills/multi-platform-content.md`.

> ⚠️ **Iron rule, restated: drafts only, never publish.** Do not log in to any platform or operate any content publishing system. Your job ends at "hand all platform versions to the owner."
