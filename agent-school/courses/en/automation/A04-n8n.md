## 🧠 Core Principles

1. **n8n is a hybrid of code and drag-and-drop.** It is neither pure no-code nor pure code — you can drag nodes and draw connections, and you can also write JavaScript or Python inside individual nodes. This makes it more flexible than purely visual tools and easier to get started with than purely code-based tools.

2. **fair-code ≠ open source — this is worth being clear about.** n8n uses the **Sustainable Use License** — source is public, self-hosting is allowed, you can extend it, but **you may not resell n8n itself as a service** (e.g., as a SaaS offering). Personal use, internal business use, and self-modification are all fine. If your host wants to build a commercial service on top of n8n, read the license terms carefully first. (Not legal advice — review the license at n8n's repository before any commercial use.)

3. **n8n-mcp is "n8n's encyclopedia, plugged into Claude".** n8n has 1,851 nodes and a huge amount of documentation. n8n-mcp structures all of that documentation so that once Claude is connected, it has a full n8n knowledge base. Claude no longer has to guess at n8n's usage from training data; it looks up accurate documentation from n8n-mcp in real time — producing more correct workflows.

4. **n8n-mcp and n8n are two separate tools.** n8n-mcp only helps Claude understand n8n; it does not run workflows itself. To actually execute a workflow you still need a running n8n instance (local or cloud).

5. **Get your host's confirmation before installing, connecting accounts, or triggering production flows.** n8n's automation capability is powerful — connecting the wrong account or firing the wrong flow can have real-world consequences: real emails sent, real data deleted. Inform your host at every step.

## 🛠 How-To

### Option 1: Start n8n quickly with npx (simplest)

No installation needed — just run:

```bash
npx n8n
```

> This requires Node.js (version 18.x or above recommended).

Once you see output like `n8n ready on 0.0.0.0, port 5678`, open your browser at:

```
http://localhost:5678
```

The first visit walks you through creating an account and activating n8n.

> ⚠️ **Inform your host before running:** `npx n8n` downloads and runs n8n, and creates a `.n8n/` directory locally to store workflows and credentials.

### Option 2: Start n8n with Docker (more stable, better for long-term use)

```bash
docker volume create n8n_data
docker run -it --rm --name n8n -p 5678:5678 -v n8n_data:/home/node/.n8n docker.n8n.io/n8nio/n8n
```

Data is stored in the `n8n_data` volume and persists across container restarts.

> ⚠️ **Get your host's confirmation before running.**

> 🌐 **Docker image tip:** n8n's image is hosted at `docker.n8n.io`. If pulling is slow in your region, check n8n docs for alternative pull options or use the npx method above.

### Understanding the n8n editor interface

```
Left panel:   All available nodes (search by category)
Center canvas: Drag-and-drop nodes + draw connections
Right panel:  Parameter configuration for the selected node

Two types of nodes:
- Trigger nodes (marked with "▶" on the left): the starting point of a workflow
- Action nodes: the steps executed after the trigger fires
```

A typical workflow looks like this:

```
[Webhook Trigger] → [Data transform (Set/Function)] → [Send Gmail] + [Write Google Sheets]
```

Notice the final step can **branch**: send the same data to multiple action nodes simultaneously.

### Connecting n8n-mcp: loading n8n knowledge into Claude

**Simplest method: use npx (no pre-installation required)**

On Linux/macOS/WSL, connect n8n-mcp to Claude Code with a single command:

```bash
claude mcp add n8n-mcp \
  -e MCP_MODE=stdio \
  -e LOG_LEVEL=error \
  -e DISABLE_CONSOLE_OUTPUT=true \
  -- npx n8n-mcp
```

Windows PowerShell:

```powershell
claude mcp add n8n-mcp `
  '-e MCP_MODE=stdio' `
  '-e LOG_LEVEL=error' `
  '-e DISABLE_CONSOLE_OUTPUT=true' `
  -- npx n8n-mcp
```

**Verify the connection worked:**

```bash
claude mcp list
claude mcp get n8n-mcp
```

Or type `/mcp` in a Claude conversation and check whether n8n-mcp appears in the tool list.

> ⚠️ **Inform your host before adding any MCP Server** — explain what you are doing and what new capabilities Claude will gain.

**Optional: connect to your local n8n instance (enables workflow management)**

If your host wants Claude to not only "know n8n docs" but to actually create and manage workflows inside a live n8n instance, add these two environment variables:

```bash
claude mcp add n8n-mcp \
  -e MCP_MODE=stdio \
  -e LOG_LEVEL=error \
  -e DISABLE_CONSOLE_OUTPUT=true \
  -e N8N_API_URL=http://localhost:5678 \
  -e N8N_API_KEY=your-n8n-API-KEY \
  -- npx n8n-mcp
```

Generate `N8N_API_KEY` in the n8n dashboard under Settings → API Keys.

> ⚠️ **The API Key must be generated and configured by your host.** The key grants operational access to n8n and must not be shared carelessly.

### Real task: have Claude build the "form submission → organize → email + spreadsheet" workflow

Once n8n-mcp is connected, try telling Claude:

> "I want to build a workflow in n8n: when someone submits a Webhook form, organize the form data, then simultaneously send me a Gmail notification and append the record to a Google Sheet. Please help me design the node structure for this workflow."

Claude will:
1. Query the node documentation in n8n-mcp
2. Produce an accurate node list (Webhook → Set → Gmail + Google Sheets)
3. Explain what parameters each node needs to be configured with
4. If `N8N_API_URL` is configured, even create the workflow directly inside n8n on your behalf

**This is the core value of n8n-mcp**: no more guessing — grounded in real documentation for all 1,851 nodes.

### A note on n8n's fair-code license

n8n uses the **Sustainable Use License**. The key rules:
- ✅ Personal use: fine
- ✅ Internal business use: fine
- ✅ Self-hosting, modifying, extending: fine
- ❌ Wrapping n8n itself as a service sold to others (SaaS form): requires an n8n commercial license

This is not a standard OSI-approved "open source license," which is why n8n calls it "fair-code." If your host is only using it for themselves, none of this is a concern.
