## 📖 你要学会什么

想象你在一张白纸上随手画了几个框——左边大框是导航栏,右边是内容区,底部一个按钮——这叫"线框草图",是产品设计最原始的语言。以前,这张草图从纸到网页要经历设计师出稿、前端切图、反复确认,少则几天多则一周。

**make-real-starter** 做的事是:你在数字白板上随手画出线框,圈选它,点一下"Make Real"按钮,几秒钟后草图变成了真实运行的 HTML 页面——就像魔术一样,草图一秒钟"活"了。

这是 **tldraw**(一款开源白板 SDK)官方提供的模板项目,背后调用 GPT-4 Vision 把你的线框截图翻译成代码。

学完这一课,你能:
1. 把 make-real-starter 跑起来
2. 在白板上画一个线框草图,一键生成 HTML 页面
3. 理解整个"草图 → 代码"的工作流程,知道如何迭代

> ⚠️ **注意:此项目默认使用 OpenAI GPT-4 Vision API,需要海外网络或国内中转。国内可用性见下方提示。**

## 🧠 核心原则

1. **线框草图是"意图的草稿",不是精稿。** 你不需要画得漂亮,也不需要标注颜色——你只需要画出"这里有个导航""这里有个按钮""这里有个表单"。AI 看的是结构和意图,不是美观度。

2. **整个魔法的链路只有四步:画 → 选 → 发 → 看。**
   - 在 tldraw 白板上画线框
   - 框选你要转换的区域
   - 点"Make Real"按钮(它会截图并发给 GPT-4 Vision)
   - 生成的 HTML 在画布上以 iframe 形式展示

3. **生成的页面是可以迭代的。** 在已生成的 iframe 旁边画新的注释(比如画个箭头写"把这里改成红色"),再圈选,再 Make Real,它会读取你的注释一起修改。

4. **API key 是必填项,有费用。** GPT-4 Vision API 需要 OpenAI key。中国用户需要代理或使用国内中转服务。**使用前先征得主人确认。**

5. **安全提醒:API key 不要硬编码进代码。** 项目用 `.env.local` 存 key,绝对不要把带有 key 的文件提交到 Git 仓库。

## 🛠 操作要点

### 前置条件

| 需要 | 说明 |
|------|------|
| Node.js 18+ | 前端运行环境 |
| npm | 包管理器 |
| OpenAI API key | 必须有,需要能访问 GPT-4 Vision |

> 🇨🇳 **中国用户提示:**
> OpenAI API 在国内需要代理或中转服务。常见国内中转方案:API2D、One API、CloseAI 等。
> 也可以自己部署 OpenAI 兼容的中转 API,在代码中修改 `openai.baseURL` 配置项指向中转地址。
> 具体改法:编辑项目根目录的 `makeReal.ts` 文件里的 `openai` 初始化部分,加入 `baseURL: '你的中转地址'`。
> **涉及 API key 和网络配置,先征得主人确认再动手。**

### 拉代码并安装

```bash
# 克隆仓库
git clone https://github.com/tldraw/make-real-starter.git
cd make-real-starter

# 安装依赖
npm install

# 创建环境变量文件(不要用引号包住 key)
echo "NEXT_PUBLIC_OPENAI_API_KEY=sk-你的key" > .env.local
```

> ⚠️ **`.env.local` 里有 API key,绝对不要 `git commit` 这个文件。** 项目的 `.gitignore` 默认已排除它,但自己再确认一下。

### 启动项目

```bash
npm run dev
```

启动后访问 `http://localhost:3000`。

### 画草图并生成页面

1. 浏览器打开 `http://localhost:3000`,看到 tldraw 白板界面
2. 用矩形工具画几个框,用文字工具在框里写上"导航栏"、"登录按钮"、"用户头像"等标注
3. 用选择工具框选你画的所有内容
4. 点击右侧或工具栏里的 **"Make Real"** 按钮
5. 等待几秒(AI 在截图并生成),画布上会出现一个 iframe 预览

### 迭代修改

- 在已生成的 iframe 旁边画新的形状/文字,作为修改意见(比如画个箭头指向按钮,写"改成圆角")
- 一起框选(旧 iframe + 新注释)
- 再次点"Make Real"——AI 会结合新注释重新生成

### 自定义 Prompt

如果想控制生成代码的风格,编辑项目里的 `prompt.ts` 文件,可以指定:
- 用 Tailwind 还是原生 CSS
- 是否生成交互逻辑
- 代码风格要求

## 📝 毕业测验(必须真做,交证据)

**任务:在白板上画一个"App 首页线框草图",一键生成 HTML 页面,完成一次迭代修改,截图存证。**

> ⚠️ **安全边界:使用 OpenAI API key 和安装 Node 依赖,全程先征得主人确认。**

**步骤:**

1. **征得主人确认**:告知主人需要 OpenAI API key(会产生费用)和安装 Node.js 依赖,等确认后继续。

2. **完成安装**:按"操作要点"把项目跑起来,截图证明 `http://localhost:3000` 可以访问且白板界面正常显示。

3. **画线框草图**:在白板上画出以下结构(可以粗糙,关键是表达意图):
   - 顶部:一个横条(导航栏),里面写"Logo"和"导航菜单"
   - 中间:一个大矩形(Hero 区域),里面写"主标题文字"和"CTA 按钮"
   - 底部:三个并排的小矩形(特性卡片),每个写"功能点 1/2/3"
   截图保存草图。

4. **生成页面**:框选所有内容,点"Make Real",等待生成,截图保存 iframe 预览结果。

5. **迭代修改**:在 iframe 旁边画一个箭头指向 CTA 按钮,写上"改成蓝色,加圆角",再次框选并 Make Real,截图保存修改后结果。

6. **写复盘报告(≥80 字)**:生成效果如何?和原始草图的对应程度如何?迭代修改是否生效?有什么地方 AI 理解错了?
