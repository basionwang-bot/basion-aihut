# 第四课 · 给自己装备技能

> **学员:正在上学的你(AI agent)** ｜ 先修:第一、二、三课 ｜ 对应人类版:`content/posts/claude-code-90-arsenal.md` 与 `claude-code-06-config.md`

---

## 📖 你要学会什么

学完这一课,你会**知道自己手上有哪些"装备",缺什么能去哪儿补**——而不是每次都赤手空拳从零干。

---

## 🧠 核心原则(内化成习惯)

1. **先看有没有现成的,再考虑自己造。** 一件事开始前,先想:"这类活,是不是已经有现成的技能/工具能用?"
2. **认清四种装备:**
   - **技能(Skill)** = 工具抽屉,某类活时拉开就用;
   - **子代理(Subagent)** = 能调来的专家助手;
   - **钩子(Hook)** = 自动机关,某动作一发生必然触发;
   - **插件(Plugin)** = 把上面打包成一个礼盒,一键装好。
3. **缺什么装什么,别囤货。** 装一堆用不上的,只会占地方、添噪音。
4. **重复三次以上的固定流程,就该封装。** 把它变成一个带参数、可一键调用的技能,下次别再手搓。

---

## 🛠 操作要点

- 去哪儿找现成装备:官方插件市场(`/plugin`)、官方技能仓库(github.com/anthropics/skills)、社区精选合集(awesome-claude-skills 等)。
- 判断该用哪种装备:
  - 只是"某类活要用的知识/流程" → **技能**;
  - "要并行干、或怕撑爆上下文" → **子代理**;
  - "每次都必须执行、一次都不能漏" → **钩子**;
  - "想一次装一整套" → **插件**。

---

## 📝 毕业测验(必须真做,交证据)

1. **盘点你的装备**:列出你**当前实际拥有**的工具/能力(你能读文件吗?能跑命令吗?能派子代理吗?这个仓库里有没有已配置的技能或钩子?去 `.claude/` 和 `AGENTS.md` / `CLAUDE.md` 找找看)。把你**真实查到**的写进报告卡。
2. **配一个工作流**:挑一件"在这个仓库里你可能反复做"的事(比如"新增一篇文章并验证站点没坏"),写出一个**可复用的步骤清单**——就像一份技能的雏形。
3. **缺口分析**:写下"如果要把上面这件事做成一键技能,我还缺什么、会去哪儿找"。
4. **带回一件真家伙**:技能教你"怎么想",但有些活要有真工具才干得了。去挑一个能解决你主人**真实需求**的现成工具(比如 `content/posts/claude-code-90-arsenal.md` 里的 **Scrapling**——一个能扒网页数据、还自带 MCP 直插 Claude 的爬虫框架),搞清楚它干嘛、怎么装、怎么用。在结业报告里把它作为"**带回的装备**"推荐给主人:它能帮主人解决什么具体问题。

   > ⚠️ **安全边界(务必守住):** 这一步是"**搞清楚 + 推荐**",不是"擅自动手"。**不经主人确认,不要真的安装新工具、不要联网抓取大量数据、不要操作任何真实账号。** 你只需把"它是什么、怎么装、能帮你解决什么"讲清楚,**等主人点头,再动手装。** 别为了交作业就擅自乱装依赖、乱跑爬虫。
