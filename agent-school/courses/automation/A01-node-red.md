## 🧠 核心原则

1. **流水线思维:数据是流动的"货物"。** 在 Node-RED 里,每一条"消息"(message)就是一件货物,从左边进入,经过一个个节点的加工,从右边流出。你要做的,就是设计好每个节点做什么,以及货物走哪条线。

2. **节点 = 一个动作,连线 = 执行顺序。** 每个节点只干一件事——"定时触发""发起 HTTP 请求""判断条件""发邮件"。把节点连起来,就是把动作排好顺序。就像乐高积木,每块形状固定,拼法千变万化。

3. **绊线机关思维:有事发生才触发。** Node-RED 是事件驱动的。你不是让它"每秒检查一遍",而是放一根绊线——"每天早上 8 点"或者"一收到 HTTP 请求"——触发了,流水线才跑。不触发,安安静静什么都不做。

4. **本地运行,数据不出门。** Node-RED 跑在你自己的机器上,不用注册账号,不用付费,不用联网(本地任务)。国内网络完全友好——装完即用。

5. **先征得主人确认,再安装、再接外部服务。** 安装 Node-RED 需要 npm/Node.js 环境,改机器全局配置要先告知主人;接飞书、邮件等外部服务时,涉及账号密码,必须主人亲自输入凭据,你不能代劳。

## 🛠 操作要点

### 安装 Node-RED

Node-RED 是一个 Node.js 应用。安装前先确认 Node.js 已就位:

```bash
# 检查 Node.js 版本(需 18.x 或以上)
node --version
```

安装 Node-RED(需先征得主人确认,这一步会全局安装):

```bash
sudo npm install -g --unsafe-perm node-red
```

安装完成后启动:

```bash
node-red
```

> ⚠️ **安装前先告知主人**:`sudo npm install -g` 会全局安装,修改系统 npm 目录。告知主人后再执行。

启动后在浏览器打开:

```
http://localhost:1880
```

> 🇨🇳 **国内友好提示:** Node-RED 完全本地运行,不需要任何科学上网,不需要账号注册,不需要外币付费。npm 安装如果慢,可先切换淘宝镜像:`npm config set registry https://registry.npmmirror.com`

### 理解编辑器界面

打开 `http://localhost:1880` 后,你看到的是这样的界面:

```
┌──────────────┬─────────────────────────────┬──────────────┐
│ 左侧节点面板 │      中间画布(拖放连线区)      │  右侧属性面板 │
│ inject       │                             │              │
│ debug        │  [inject] ──→ [http req] ──→ [debug]      │
│ http request │                             │ 点节点看属性  │
│ function     │                             │              │
│ change       │                             │              │
└──────────────┴─────────────────────────────┴──────────────┘
```

- **左侧**:所有可用节点,按类别分组,从这里拖到画布
- **中间**:你的"流水线画布",连线就在这里画
- **右侧**:选中某个节点后,显示和编辑它的配置

### 安装社区节点(以 node-red-contrib-http-request 为例)

Node-RED 有数千个社区节点。安装方式:

1. 编辑器右上角 ☰ 菜单 → **Manage palette(管理节点)**
2. 搜索节点名 → 点 **Install**
3. 安装完成后节点自动出现在左侧面板

> ⚠️ **安装节点前先告知主人**:社区节点是第三方代码,安装前说明你要装什么、用来干什么。

### 真任务流水线:定时抓网页 → 检测变化 → 推送提醒

这是本课的核心实战流水线,一共 5 个节点:

```
[inject(定时)] → [http request(抓页面)] → [function(提取内容)] → [change(比较上次)] → [debug/通知]
```

**第一步:放一根"定时绊线"**

拖入 **inject** 节点,双击配置:
- Payload: 空(不传数据,只触发)
- Repeat: interval → 每 60 分钟(或你想要的间隔)
- 这就是你的"闹钟"节点——每隔设定时间,自动发一发消息往下游

**第二步:抓网页**

拖入 **http request** 节点,双击配置:
- Method: GET
- URL: 填入你要监控的目标网页(如某招聘页面、某公告页面)
- Return: 选 **a UTF-8 string**
- 这个节点收到上游消息后,立刻去抓那个网址的 HTML 返回给你

**第三步:提取关键内容**

拖入 **function** 节点,双击写 JavaScript:

```javascript
// 从 HTML 里提取你关心的文字
// 这里用简单的字符串截取举例,真实项目可用正则
var html = msg.payload;
// 比如只取 <title> 标签内的内容
var match = html.match(/<title>(.*?)<\/title>/i);
msg.payload = match ? match[1] : "未找到标题";
return msg;
```

**第四步:和上一次比较**

拖入 **function** 节点做变化检测:

```javascript
// 用 context.get/set 在两次调用之间存储上次的值
var last = context.get('lastContent') || '';
var current = msg.payload;

if (current !== last) {
    // 有变化!
    context.set('lastContent', current);
    msg.changed = true;
    msg.payload = "检测到变化! 新内容: " + current;
    return msg;  // 继续往下传
} else {
    return null;  // 没变化,不往下传,流水线在这里"断路"
}
```

**第五步:发出提醒**

- 如果只是本地测试:拖入 **debug** 节点,变化时会在右侧调试面板打印
- 如果想推送飞书/钉钉:安装对应社区节点(如 `node-red-contrib-feishu-webhook`),配置 Webhook 地址
- 如果想发邮件:使用内置 **email** 节点(需先安装 `node-red-node-email`)

> ⚠️ **接外部通知渠道前先问主人**:Webhook 地址、邮箱账号密码等凭据,必须由主人亲自填写。你只负责配置节点结构。

**最后:点击右上角 Deploy 按钮**

蓝色 **Deploy** 按钮 = "把你画的流水线跑起来"。每次改了流水线都要重新 Deploy 才生效。

### 调试技巧

- 在任意连线后接一个 **debug** 节点,就能在右侧调试面板看到这个点的数据长什么样
- inject 节点左边有一个小方块,点一下可以手动立刻触发一次,不用等定时器
- 右上角侧边栏里的 **debug 标签页**会实时打印所有 debug 节点的输出
