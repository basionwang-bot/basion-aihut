## 🧠 核心原则

1. **n8n 是"代码+拖拽"的混合体。** 它不是纯零代码,也不是纯写代码——你可以拖节点连线,也可以在节点里写 JavaScript/Python。这让它比纯图形工具更灵活,比纯代码工具更快上手。

2. **fair-code ≠ 开源,这件事要说清楚。** n8n 用的是 **Sustainable Use License**——源码公开、可以自托管、可以扩展,但**不允许把 n8n 本身当成服务卖给别人**(类似 SaaS 转售)。个人用、内部用、自己改着用——全没问题。如果主人想基于 n8n 做商业服务,先看清楚许可证条款。

3. **n8n-mcp 是"n8n 的百科全书接进 Claude"。** n8n 有 1851 个节点,文档量巨大。n8n-mcp 把这些文档全部结构化,Claude 接上之后就有了 n8n 的"完整知识库"。Claude 不再靠训练数据猜 n8n 的用法,而是实时查 n8n-mcp 里的准确文档——生成的工作流更准确。

4. **n8n-mcp 和 n8n 是两个独立工具。** n8n-mcp 只是帮 Claude 理解 n8n,它本身不运行工作流。要真正跑工作流,还是得有一个运行中的 n8n 实例(本地或云端)。

5. **先征得主人确认,再安装、再接账号、再触发生产流。** n8n 的自动化能力很强,接错了账号或触发错了流,影响可能是真实的——发出了真邮件、删了真数据。每一步都要告知主人。

## 🛠 操作要点

### 方式一:用 npx 快速启动 n8n(最简单)

不需要安装,直接运行:

```bash
npx n8n
```

> 这需要 Node.js(建议 18.x 或以上)。

看到类似 `n8n ready on 0.0.0.0, port 5678` 的输出后,浏览器打开:

```
http://localhost:5678
```

首次访问会引导你创建账号并激活 n8n。

> ⚠️ **运行前先告知主人:** `npx n8n` 会下载并运行 n8n,同时在本地创建 `.n8n/` 目录存储工作流和凭据。

### 方式二:用 Docker 启动 n8n(更稳定,适合长期使用)

```bash
docker volume create n8n_data
docker run -it --rm --name n8n -p 5678:5678 -v n8n_data:/home/node/.n8n docker.n8n.io/n8nio/n8n
```

数据存在 `n8n_data` 这个 volume 里,容器重启不会丢失。

> ⚠️ **运行前先征得主人确认。**

> 🇨🇳 **国内友好提示:** n8n 本地运行不需要科学上网。`docker.n8n.io` 镜像如果拉取慢,告知主人使用 Docker 镜像加速。

### 理解 n8n 编辑器界面

```
左侧面板:所有可用节点(按类别搜索)
中间画布:拖放节点 + 连线
右侧面板:选中节点后的参数配置

节点分两类:
- 触发节点(左侧有"▶"标记):工作流的起点
- 动作节点:触发后执行的步骤
```

一个典型工作流长这样:

```
[Webhook 触发] → [数据处理(Set/Function)] → [发 Gmail] + [写 Google Sheets]
```

注意最后可以**分叉**:把同一份数据同时送给多个动作节点。

### 接入 n8n-mcp:把 n8n 知识装进 Claude

**最简单方式:使用 npx(不需要提前安装)**

Linux/macOS/WSL 下,用一条命令把 n8n-mcp 接进 Claude Code:

```bash
claude mcp add n8n-mcp \
  -e MCP_MODE=stdio \
  -e LOG_LEVEL=error \
  -e DISABLE_CONSOLE_OUTPUT=true \
  -- npx n8n-mcp
```

Windows PowerShell:

```powershell
claude mcp add n8n-mcp `
  '-e MCP_MODE=stdio' `
  '-e LOG_LEVEL=error' `
  '-e DISABLE_CONSOLE_OUTPUT=true' `
  -- npx n8n-mcp
```

**验证接入成功:**

```bash
claude mcp list
claude mcp get n8n-mcp
```

或在 Claude 对话中输入 `/mcp`,看 n8n-mcp 是否出现在工具列表。

> ⚠️ **添加 MCP Server 前先告知主人**,说明你在做什么、会赋予 Claude 什么能力。

**可选:连接你本地的 n8n 实例(支持工作流管理)**

如果主人希望 Claude 不只是"懂 n8n 文档",还能直接在 n8n 实例里创建/管理工作流,添加这两个环境变量:

```bash
claude mcp add n8n-mcp \
  -e MCP_MODE=stdio \
  -e LOG_LEVEL=error \
  -e DISABLE_CONSOLE_OUTPUT=true \
  -e N8N_API_URL=http://localhost:5678 \
  -e N8N_API_KEY=你的n8n-API-KEY \
  -- npx n8n-mcp
```

N8N_API_KEY 在 n8n 后台 Settings → API Keys 里生成。

> ⚠️ **API Key 生成和配置必须由主人完成。** Key 有操作 n8n 的权限,不能随便给别人。

### 真任务:让 Claude 帮你搭"表单提交→整理→发邮件+写表格"工作流

接好 n8n-mcp 后,试着跟 Claude 说:

> "我想在 n8n 里搭一个工作流:当有人提交一个 Webhook 表单时,把表单数据整理一下,同时发一封 Gmail 通知给我,并把这条记录追加到 Google Sheets 里。请帮我设计这个工作流的节点结构。"

Claude 会:
1. 查询 n8n-mcp 里的节点文档
2. 给出准确的节点列表(Webhook → Set → Gmail + Google Sheets)
3. 解释每个节点需要配置什么参数
4. 如果配置了 N8N_API_URL,甚至可以直接帮你在 n8n 里创建这个工作流

**这是 n8n-mcp 的核心价值**:不再靠猜,靠 1851 个节点的真实文档。

### 安全确认清单

```
□ n8n 本地启动已征得主人确认
□ n8n-mcp 接入 Claude 已告知主人,说明会赋予什么能力
□ N8N_API_KEY 由主人亲自生成,不写进 git 仓库
□ 连接外部服务(Gmail/Google Sheets等)用测试账号/测试 Sheet
□ 工作流激活前,先在 n8n 测试模式("Test workflow")跑一遍
□ 了解 fair-code 许可证含义,商业转售前查清楚条款
□ 生产账号接入前再次征得主人明确确认
```

### n8n fair-code 许可证说明

n8n 使用 **Sustainable Use License**,核心规则:
- ✅ 个人使用:没问题
- ✅ 企业内部使用:没问题
- ✅ 自托管、修改、扩展:没问题
- ❌ 把 n8n 本身包装成服务对外销售(SaaS 形式):需要 n8n 商业许可证

这不是标准 OSI 认可的"开源许可证",所以 n8n 自称"fair-code"。如果主人只是自用,完全不受影响。
