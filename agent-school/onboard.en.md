# 🚀 Onboard your AI (recommended cold-start entry)

> 🌐 English ｜ [中文](onboard.md)
>
> **Don't just school your AI — onboard it like a new hire.** Just installed Claude Code and
> haven't tasted the payoff yet? Don't grind through 132 courses. Paste the block below to your
> agent and in **~30 minutes** it will: interview you → speed-run the 2–3 most relevant courses →
> immediately use what it learned to **get one real thing done for you.**
>
> Want systematic depth (the full 132 courses)? Go to [enroll.en.md](enroll.en.md).

---

## 🟢 Copy this to your agent

```text
You're going to "onboard" with me — get up to speed fast and immediately get one real thing
done, instead of slowly finishing every course. Follow these four steps strictly:

【Step 1 · Intake interview (diagnose before teaching)】Ask me these 3 questions and listen,
  don't rush to work:
  1. What do I mainly do for work/study?
  2. What repetitive, tedious task do I want to offload lately? (Have me give 1–2 concrete examples)
  3. What tools or file types do I use most on this machine?

【Step 2 · Prescribe courses (task-driven)】Based on my answers, pick the 2–3 most relevant
  AgentForge courses (MUST include J01 "explore before acting" as the baseline). Prefer
  agent-school/选课索引.md as the basis; otherwise use these common mappings, or MCP
  list_courses / the course map (agent-school/课程地图.md):
   - content / creator work → J01 + Tools (OCR / screenshot) + Media (video editing)
   - spreadsheets / data → J01 + T-pandas + T-DuckDB
   - file tidying / batch ops → J01 + relevant Tools
   - web / design → J01 + Design (screenshot-to-code / shadcn)
  Tell me "which courses, why, estimated time" and get my OK before starting.

【Step 3 · Speed-run graduation (discipline intact)】Do each course's graduation test FOR REAL —
  real commands + real output as evidence, pass an independent proctor or the low-spec fallback
  (see agent-school/校规.md), distill one skill card per course.
  After graduating, per agent-school/graduation-deploy.en.md (WITH MY CONSENT) install the skill
  cards into my global skills directory ~/.claude/skills/.

【Step 4 · Ship on day one (see results now)】Immediately use what you just learned to do the
  real task I mentioned in Step 1 (or a minimal viable version of it). Hand me the result + a
  short onboarding report (using the structure of
  agent-school/templates/onboard-report-template.md): interview notes / courses taken /
  where skill cards were deployed / the first deliverable / 3 things you can do next for me.

Start with Step 1 now — ask me those 3 questions.
```

## Onboarding vs. full depth

| | 🚀 Quick onboard (this) | 🎓 Full study ([enroll.en.md](enroll.en.md)) |
|--|--|--|
| Time | **~30 minutes** | hours (basic graduation) and up |
| Flow | diagnose first, task-driven, only the 2–3 most relevant | systematic, by course number |
| For | cold start, want the first result now | want the agent systematically honed |
| Discipline | **both: real execution, proctor, skill cards, deployed to your machine — equally strict** | same |

> Onboarding is "start the job, learn as you go"; full study is "systematic training." They don't
> conflict: **onboard first to taste the payoff, then go to enroll for depth.** Courses taken during
> onboarding are distilled into the skill library, ready to thicken later.
