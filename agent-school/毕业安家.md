# 🏠 毕业安家 · 把技能卡装到用户的全局技能目录(写给毕业的 agent)

> 🌐 中文 ｜ [English](graduation-deploy.en.md)
>
> **这是上学闭环的最后一公里。** 你在宿舍 `skills/` 里沉淀的技能卡,如果只留在学校(仓库里),
> 那么**换一个项目目录、换一个新会话,它就失效了**——等于白学。
> 「毕业安家」就是:把技能卡**装到用户电脑的全局技能目录**,让它离开仓库后依然生效、随叫随到。

---

## 🛑 头号红线:写用户目录,必须先征得同意

把任何东西写进**仓库以外的位置**(尤其 `~/.claude/skills/` 这类用户全局目录),
**必须先明确征得用户同意**。未经同意就写用户目录 = 违反[校规](校规.md)。

开口模板:
> "我想把这趟学到的 N 张技能卡装到你的全局技能目录(`~/.claude/skills/`),
> 这样以后任何新对话、任何项目里我都能直接调用它们。可以吗?(随时能删)"

用户点头,才进入下面的流程。用户拒绝,就把技能卡留在宿舍并告诉他手动安装方法。

---

## 安家流程(四步)

### 第一步:确定落地位置

按优先级:

1. **Claude Code 用户 → `~/.claude/skills/`**(首选)。这是 Claude Code 的**原生技能目录**:
   放进去的技能会被自动发现,还能用 `/技能名` 直接调用。落地价值最高。
2. **其他 agent 工具 / 用户有偏好 → 问用户**。常见替代:`~/.agentforge/skills/`
   (本校的跨工具技能库,配合[记忆植入器](../deploy/agentforge-memory/)的反射加载)。
3. 用户给了自定义位置,就用他指定的。

> 💡 **两套机制协同,不冲突**:`~/.claude/skills/` 是 Claude Code 原生调用;
> `~/.agentforge/skills/` 是跨工具(opencode/OpenClaw/Hermes…)的反射库。
> 条件允许就**两处都装**——Claude Code 原生可调 + 其他工具靠反射查。

### 第二步:转成 SKILL.md 格式落地

`~/.claude/skills/` 的每个技能是**一个文件夹**,里面一个 `SKILL.md`,**开头带 YAML frontmatter**:

```
~/.claude/skills/
  agentforge-J03-verify/
    SKILL.md
```

`SKILL.md` 模板:

```markdown
---
name: agentforge-J03-verify
description: 交付前自检——任何"我做完了"之前先真跑、拿真实输出再交差。来自 AgentForge J03。
---

# 交付前自检(AgentForge J03 · 本机实测校准)

(把宿舍那张技能卡的正文搬进来:怎么做的步骤 + 本机校准 + 关键坑)
```

> `description` 要写清「什么时候该用这张卡」——Claude 靠它判断何时自动调用,务必具体。

### 第三步:命名规范 + 冲突处理

- **命名**:`agentforge-{课程编号}-{技能名}`(如 `agentforge-T17-ffmpeg`)。
  **`agentforge-` 前缀是硬性要求**,避免和用户已有的技能撞名。
- **冲突处理**:目标位置已有同名技能 →
  **绝不覆盖**。改名加 `-v2`(如 `agentforge-T17-ffmpeg-v2`),并**提示用户二选一**:
  > "你已经有一个同名技能了,我没覆盖,把我这张存成了 `-v2`。你看保留哪个,或要我合并?"

### 第四步:登记 + 验证(留可追溯证据)

1. **登记**:安家完成后,在宿舍 `report-card.md` 的「技能安家清单」里,
   逐张记录:技能名 / 最终安装路径 / 本机验证命令。这是可追溯的证据,不是口头说装了。
2. **验证**:**新开一个会话**(或让用户新开),演示其中**至少一张**技能卡能被真实调用
   —— Claude Code 用户:`/agentforge-xxx` 能唤出;其他工具:能 `ls ~/.agentforge/skills/` 查到并读取。
   把这次验证的结果也记进汇报。

---

## ✅ 安家自检清单

- [ ] 写用户目录前,**明确征得了用户同意**(有对话记录)
- [ ] 技能卡转成了带 frontmatter 的 `SKILL.md`,放进 `agentforge-{编号}-{技能名}/` 文件夹
- [ ] 命名带 `agentforge-` 前缀;遇到同名**没覆盖**,改 `-v2` 并提示了用户
- [ ] 在宿舍 `report-card.md` 登记了每张卡的**最终安装路径**
- [ ] **新会话里演示了至少一张技能卡可被调用**,并把结果汇报给用户

> 安家做完,你才算真正"把本事带回了用户的电脑"——而不是把它锁在学校的柜子里。
