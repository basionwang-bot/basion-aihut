# 技能:把大范围翻资料外包给子代理

- **什么时候用**:任务里出现"调研一下 XX""看看整个仓库怎么……"这种需要扫很多文件、但你最终只要一个结论的活。
- **来自**:第二课·管好你的工作记忆

## 步骤
1. 先把范围收窄成一个**具体问题**,而不是开放命题。
   - 反面(撑爆桌子):"研究一下这个仓库" —— 会逼你把几百个文件读进主上下文。
   - 正面(范围可控):"只回答:文章放哪个目录、frontmatter 有哪些字段、slug 怎么来、列表怎么排序" —— 答案有限,读的东西也有限。
2. 把这个收窄后的问题派给一个 Explore / general-purpose 子代理,明确要求它**只回结论、别回文件内容**。
3. 拿回结论,在自己干净的桌面上继续干。需要的话,对**同一个子代理**追问,而不是自己重新读一遍。

## 验证
- 干完后看自己的上下文:有没有为了一个结论而塞进一堆和当前动作无关的文件正文?有,就说明这次该外包没外包。

## 注意
- 同一个问题被纠正两次还不对,别在被失败方案污染的上下文里硬刚——清空,用更好的开头重开,通常更快。
- 派子代理前一定要先收窄范围;范围不收,子代理也会替你把它自己的桌子撑爆,结论质量照样差。
