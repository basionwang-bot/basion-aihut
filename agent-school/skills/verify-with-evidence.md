# 技能:交付前先跑尺子,再摆证据

- **什么时候用**:任何"我改完了"的时刻。准备说"应该没问题"之前。
- **来自**:第三课·先自检,再交作业

## 步骤
1. **找尺子**:先看 `package.json` 的 `scripts` 有没有 test / lint / build。本仓库只有 `build`,所以验证尺子就是 `npx next build`。
2. **真跑一遍**,别脑补输出。
3. **读输出**:确认关键信号——本仓库看 `✓ Compiled successfully`、`Finished TypeScript`、`Generating static pages (N/N)` 全过。
4. **摆证据交付**:把"我跑了什么命令 + 真实输出关键行"贴出来,而不是口头宣布成功。
5. 大改动 / 怕偏袒自己时,派一个干净上下文的子代理复查,只让它报真问题、不报风格偏好。

## 验证
- 差的交付:"我改好了,应该没问题。"
- 好的交付:"我改好了,跑了 `npx next build`,输出 `✓ Compiled successfully` + 17/17 静态页生成,验证通过。"

## 注意
- 绝不为了让报错消失而把检查关掉、把错误吞掉——改根因。
- 没有现成尺子时,自己造一把能跑的(最小复现脚本 / 起服务点一下),也别用"我觉得"交差。
