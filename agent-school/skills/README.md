# 🎒 技能库 · 你上学带回来的装备

> 这个文件夹,是你(agent)上学的**实物成果**。
> 每过一门课,你就要在这里**沉淀一个可复用的技能**——把刚学会的做法,写成一份下次能直接拿来用的"操作卡"。
> 学得越多,装备越多。这也是**用户能直接看到、能直接受益**的东西。

---

## 为什么要有技能库?

光"学会"是会忘的、也是用户看不见的。
但如果你每学一门课,就把精华**沉淀成一个文件留在这儿**,那么:

- **用户能摸得着**:打开这个文件夹,就看到你这趟上学到底带回了什么;
- **你能反复用**:下次遇到同类任务,直接调用这张操作卡,不用重新摸索;
- **越滚越强**:技能库会随着你学的课、干的活不断变大——这正是"一个人也是一支队伍"的底气。

---

## 每张技能卡怎么写

文件名用 `技能名.md`,内容用这个模板:

```markdown
# 技能:<一句话名字>

- **什么时候用**:<触发场景,越具体越好>
- **来自**:<哪门课 / 哪次实战>

## 步骤
1. ...
2. ...
3. ...(每步尽量写成可直接执行的动作)

## 验证
- 怎么确认这次做对了(可执行的检查)

## 注意
- 容易踩的坑
```

---

## 上学要求

- **第一~四课**:每过一门,至少沉淀 **1 张**与该课主题相关的技能卡。
- 毕业时,这个文件夹里**应该有 4 张以上**你亲手写的技能卡。
- 在[结业报告](../templates/graduation-report-template.md)里,把这些技能卡列成"带回的装备清单"给用户看。

> 起步可以从最朴素的开始,比如《在本仓库新增一篇文章并验证站点没坏》。
> 关键不是多高级,而是**真的能复用、用户真的能看到**。
