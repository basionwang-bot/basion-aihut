# 技能:动手前先探一遍地形

- **什么时候用**:接到一个"不止改一行"的任务,而且对这片代码不熟时。改错别字、加一行日志这种一句话能说清的小活,跳过本卡直接干。
- **来自**:第一课·先探索,再动手

## 步骤
1. 先别碰任何文件。用 `ls` / `find` / glob 摸清目录结构,定位"这类东西放在哪"。
2. 找到"加工现有内容的那段代码"(数据加载器 / 路由 / 配置),读懂它对输入有什么硬性要求。本仓库就是 `src/lib/posts.ts`(文章加载器)和 `src/lib/course.ts`(课程目录)。
3. 抄一份现有同类样本看格式(本仓库:任取一篇 `content/posts/*.md` 看 frontmatter)。
4. 把结论写成一句话计划:**哪个文件、什么格式、放哪、还要顺手改哪个登记表**。讲给人听,让他拍板,再动手。

## 验证
- 你能不能用一句话说清这次改动落在"哪个文件、什么字段、什么命名"?说不清,说明还没探够。

## 注意
- 最大的坑:只新建内容文件,忘了"还有一张手维护的登记表要同步"。本仓库里 `claude-code-*` 系列文章,光建 `content/posts/xx.md` 不够——还得往 `src/lib/course.ts` 的 `courseEntries` 里加一行,否则目录页和上一篇/下一篇导航里都看不到它。普通(非课程)文章则不需要。
