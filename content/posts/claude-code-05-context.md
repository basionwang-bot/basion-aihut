> **本模块教学目标：** 掌握"记忆管理"的全套操作。这是所有规律的核心，也是高手和普通用户最大的差距所在。

---

## 回顾那张"桌子"

还记得开篇那个比喻吗？Claude 干活时脑子里有一张**有限大小的桌子**，你们的对话、它读过的文件、命令输出，全堆在上面。桌子堆满了，它就开始忘事、抓不住重点、犯低级错。

**这一篇，就是教你怎么收拾这张桌子。** 学会了，你就跨进高手的门槛。

---

## 一、纠偏与回滚：随时把它拉回正轨

### `Esc` —— 随时打断

它正在往错的方向跑？按 **`Esc`** 立刻打断。**上下文会保留**，你直接补一句话重新引导就行，不用从头来。

### `Esc + Esc` 或 `/rewind` —— 打开回滚菜单

连按两下 `Esc`（或输入 `/rewind`），打开回滚菜单。你可以选择：

- **只回滚对话**（代码保留）
- **只回滚代码**（对话保留）
- **两者都回滚**

每条提示词都会**自动创建一个检查点**，而且跨会话保留——相当于一颗"后悔药"。

> ⚠️ **重要提醒：** 回滚只追踪 **Claude 自己的改动**，它**不能替代 git**。你手动改的、或者更复杂的版本管理，还得靠 git。把 `/rewind` 当"小范围撤销"，把 git 当"正经版本控制"。

### "撤销刚才的改动" —— 直接说

最简单的方式：直接打字让它自己退回去。

```text
撤销你刚才对登录逻辑的改动
```

## 二、🌟 核心规律（本模块最重要的一段，请反复读）

> **同一个问题，纠正它两次还不对，就别再纠第三次了。**

为什么？因为这时候，**它的工作记忆已经被一堆失败方案污染了。** 桌子上堆满了"试过 A 不行、试过 B 不行、又试 C 还不行"的垃圾，它越想越乱，你越纠越糟。

正确做法：

1. **`/clear` 清空**——把那张乱桌子整个掀干净。
2. 把刚才两次失败**学到的教训**，写进一个**更好的初始提示词**。
3. 用这个新提示词，**开个干净会话**重来。

> **一个干净会话 + 一个更好的提示词，几乎总是胜过一个堆满纠错记录的长会话。**

新手最反直觉、但最值钱的一课就是这个：**有时候"重开"比"硬刚"快得多。** 别舍不得那段对话——它已经是负资产了。

## 三、日常记忆操作（高频命令）

| 命令 | 干什么 | 什么时候用 |
|------|--------|------------|
| `/clear` | 清空全部上下文 | **不相关的两个任务之间，必清。** 这是好习惯里的第一条。 |
| `/compact 聚焦 API 改动` | 定向压缩，告诉它保留什么 | 任务还没完但记忆快满了，压缩一下接着干 |
| `/btw` | 问个小问题，不进对话历史 | 临时想问点别的，又不想污染当前任务 |
| `/rename` | 给会话命名（如 `oauth-migration`） | 把会话当 git 分支管理 |

补充说明：

- **在 CLAUDE.md 里写压缩规则**：比如写一句"压缩时永远保留改动文件清单和测试命令"，这样每次 `/compact` 都不会丢掉关键信息。（CLAUDE.md 是什么？模块六详讲。）
- **`/btw` 的妙处**：答案在一个浮层里看完即走，**完全不占当前任务的上下文**。想随手查点东西又不想"弄脏桌子"时，神器。
- **会话当分支管理**：`/rename` 命名后，用 `claude --continue` 续上最近一次，用 `claude --resume` 从列表里挑一个继续。多个任务并行时，这套管理方式能救你的命。

## 四、🔍 子代理（Subagent）调研法：读一堆文件却不弄脏你的桌子

这是上下文管理里最优雅的一招，新手一定要学会。

**问题场景：** 你想让它"调研一下我们的认证系统怎么处理 token 刷新"。但"调研"意味着它要读一大堆文件——读完，你的主对话桌子就被几十个文件塞满了，后面没法干正事了。

**解法：派一个子代理去读。**

```text
用子代理去调研我们的认证系统怎么处理 token 刷新，
以及有没有现成的 OAuth 工具类可以复用。
```

子代理会在**它自己独立的桌子上**读那一堆文件，读完**只把结论汇报回主对话**——那几十个文件的原文，全留在子代理那边，**不进你的主上下文**。

> 💡 一句话理解：**让别人在别的房间里翻完资料，只回来跟你说结论。** 你的桌子始终干净，专注于实现。

---

## ✍️ 学员练习

刻意练习这三个习惯，做满一周：

1. **每换一个不相关任务，先 `/clear`。** 强迫自己养成肌肉记忆。
2. **下次又陷入"纠正第二次还不对"时，忍住别纠第三次**——`/clear`，把教训写成更好的提示词重开。记录一下这次重开是不是比硬刚快。
3. **下次要做"大范围调研"时，用子代理去读**，体会主对话保持干净的好处。

## 本模块一句话总结

> **管好那张桌子，你就赢了一半。** 不相关任务之间 `/clear`；纠两次还不对就清空重来；大范围调研派子代理去读。记忆干净，表现才稳。
