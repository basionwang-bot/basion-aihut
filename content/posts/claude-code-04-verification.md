> **本模块教学目标：** 让 Claude 自己检查自己。这是"助手"和"员工"的真正分界线。

---

## 一个扎心的问题：你怎么知道它干对了？

新手的典型流程是这样的：让 Claude 干活 → 它说"完成了！" → 你**自己跑一遍 / 自己看一遍**发现有 bug → 你回去让它改 → 它又说"完成了！" → 你又得自己验……

发现问题了吗？**你成了人肉验证环节。** 每一个错误，都得等你亲自发现。这样的会话，你一秒都不敢走开。

核心原则一句话：

> **给 Claude 一个它自己能跑的检查（测试、构建、截图对比）。没有这个检查，"看起来完成了"就是它唯一的信号——而那个信号，靠不住。**

有了自检手段，区别立刻出现：

| | 没有验证手段 | 有验证手段 |
|---|---|---|
| 它的"完成"信号 | "我觉得写好了"（不可靠） | "测试全过了，这是输出"（可靠） |
| 你的角色 | 人肉测试机，不能走开 | 主管，可以走开 |
| 错误何时被发现 | 等你亲自发现 | 它自己当场发现并修复 |

**这就是"你盯着的会话"和"你能走开的会话"的区别。** 而"能走开"，正是通往模块七（自动化）的门票。

## 三个改写示范（给它验证标准）

### 示范 1：给出明确的验证标准

| | |
|---|---|
| ❌ 之前 | "实现一个邮箱校验函数" |
| ✅ 之后 | "写 `validateEmail` 函数。**测试用例：`user@example.com` 为 true，`invalid` 为 false，`user@.com` 为 false。实现后跑测试。**" |

你把"怎么算对"明明白白列出来，它就能自己跑、自己确认。

### 示范 2：UI 任务用"视觉验证"

| | |
|---|---|
| ❌ 之前 | "把仪表盘做好看点" |
| ✅ 之后 | "[贴设计图] 照这个实现。**完成后截图，和原图对比，列出差异并修复。**" |

"好看"是没法验证的主观词；"截图对比设计图、列差异"是它自己能执行的客观检查。

### 示范 3：修根因，不修症状

| | |
|---|---|
| ❌ 之前 | "构建失败了" |
| ✅ 之后 | "构建报这个错：[贴报错]。修复它并**验证构建通过**。要**解决根本原因，不许把报错压掉**。" |

最后那句很重要——不加的话，它可能为了"让错误消失"而把检查关掉、把报错吞掉，治标不治本。

## 验证的四个层级（由轻到重）

不是每个任务都要上最重的手段。按需选择：

1. **提示词内验证（最轻）** —— 就在同一条消息里要求它"**跑检查并迭代到通过**"。日常最常用，写一句话的事。

2. **会话级验证** —— 用 `/goal` 设定一个达标条件。每轮结束后，由一个独立的评估器复查，不达标就继续干。适合一个会话要反复打磨到某个标准的场景。

3. **确定性门槛（硬卡）** —— 用 **Stop hook** 跑你的检查脚本：**不通过就不许它结束这一回合。** 这是"铁律"级别的保障（连续被拦 8 次后系统会放行，避免死循环）。配置方法见模块六、模块七。

4. **第二意见（最重）** —— 让一个**干净上下文**的子代理或独立验证流程来挑错。

   > 为什么要"干净上下文"？因为**干活的人不能给自己打分。** 刚写完代码的会话会"偏袒"自己的产出，下意识觉得"我写的没问题"。换一个没有偏见的新视角，才挑得出真问题。

新手建议：**先把第 1 层用熟**——养成每次都加一句"跑测试并修到通过"的习惯，你的体验立刻上一个台阶。

---

## 📌 配套要求：让它"出示证据"，而不是口头宣布成功

这是很多人忽略的一招。别接受"我完成了"这种空口白话，要求它**拿证据来**：

- 测试输出（实际跑出来的结果，不是它说"应该会过"）
- 跑了什么命令、返回了什么
- 结果截图

```text
完成后，把你跑的命令和实际输出贴给我，别只说"做好了"。
```

**好处有两个：**

1. 看证据比你自己重跑一遍验证**快得多**。
2. 这招对你**没盯着的会话**同样有效——回来一看证据就知道靠不靠谱。

---

## ✍️ 学员练习

把你常给 Claude 的一个任务，改造成"带验证"的版本：

1. 原指令是什么？（大概率没有验证标准）
2. 它"怎么算做对了"？把这个标准写成它**自己能跑的检查**（测试用例 / 构建通过 / 截图对比）。
3. 在指令末尾加一句"跑检查并迭代到通过，完成后把输出贴给我"。

跑一次，体会"我不用自己验了"的解放感。

---

## 本模块一句话总结

> **没有自检手段的任务，你永远走不开。** 给它一个它自己能跑的检查，并要求它出示证据——这是从"AI 助手"升级到"AI 员工"的那道门槛。
