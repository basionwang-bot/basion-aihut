## 先说人话：这门课到底教什么？

你大概率已经用过 AI 聊天（DeepSeek、豆包、ChatGPT）。那种体验是：**你问，它答，你复制粘贴。**

Claude Code 不是这个东西。它是一个**能动手的 AI**：

- 它能**读你电脑里的文件**（不是你贴给它，是它自己去翻）
- 它能**跑命令**（装依赖、跑测试、提交代码）
- 它能**自己改文件、自己验证、自己迭代**
- 你可以盯着它干，随时喊停纠正；也可以**直接走开**，回来收结果

一句话：**聊天 AI 是"顾问"，Claude Code 是"能上手干活的初级员工"。**

这门课的全部目的，就是教你怎么把这个"员工"管好——管到一个人能干出一支小团队的活。

> ⚠️ 注意：这门课假设你会用 Claude Code 这个**命令行工具**（在终端里输入 `claude` 启动那种）。如果你连"终端"是什么都还没概念，建议先看小屋里那篇《AI 工具总览》，从 Cursor 这类图形界面工具入门，再回来学这门课。

## 整门课的底层逻辑（只有一句话，但贯穿始终）

请把这句话刻进脑子，后面所有技巧都是它的推论：

> **🧠 Claude 的"上下文窗口"（Context Window）就是它的工作记忆。填得越满，它表现越差。**

什么叫"上下文窗口"？

打个比方：Claude 干活的时候，脑子里有一张**有限大小的桌子**。你说的每句话、它读过的每个文件、它跑命令返回的每一行输出，**全都堆在这张桌子上**。

桌子就那么大。堆太满了会怎样？

- 它开始**忘事**（前面说好的规则，后面不认了）
- 它开始**抓不住重点**（被一堆无关信息淹没）
- 它开始**犯低级错误**（明明文件里写着，它却没看见）

所以，**高手和普通用户的差距，根本不在"会不会写提示词"，而在"会不会管理这张桌子"。**

几乎所有技巧，本质都在回答三个问题：

1. 怎么让它**第一次就听懂**？（→ 模块二：精准提问）
2. 怎么让它**自己验证自己**？（→ 模块四：验证闭环）
3. 怎么管理它的**记忆与注意力**？（→ 模块五：上下文管理）

把这三件事做好的人，才有资格进第四阶段：**自动化与规模化**——让好几个 Claude 同时替你干活（模块七）。

## 学习路线图：你该按什么顺序学？

这门课一共八个模块，但**你不必一口气全学**。按下面这张表对号入座：

| 你是谁 | 推荐路线 | 学完能干嘛 |
|--------|----------|------------|
| **完全新手 / 非技术 / 老师、独立创业者** | 模块 ① ② ③ ⑧ | 能稳定地让 Claude 帮你完成一件件具体的事，不返工、不翻车 |
| **想认真提效的个人** | 加学 ④ ⑤ ⑥ | 能让 Claude 自己验证、自己管记忆，配好你的专属工作环境 |
| **想"一个人开一支队伍"的狠人** | 再加 ⑦ | 能让 Claude 在你睡觉时自动干活、批量处理、多会话并行 |

**建议所有人都先把 ①②③⑧ 这四块吃透。** 这是地基。地基不稳，后面学再多花活都是空中楼阁。

## 这门课的每一篇长什么样

为了让新手真的看得懂，我把官方那份"专家向"的密集大纲，**逐条翻译成了人话**，每篇都按这个结构展开：

- **这是什么 / 为什么重要**（先讲清楚动机，不让你死记）
- **怎么做**（一步一步，能照着操作）
- **现场例子**（真实场景，不用玩具案例）
- **新手最容易踩的坑**（提前帮你避雷）

## 八个模块速览（点进去看详细版）

1. **模块一·换脑子** —— 从"下指令"到"管理一个干活的人"。建立正确心智，这是一切的前提。
2. **模块二·提问的精度** —— 提问越准，返工越少。四个改写模板 + 让 AI 反过来采访你的杀手锏。
3. **模块三·黄金工作流** —— 探索 → 计划 → 实现 → 提交。官方推荐的标准四步，以及"什么时候别用它"。
4. **模块四·验证闭环** —— 让 Claude 自己检查自己。这是"助手"和"员工"的分界线。
5. **模块五·上下文管理** —— 管理那张"桌子"的全套操作。高手和普通人的真正分水岭。
6. **模块六·环境配置** —— 一次配置，长期省力。CLAUDE.md、权限、Skills、Hooks、MCP。
7. **模块七·自动化** —— 让 Claude 在你不在场时干活。定时任务、批量扇出、多会话并行。
8. **模块八·五大翻车现场** —— 官方总结的五个常见失败，每个配"症状 + 解药"。最适合新手提前看。

最后还有一篇 **《附录·命令速查表 + 官方资料》**，建议收藏，随时翻。

## 给新手的三句心里话

1. **别想着"一句话让它干完一切"。** Claude Code 强在"被你管得好"，不强在"读心术"。你越会拆任务、越会给标准，它越神。
2. **翻车是正常的，关键是会复盘。** 模块八的五个坑，你大概率每个都会踩一遍。踩了别慌，回来看解药。
3. **从今天起，把它当"同事"而不是"工具"。** 你怎么跟一个靠谱的初级同事交代活，就怎么跟它说话。这个心态转过来了，这门课就成功了一半。
